7.5 Stable Diffusion Deep Dive

Stable Diffusion is latent diffusion: it denoises compressed image latents instead of raw pixels, while text embeddings guide the generation.

Prerequisites7.3 Diffusion, 7.1 VAE, 5.3 Attention
Estimated time80-110 min
Job relevanceHigh for GenAI/CV roles
Research depthSolid

The full pipeline

1. Text promptTokenized and encoded by a CLIP text encoder.
2. Random latentStart from Gaussian noise in compressed latent space.
3. UNet denoiserPredicts noise conditioned on text and timestep.
4. SchedulerUpdates the latent through reverse diffusion steps.
5. VAE decoderDecodes final latent into pixels.

The key engineering idea is not only diffusion. It is where diffusion happens: in a smaller latent space, which makes training and sampling much cheaper than pixel-space diffusion.

Latent diffusion objective

Instead of noising an image x directly, Stable Diffusion first encodes it into a latent z with a VAE encoder. Diffusion then learns denoising in that latent space.

L = E || epsilon - epsilon_theta(z_t, t, c) ||^2
Что означают символы
zimage latentCompressed representation produced by the VAE encoder.
z_tnoisy latentLatent after t noise steps.
cconditioningText embedding from the prompt, usually from a CLIP text encoder.
epsilon_thetaUNet predictionPredicted noise for the current latent, timestep and text condition.
Интуиция: the model learns which part of a noisy compressed image is noise, using the prompt as a guide.
Аналогия: like restoring a blurred sketch while reading a written description of what it should become.
Числовой пример: if the VAE compresses 512x512x3 pixels into 64x64x4 latents, diffusion operates on about 16k values instead of 786k pixels.

Why latent space matters

Pixel-space diffusion is expensive because every denoising step touches every pixel. Latent diffusion uses a learned compression where perceptually important structure is preserved but the spatial size is much smaller.

  • VAE encoder: image to latent. Used during training and image-to-image workflows.
  • UNet: denoises latent variables, not RGB pixels.
  • VAE decoder: latent to final image. Used after sampling.
Что это значит на практике
Most generation quality comes from the denoising UNet and conditioning. The VAE controls the visual bottleneck: if the VAE cannot encode fine details, the diffusion model cannot reliably recover them at the end.

Text conditioning through cross-attention

The text prompt is not just appended as a label. UNet blocks use cross-attention: image latent features ask questions, text token embeddings provide keys and values.

CrossAttn(Q_img, K_text, V_text) = softmax(Q_img K_text^T / sqrt(d)) V_text
Что означают символы
Q_imgimage queriesLatent image features asking what text tokens are relevant.
K_texttext keysToken descriptors used for matching image locations to words.
V_texttext valuesInformation injected into image features after attention weights are computed.
sqrt(d)scale factorPrevents attention logits from becoming too large as hidden dimension grows.
Интуиция: each image region can selectively pull information from words in the prompt.
Аналогия: like an artist repeatedly checking different words of the brief while refining different parts of the sketch.
Числовой пример: a region that attends strongly to "red" and "car" receives text information that biases it toward red car-like features.

Classifier-free guidance

CFG makes the sample follow the prompt more strongly by comparing conditional and unconditional noise predictions.

eps_hat = eps_uncond + s * (eps_cond - eps_uncond)
Что означают символы
eps_condconditional predictionNoise prediction when the prompt is provided.
eps_uncondunconditional predictionNoise prediction with an empty or dropped condition.
sguidance scaleHow aggressively the sample is pushed toward the prompt. Common values: 5-12.
eps_hatguided predictionFinal noise estimate used by the scheduler step.
Интуиция: CFG amplifies the difference between "generic image" and "image matching this prompt".
Аналогия: like turning up the prompt volume: clearer instruction, but too loud can distort the result.
Числовой пример: if eps_uncond=1.0, eps_cond=0.7 and s=7, then eps_hat=1+7*(-0.3)=-1.1.

Common mistakes

  • Thinking CLIP generates the image. CLIP text embeddings condition the denoising UNet; the UNet does generation.
  • Ignoring the VAE bottleneck. Some artifacts come from latent compression/decoding, not from diffusion itself.
  • Setting CFG too high. Higher prompt adherence can reduce realism, diversity and composition quality.

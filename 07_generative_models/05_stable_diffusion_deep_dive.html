7.5 Stable Diffusion Deep Dive

Stable Diffusion is latent diffusion: it denoises compressed image latents instead of raw pixels, while text embeddings guide the generation.

Почему важно

Stable Diffusion — это диффузия в латентном пространстве VAE. Latent Diffusion + CLIP text conditioning + U-Net с cross-attention = SOTA генерация изображений по тексту.

Что узнаешь
  • Latent diffusion — почему диффузия в z-space быстрее, чем в pixel-space
  • Text conditioning через cross-attention между image и text embeddings
  • Classifier-Free Guidance — как усилить соответствие промпту
  • ControlNet и LoRA для SD — адаптация pre-trained модели

The full pipeline

1. Text promptTokenized and encoded by a CLIP text encoder.
2. Random latentStart from Gaussian noise in compressed latent space.
3. UNet denoiserPredicts noise conditioned on text and timestep.
4. SchedulerUpdates the latent through reverse diffusion steps.
5. VAE decoderDecodes final latent into pixels.

The key engineering idea is not only diffusion. It is where diffusion happens: in a smaller latent space, which makes training and sampling much cheaper than pixel-space diffusion.

Latent diffusion objective

Instead of noising an image x directly, Stable Diffusion first encodes it into a latent z with a VAE encoder. Diffusion then learns denoising in that latent space.

L = E || epsilon - epsilon_theta(z_t, t, c) ||^2
Что означают символы
zimage latentCompressed representation produced by the VAE encoder.
z_tnoisy latentLatent after t noise steps.
cconditioningText embedding from the prompt, usually from a CLIP text encoder.
epsilon_thetaUNet predictionPredicted noise for the current latent, timestep and text condition.
Интуиция: the model learns which part of a noisy compressed image is noise, using the prompt as a guide.
Аналогия: like restoring a blurred sketch while reading a written description of what it should become.
Числовой пример: if the VAE compresses 512x512x3 pixels into 64x64x4 latents, diffusion operates on about 16k values instead of 786k pixels.

Why latent space matters

Pixel-space diffusion is expensive because every denoising step touches every pixel. Latent diffusion uses a learned compression where perceptually important structure is preserved but the spatial size is much smaller.

  • VAE encoder: image to latent. Used during training and image-to-image workflows.
  • UNet: denoises latent variables, not RGB pixels.
  • VAE decoder: latent to final image. Used after sampling.
Что это значит на практике
Most generation quality comes from the denoising UNet and conditioning. The VAE controls the visual bottleneck: if the VAE cannot encode fine details, the diffusion model cannot reliably recover them at the end.

Text conditioning through cross-attention

The text prompt is not just appended as a label. UNet blocks use cross-attention: image latent features ask questions, text token embeddings provide keys and values.

CrossAttn(Q_img, K_text, V_text) = softmax(Q_img K_text^T / sqrt(d)) V_text
Что означают символы
Q_imgimage queriesLatent image features asking what text tokens are relevant.
K_texttext keysToken descriptors used for matching image locations to words.
V_texttext valuesInformation injected into image features after attention weights are computed.
sqrt(d)scale factorPrevents attention logits from becoming too large as hidden dimension grows.
Интуиция: each image region can selectively pull information from words in the prompt.
Аналогия: like an artist repeatedly checking different words of the brief while refining different parts of the sketch.
Числовой пример: a region that attends strongly to "red" and "car" receives text information that biases it toward red car-like features.

Classifier-free guidance

CFG makes the sample follow the prompt more strongly by comparing conditional and unconditional noise predictions.

eps_hat = eps_uncond + s * (eps_cond - eps_uncond)
Что означают символы
eps_condconditional predictionNoise prediction when the prompt is provided.
eps_uncondunconditional predictionNoise prediction with an empty or dropped condition.
sguidance scaleHow aggressively the sample is pushed toward the prompt. Common values: 5-12.
eps_hatguided predictionFinal noise estimate used by the scheduler step.
Интуиция: CFG amplifies the difference between "generic image" and "image matching this prompt".
Аналогия: like turning up the prompt volume: clearer instruction, but too loud can distort the result.
Числовой пример: if eps_uncond=1.0, eps_cond=0.7 and s=7, then eps_hat=1+7*(-0.3)=-1.1.

Common mistakes

  • Thinking CLIP generates the image. CLIP text embeddings condition the denoising UNet; the UNet does generation.
  • Ignoring the VAE bottleneck. Some artifacts come from latent compression/decoding, not from diffusion itself.
  • Setting CFG too high. Higher prompt adherence can reduce realism, diversity and composition quality.

Пошагово Stable Diffusion Inference Pipeline — from prompt to pixel
1 /
Шаг 01

Encode the prompt with the CLIP text encoder

Tokenize the input text and pass it through a frozen CLIP text transformer. The output is a sequence of token embeddings (e.g. 77 tokens × 768 dimensions for CLIP ViT-L/14). These embeddings are not a single vector — they retain per-token granularity so that each word can influence different spatial regions of the image through cross-attention.

Шаг 02

Initialize latent z_T ~ N(0,I) in 4×64×64 space

Sample a random noise tensor of shape [4, 64, 64] — this is the starting point for generation of a 512×512 image. The factor of 8 spatial compression comes from the VAE encoder (512/64 = 8). The 4 channels correspond to the VAE's learned latent features. The denoising loop will iteratively refine this noise tensor over T steps.

Шаг 03

U-Net predicts noise conditioned on text embedding

The U-Net receives the noisy latent z_t, the current timestep t (as a sinusoidal embedding), and the CLIP token embeddings. Residual blocks handle the latent features; cross-attention layers let latent patch features query the text token embeddings. The U-Net outputs a predicted noise tensor ε̂ of the same [4, 64, 64] shape. Classifier-free guidance then combines conditional and unconditional predictions: ε̂_guided = ε̂_uncond + s·(ε̂_cond − ε̂_uncond).

Шаг 04

DDIM step updates z, repeat T times

Apply the DDIM update rule to compute z_{t-1} from z_t and ε̂_guided. DDIM is deterministic: given the same initial z_T and same noise seed, you always get the same output. Repeat this U-Net forward pass and update for each timestep from T down to 1, typically 20–50 steps. Each step moves the latent from pure noise toward a structured representation matching the prompt.

Шаг 05

VAE decoder maps z_0 to pixel space

After the final denoising step, the clean latent z_0 of shape [4, 64, 64] is passed through the VAE decoder. The decoder upsamples the latent 8× spatially and converts the 4 latent channels into RGB pixels, producing the final 512×512×3 image. The VAE decoder is the only part of the pipeline operating in pixel space — all diffusion happens in the compressed latent space.

Проверь себя

Stable Diffusion — latent space, CFG, and VAE

  1. Why does Stable Diffusion work in latent space rather than pixel space?

    A 512×512×3 image has ~786k values. Diffusion in pixel space requires the U-Net to process and denoise all of them at every timestep, which is extremely expensive. The VAE compresses this to 64×64×4 ≈ 16k values — about 48× fewer. This makes both training and inference dramatically faster with comparable quality, because the VAE has already learned to preserve perceptually important structure in the latent representation.

  2. What is classifier-free guidance and how does it strengthen prompt adherence?

    During training, the conditioning text c is randomly dropped (replaced with an empty token) with ~10–20% probability, so the model learns both conditional ε_θ(z_t, t, c) and unconditional ε_θ(z_t, t, ∅) noise prediction. At inference, CFG extrapolates: ε̂ = ε_uncond + s·(ε_cond − ε_uncond). The scale s (typically 7–12) amplifies the direction from unconditional toward conditional prediction, making the output more closely match the prompt at the cost of diversity.

  3. What does the VAE encoder and decoder do in Stable Diffusion, and when is each used?

    The VAE encoder (used during training and image-to-image workflows) compresses a 512×512×3 pixel image into a 64×64×4 latent. The VAE decoder (used at the end of every inference run) expands the final denoised latent z_0 back into a 512×512×3 pixel image. In standard text-to-image generation, only the decoder is used at inference time — the encoder is not needed because the process starts from random latent noise rather than from an input image.

Что ломается без этого

Without classifier-free guidance (CFG scale = 1.0, meaning unconditional only), the U-Net denoises purely according to the general image distribution it learned, ignoring the text prompt almost entirely. The outputs are statistically plausible images but semantically incoherent with respect to the prompt — blurry, averaged-looking scenes with no specific subject or style. Conversely, setting CFG too high (e.g. scale = 20+) pushes the denoising direction so far toward the conditional prediction that small floating-point errors accumulate, producing oversaturated colors, burned-out highlights, and distorted anatomy.

Вопросы для самопроверки

  1. Что такое Latent Diffusion Model (LDM) и чем он отличается от Pixel Diffusion?

    LDM проводит диффузию в сжатом латентном пространстве автоэнкодера (обычно 8× сжатие). Это снижает вычислительную стоимость в 64× при той же сложности задачи. Stable Diffusion = LDM + CLIP text encoder. Качество сопоставимо с pixel-diffusion, но скорость намного выше.

  2. Как CLIP encoder интегрируется в Stable Diffusion?

    Текст кодируется CLIP text encoder в последовательность токен-эмбеддингов. U-Net диффузионной модели использует cross-attention: query — латентные патчи изображения, key и value — CLIP embeddings. Это позволяет изображению «обращаться» к семантике текста на каждом шаге денойзинга.

  3. Что такое VAE в Stable Diffusion и почему нужен именно он?

    VAE кодирует изображение 512×512×3 в латентный код 64×64×4 (encoder) и декодирует обратно (decoder). Диффузионный процесс работает только в 64×64×4 пространстве — это быстро. VAE нужен для финального «перевода» генерированного латента в пиксели без потери деталей.

  4. Что такое DreamBooth и как он персонализирует диффузионные модели?

    DreamBooth fine-tuning: обучаем всю модель на 3-20 фотографиях субъекта с уникальным идентификатором («a sks dog»). Модель учится ассоциировать идентификатор с конкретным субъектом. Prior preservation loss предотвращает catastrophic forgetting: параллельно генерируем обычных собак.

8.3 Profiling and Performance

Performance tuning начинается не с догадок, а с измерений. Нужно понять, где уходит время: загрузка данных, CPU preprocessing, transfer на GPU, forward, backward, optimizer или distributed communication.

Главная формула step time

step_time = data_loading + host_to_device + forward + backward + optimizer + synchronization + logging

Если оптимизировать не тот компонент, training почти не ускорится. Поэтому сначала измерение, потом изменение.

Что смотреть первым

GPU utilization
Если низкая, GPU ждет данные, CPU или synchronization.
Step time
Стабильный breakdown показывает, где bottleneck.
Memory
Пики memory важнее среднего значения: OOM случается на максимуме.

PyTorch profiler

# PyTorch profiler for a training loop.
# Input: train_loader returns x, y batches.
# Output: trace files viewable in TensorBoard.
import torch
from torch.profiler import profile, ProfilerActivity, schedule, tensorboard_trace_handler

with profile(
    activities=[ProfilerActivity.CPU, ProfilerActivity.CUDA],
    schedule=schedule(wait=1, warmup=1, active=3),
    on_trace_ready=tensorboard_trace_handler("./profiler_logs"),
    record_shapes=True,
    profile_memory=True,
) as prof:
    for step, (x, y) in enumerate(train_loader):
        x, y = x.cuda(non_blocking=True), y.cuda(non_blocking=True)
        optimizer.zero_grad(set_to_none=True)
        loss = criterion(model(x), y)
        loss.backward()
        optimizer.step()
        prof.step()
        if step > 8:
            break

# Gotchas:
# 1. Skip warmup steps; first steps include one-time overhead.
# 2. Profiling itself adds overhead, so profile short windows.

Dataloader bottlenecks

Очень часто GPU простаивает не из-за модели, а из-за данных. Симптомы: sawtooth utilization, CPU на 100%, высокий time между steps, ускорение при `num_workers` tuning.

  • Увеличь `num_workers` и включи `pin_memory=True`.
  • Сократи тяжелые transforms в Python.
  • Кэшируй preprocessing, если он deterministic.
  • Проверь скорость диска и network storage.

Performance checklist

Быстрые wins
AMP/BF16, larger batch if memory allows, fused optimizer, compiled transforms, меньше logging внутри step.
Глубже
torch.compile, FlashAttention, better kernels, distributed overlap, async data prefetch.

Частые ошибки

  • Смотреть только average step time и игнорировать p95/p99.
  • Профилировать один первый batch, где много cold-start overhead.
  • Логировать изображения/большие tensors каждый step.
  • Менять несколько вещей одновременно и не понимать, что помогло.

Разбор ключевой формулы

throughput = samples or tokens / time
Что означают символы
throughputскорость обработкисколько samples или tokens система обрабатывает за единицу времени.
samples or tokensобъём работыединица полезной нагрузки для training или inference.
timeвремяреальное wall-clock время, включая data loading, kernels и синхронизации.
Интуиция: profiling нужен, чтобы понять, где именно тратится время, а не просто увидеть медленный run.
Числовой пример: если 4096 tokens обрабатываются за 0.5 секунды, throughput равен 8192 tokens/sec.

1.2 Матанализ

Производные · chain rule · gradient · Jacobian · Hessian · Taylor intuition · связь с backprop и оптимизацией

📈 Зачем матанализ в ML?

Обучение нейросети — это поиск таких параметров, при которых функция потерь становится минимальной. Чтобы понять, в какую сторону двигать веса, нам нужны производные.

θ ← θ - η ∇θL(θ)

Производная

Как быстро меняется функция при изменении переменной.

Градиент

Обобщение производной на много переменных.

Chain rule

Правило композиции, на котором стоит backpropagation.

💡 Одна фраза: матанализ в ML нужен, чтобы измерять чувствительность выходов и loss к параметрам модели.

🧠 Интуиция производной

Формальное определение

f'(x) = limh→0 (f(x+h) - f(x)) / h

Смысл

1. насколько быстро меняется функция в точке x
2. наклон касательной к графику
3. локальная чувствительность: маленькое изменение x → изменение f(x)
ℹ️ Если f′(x) > 0, функция локально растёт. Если f′(x) < 0, убывает. Если f′(x)=0, это кандидат на экстремум или седло.

🎛️ Интерактив: производная как наклон касательной

Выбери функцию и двигай точку x₀

Функция
x₀ = 1.00

Пошаговая интерпретация

💡 Касательная — это лучшая линейная аппроксимация функции в очень маленькой окрестности точки. Именно эта идея потом превращается в gradient descent и Taylor expansion.

⛓️ Chain Rule — фундамент backpropagation

Одна переменная

y = f(g(x)) ⟹ dy/dx = (dy/dg)(dg/dx) = f'(g(x)) · g'(x)

Интуиция

Если x влияет на g, а g влияет на y, то чувствительность y к x получается как произведение локальных чувствительностей на каждом шаге.

Интерактив: цепочка вычислений

Берём h(x)=sin(x²+1)

x = 1.20

Разложение по узлам

u = x²
v = u + 1
h = sin(v)
dh/dx = dh/dv · dv/du · du/dx
💡 Это и есть backprop в миниатюре: сначала считаем forward, потом идём назад по графу вычислений и перемножаем локальные производные.

🌊 Градиент функции многих переменных

Если функция зависит не от одного x, а от целого вектора x = (x₁, ..., xₙ), то обычная производная превращается в градиент.

f: ℝⁿ → ℝ,   ∇f(x) = [ ∂f/∂x₁, ∂f/∂x₂, ..., ∂f/∂xₙ ]⊤

Смысл градиента

1. направление наибольшего роста функции
2. его норма показывает, насколько быстро функция растёт
3. антиградиент -∇f указывает направление спуска
ℹ️ В оптимизации мы почти всегда движемся по -∇f, потому что хотим минимизировать loss.

🧮 Интерактив: gradient descent на квадратичной функции

Функция: f(x,y)=ax²+by²

a = 1.0
b = 2.0
η = 0.12

Что происходит

∇f(x,y) = [2ax, 2by]⊤
[x, y] ← [x, y] - η ∇f(x,y)
Если η слишком большой, можно начать прыгать через минимум или даже расходиться. Если слишком маленький — обучение будет очень медленным.

📚 Частные производные и Jacobian

Когда вход и выход — векторы, производная становится матрицей Jacobian.

f: ℝⁿ → ℝᵐ,   Jf(x) = [ ∂fᵢ/∂xⱼ ] ∈ ℝm×n

Пример

f(x,y) = [ x² + y, sin(xy) ]
J = [ 2x   1 ycos(xy)   xcos(xy) ]

Интерактив: Jacobian в точке

x = 1.00
y = 0.50

Смысл Jacobian

строки — как меняется каждый выход
столбцы — по каким входам измеряется чувствительность
локально Jacobian — лучшая линейная аппроксимация векторной функции
ℹ️ Jacobian постоянно скрыт внутри backprop, attention, нормализаций, softmax и вообще любых векторных преобразований.

🏔️ Hessian и кривизна

Если градиент — это вектор первых производных, то Hessian — матрица вторых производных.

Hf(x) = [ ∂²f / ∂xᵢ∂xⱼ ]

Что показывает Hessian

1. локальную кривизну поверхности
2. насколько быстро меняется сам градиент
3. min / max / saddle через знаки собственных значений
СлучайСобственные значения HessianИнтерпретация
локальный минимумвсе > 0поверхность "чаша"
локальный максимумвсе < 0поверхность "купол"
седлоразных знаковв одном направлении вверх, в другом вниз

Интерактив: квадратичная форма

f(x,y)=ax²+bxy+cy²

a = 1.0
b = 0.0
c = 1.5

Формулы

∇f = [ 2ax + by, bx + 2cy ]
H = [ 2a   b b   2c ]
💡 Hessian особенно важен в теории оптимизации: он говорит не только куда идти, но и насколько "крутая" поверхность. Это связывает матанализ с Newton-type методами и curvature-aware оптимизацией.

🪜 Taylor expansion: линейная и квадратичная аппроксимация

Первый порядок

f(x + Δx) ≈ f(x) + f'(x)Δx

Для многих переменных

f(x + Δx) ≈ f(x) + ∇f(x)⊤Δx

Второй порядок

f(x + Δx) ≈ f(x) + ∇f(x)⊤Δx + 1/2 · Δx⊤HΔx
ℹ️ Первый порядок говорит о наклоне, второй — о кривизне. Поэтому gradient descent использует только первый порядок, а Newton-like методы — ещё и второй.

🔗 Как всё это связано с backprop

Пусть сеть — это композиция функций:

x → z¹ → a¹ → z² → a² → ... → L

Тогда по chain rule:

∂L/∂w = ∂L/∂a · ∂a/∂z · ∂z/∂w
forward pass строит вычислительный граф
backward pass применяет chain rule от конца к началу
в каждом узле считаются локальные производные
💡 Backpropagation — это не "магия нейросетей", а просто систематическое применение матанализа к большой композиции функций.

💻 NumPy / PyTorch: аналитика и autograd

import numpy as np # h(x) = sin(x^2 + 1) x = 1.2 u = x**2 v = u + 1 h = np.sin(v) # chain rule dh_dv = np.cos(v) dv_du = 1.0 du_dx = 2*x dh_dx = dh_dv * dv_du * du_dx print("h(x) =", h) print("dh/dx =", dh_dx)
import torch x = torch.tensor(1.2, requires_grad=True) h = torch.sin(x**2 + 1) h.backward() print("h =", h.item()) print("dh/dx =", x.grad.item()) # autograd сам построил вычислительный граф # и применил chain rule назад
import torch # f(x, y) = x^2 + 2y^2 xy = torch.tensor([2.0, -1.0], requires_grad=True) f = xy[0]**2 + 2*xy[1]**2 f.backward() print("f =", f.item()) print("grad =", xy.grad) # [2x, 4y] = [4, -4] # gradient descent step lr = 0.1 with torch.no_grad(): xy -= lr * xy.grad xy.grad.zero_() print("new point =", xy)

🎯 Что обязательно вынести из темы

Производная = локальная скорость изменения функции
Касательная = лучшая линейная локальная аппроксимация
Chain rule = основа backpropagation
Градиент = направление наибольшего роста
-∇f = направление спуска
Jacobian = производная векторной функции
Hessian = матрица кривизны
💡 Если совсем упростить: матанализ в ML — это наука о том, как маленькие изменения параметров распространяются по модели и влияют на итоговый loss.

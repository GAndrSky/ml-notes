4.2 Оптимизаторы

SGD → Momentum → Adam → AdamW — откуда берутся формулы и почему они работают

🗺️ Общая задача оптимизации

После backprop у нас есть градиент g = ∂L/∂θ. Оптимизатор решает: как именно обновить θ?

θ_{t+1} = θ_t − η · update(g_t, g_{t-1}, ...) update зависит от оптимизатора: SGD: update = g_t Momentum: update = экспон. скользящее среднее g Adam: update = нормированное EMA g / √(EMA g²)
💡 Все оптимизаторы решают одну задачу: двигаться вниз по loss surface, но делают это по-разному. SGD — прямолинейно. Momentum — с инерцией. Adam — адаптивно для каждого параметра.

1️⃣ SGD — Stochastic Gradient Descent

θ_{t+1} = θ_t − η · g_t g_t = ∂L_batch/∂θ (на мини-батче)

«Стохастический» — потому что используем случайный мини-батч, а не весь датасет. Это добавляет шум, но ускоряет сходимость и помогает выйти из локальных минимумов.

⚠️ Проблемы SGD:
• Одинаковый lr для всех параметров
• Медленная сходимость в «ущельях» (ravines)
• Осциллирует поперёк направления минимума
• Застревает у седловых точек

Размер батча vs lr

Batch size B: 32
💡 Linear scaling rule: при удвоении B → удваивай lr.
Градиент мини-батча ≈ среднее по B примерам — при большем B он точнее, но разнообразия меньше.

2️⃣ SGD + Momentum

v_t = β·v_{t-1} + g_t (EMA градиентов) θ_{t+1} = θ_t − η·v_t β обычно = 0.9

v_t — «скорость» (velocity). Аналог физической инерции: шарик катится вниз, набирает скорость, не меняет направление резко.


Раскрытие суммы (почему EMA?)

v_t = g_t + β·g_{t-1} + β²·g_{t-2} + ... = Σ_{k=0}^{t} β^k · g_{t-k} → экспоненциально взвешенное среднее прошлых градиентов с весами β^k
💡 Эффективное окно памяти ≈ 1/(1−β).
β=0.9: ~10 шагов. β=0.99: ~100 шагов.

Эффект momentum на ущелье

β = 0.90

3️⃣ Adam — Adaptive Moment Estimation

m_t = β₁·m_{t-1} + (1−β₁)·g_t ← 1й момент (EMA g) v_t = β₂·v_{t-1} + (1−β₂)·g_t² ← 2й момент (EMA g²) m̂_t = m_t/(1−β₁ᵗ) ← bias correction v̂_t = v_t/(1−β₂ᵗ) θ_{t+1} = θ_t − η · m̂_t / (√v̂_t + ε) Гиперпараметры: β₁=0.9, β₂=0.999, ε=1e-8
💡 m̂ / √v̂ — нормированный градиент.
Если g постоянно большой → v̂ большой → шаг маленький.
Если g маленький → v̂ маленький → шаг относительно большой.
Каждый параметр получает свой эффективный lr!

Bias correction — зачем?

При t=1, β₁=0.9: m_1 = 0.9·0 + 0.1·g_1 = 0.1·g_1 m̂_1 = 0.1·g_1 / (1−0.9¹) = 0.1·g_1/0.1 = g_1 ✓ Без коррекции m_1 = 0.1·g_1 → недооценка в 10×! Коррекция нужна только в начале (β^t → 0 при t→∞)

Интуиция: Adam = Momentum + RMSProp

Momentum: m̂_t (направление)
RMSProp: 1/√v̂_t (масштаб шага)
Adam: m̂_t / √v̂_t (оба вместе)

4️⃣ AdamW — Adam + правильный Weight Decay

Adam с L2 (неправильно!): g_t = ∂L/∂θ + λ·θ ← λθ попадает в EMA θ_{t+1} = θ_t − η·m̂/(√v̂+ε) Проблема: λθ масштабируется на 1/√v̂ ← некорректно AdamW (правильно): g_t = ∂L/∂θ ← чистый градиент θ_{t+1} = θ_t − η·m̂/(√v̂+ε) − η·λ·θ_t ↑ weight decay отдельно!
⚠️ В Adam+L2 weight decay масштабируется адаптивными ставками v̂, из-за чего параметры с большим v̂ (частые, шумные) регуляризируются слабее. AdamW применяет decay напрямую к параметрам — равномерно.

📉 Learning Rate Scheduling

Постоянный lr редко оптимален. Стратегии:

Step decay: η_t = η₀ · γ^(t//step) Cosine: η_t = η_min + ½(η₀−η_min)·(1+cos(πt/T)) Warmup: η_t = η₀ · t/t_warmup (первые t_warmup шагов) Linear decay: η_t = η₀ · (1 − t/T)
💡 Warmup важен для Adam: в начале m и v ещё не прогреты, эффективный lr нестабилен. Маленький начальный lr даёт EMA "набрать" статистику. BERT, GPT все используют warmup + cosine decay.

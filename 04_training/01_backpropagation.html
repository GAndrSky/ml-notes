4.1 Backpropagation

Chain rule на вычислительном графе — полный вывод с нуля

Почему важно

Backpropagation — алгоритм, без которого не существовало бы ни ResNet, ни GPT. Это применение правила цепочки к вычислительному графу с умным кэшированием промежуточных значений.

Что узнаешь
  • Как правило цепочки применяется слой за слоем в обратном направлении
  • Что вычисляется в forward pass и что — в backward pass
  • Почему gradients «взрываются» или «исчезают» в глубоких сетях
  • Как autograd в PyTorch реализует backprop автоматически
Нужно знать
  • Forward pass и Loss (3.3–3.4)
  • Производная и правило цепочки (1.2)
  • Линейная алгебра (1.1)

💡 Идея в одной фразе

Backprop — это алгоритм эффективного вычисления всех частных производных ∂L/∂θᵢ за один проход назад по вычислительному графу, используя цепное правило.

Брутфорс: вычислить ∂L/∂θᵢ для каждого из P параметров отдельно → O(P) forward pass'ов Backprop: один forward + один backward → O(1) по числу проходов
Что означают символы
∂L/∂θᵢградиент по i-му параметруНасколько loss меняется при малом изменении параметра θᵢ. Именно это нужно вычислить для каждого веса, чтобы сделать шаг градиентного спуска.
Pчисло параметровОбщее количество весов в сети. У GPT-3 P=175 млрд. Брутфорс требует P отдельных forward pass-ов — практически невозможно.
O(P)сложность брутфорсаЛинейная сложность по числу параметров: каждый из P градиентов вычисляется независимо через отдельный forward pass с numerical perturbation.
O(1)сложность backprop по проходамBackprop вычисляет ВСЕ P градиентов за один forward + один backward. Число проходов не зависит от P — это и есть ключевое преимущество.
Интуиция: Брутфорс считает «что будет, если чуть-чуть изменить θ₁? а θ₂? а θ₃?» — по одному вопросу за раз. Backprop отвечает на все P вопросов одновременно, используя то, что граф вычислений уже построен в forward pass.
Аналогия: Брутфорс — опрашивать 175 млрд работников поодиночке. Backprop — разослать одно сообщение по всей цепочке командования: каждый звено пересылает его вниз, добавив свою долю ответственности.
Числовой пример: Сеть с P=1000 параметрами. Брутфорс: 1000 forward pass-ов × 0.1 мс = 100 мс на один update. Backprop: 1 forward (0.1 мс) + 1 backward (≈0.2 мс) = 0.3 мс. Выигрыш: 333×.
💡 При P=175 млрд параметров (GPT-3) брутфорс невозможен. Backprop вычисляет все градиенты за одну пару forward+backward — то же время, что один forward pass.
Теория глубже

Backprop решает задачу распределения ответственности

Forward pass строит prediction, а backprop отвечает на другой вопрос: какие промежуточные решения внутри сети больше всего виноваты в loss. Это credit assignment problem. Без него мы знали бы только итоговую ошибку, но не знали бы, какие веса менять.

Backprop эффективен потому, что переиспользует локальные производные и проходит граф один раз назад. Он не перебирает каждый параметр отдельно, а передаёт один градиентный сигнал через всю композицию функций.

  • Локальная производная говорит, как узел реагирует на свой вход.
  • Upstream gradient говорит, насколько итоговый loss чувствителен к выходу узла.
  • Их произведение даёт вклад узла в ошибку всей сети.
Пошагово Backprop на числах — сеть x→h→ŷ за 8 шагов
Шаг 1 из
Шаг 1 — Сеть и данные

Фиксируем параметры

Простейшая сеть: один вход, один скрытый нейрон, один выход. Активация — ReLU. Loss — MSE.

x = 2.0 (вход) w₁ = 0.5 (вес input→hidden) w₂ = 0.3 (вес hidden→output) b₁ = b₂ = 0 y = 1.0 (правильный ответ)
Задача: вычислить ∂L/∂w₁ и ∂L/∂w₂, чтобы сдвинуть w в сторону правильного ответа.
Шаг 2 — Forward: скрытый слой

Считаем h = ReLU(w₁ · x)

Сначала линейное преобразование, потом нелинейность.

z₁ = w₁ · x = 0.5 × 2.0 = 1.0 h = ReLU(z₁) = ReLU(1.0) = 1.0 ← 1.0 > 0, поэтому не режется
h = 1.0 — выход скрытого нейрона перед передачей на output.
Шаг 3 — Forward: выход

Считаем ŷ = w₂ · h

Выходной нейрон — линейный (без активации, так как задача регрессии).

ŷ = w₂ · h = 0.3 × 1.0 = 0.3
ŷ = 0.3 — предсказание модели. Правильный ответ y = 1.0 — далеко.
Шаг 4 — Loss

L = (ŷ − y)²

MSE loss для одного примера — квадрат ошибки предсказания.

L = (ŷ − y)² = (0.3 − 1.0)² = (−0.7)² = 0.49
L = 0.49 — это число мы хотим уменьшить. Начинается обратный проход.
Шаг 5 — Backward: ∂L/∂ŷ

Градиент loss по выходу

Производная MSE по предсказанию — первый шаг backward pass.

∂L/∂ŷ = 2(ŷ − y) = 2 × (0.3 − 1.0) = 2 × (−0.7) = −1.4
∂L/∂ŷ = −1.4 — отрицательный знак означает: нужно увеличить ŷ.
Шаг 6 — Backward: ∂L/∂w₂

Градиент по весу output-слоя

Цепное правило: ∂L/∂w₂ = ∂L/∂ŷ · ∂ŷ/∂w₂. Поскольку ŷ = w₂·h, то ∂ŷ/∂w₂ = h.

∂L/∂w₂ = ∂L/∂ŷ · h = (−1.4) × 1.0 = −1.4
∂L/∂w₂ = −1.4 — w₂ нужно увеличить (отрицательный градиент → спуск).
Шаг 7 — Backward: ∂L/∂w₁ (через ReLU)

Градиент через ReLU к w₁

ReLU'(z₁) = 1, поскольку z₁ = 1.0 > 0. Если бы z₁ ≤ 0 — градиент был бы 0 (мёртвый нейрон).

∂L/∂h = ∂L/∂ŷ · w₂ = (−1.4) × 0.3 = −0.42 ∂L/∂z₁ = ∂L/∂h · ReLU'(z₁) = (−0.42) × 1 = −0.42 ∂L/∂w₁ = ∂L/∂z₁ · x = (−0.42) × 2.0 = −0.84
∂L/∂w₁ = −0.84 — цепное правило прошло через три узла.
Шаг 8 — Обновление весов (η = 0.1)

w ← w − η · ∂L/∂w

Один шаг градиентного спуска. Новые веса дают ŷ ближе к y = 1.0.

w₁_new = 0.5 − 0.1 × (−0.84) = 0.5 + 0.084 = 0.584 w₂_new = 0.3 − 0.1 × (−1.4) = 0.3 + 0.14 = 0.440
Новый ŷ = w₂_new · ReLU(w₁_new · x) = 0.44 × ReLU(0.584 × 2) = 0.44 × 1.168 ≈ 0.514. Loss упал с 0.49 → 0.235. ✓

🔗 Chain rule — основа backprop

Скалярный случай

z = g(y), y = f(x) dz/dx = dz/dy · dy/dx
Что означают символы
x, y, zскалярные переменные цепочкиТри числа, связанные двумя функциями: x → f → y → g → z. Каждая стрелка — один «узел» вычислительного графа.
f, gлокальные функцииПроизвольные дифференцируемые функции. В нейросети это может быть: умножение на вес, сложение с bias, ReLU, sigmoid — любая операция узла.
dz/dxитоговая производнаяНасколько z (например, loss) меняется при изменении x (например, параметра). Именно это нам нужно для градиентного спуска.
dz/dyupstream gradientПроизводная z по промежуточной переменной y — «сигнал» от следующего узла. В backprop это то, что пришло «сверху» (из правой части графа).
dy/dxлокальная производнаяПроизводная функции f в данном узле. Вычисляется локально — каждый узел знает только свою производную, не всю цепочку.
·умножение производныхChain rule: итоговая производная = произведение всех локальных производных по цепочке. Backprop проходит граф назад, накапливая это произведение.
Интуиция: Каждый узел графа «знает» только свою локальную производную dy/dx. Backprop передаёт upstream gradient dz/dy от выхода ко входу, умножая на локальную производную в каждом узле. Итог — производная по любому параметру без повторных вычислений.
Аналогия: Эстафета: последний бегун (loss) передаёт «градиентную палочку» предыдущему, умноженную на скорость своего участка. Каждый умножает полученное на свою скорость и передаёт дальше. Первый бегун (параметр) получает произведение всех скоростей.
Числовой пример: y = f(x) = x², z = g(y) = 3y. dy/dx = 2x, dz/dy = 3. При x=4: dz/dx = 3 · 2·4 = 24. Проверка: z = 3x² → dz/dx = 6x = 24 ✓

Векторный случай (VJP)

z = g(y), y = f(x), x,y,z ∈ ℝⁿ ∂z/∂x = (∂y/∂x)ᵀ · ∂z/∂y = Jᵀ · v v = ∂z/∂y — вектор из выше, J = ∂y/∂x — якобиан слоя
Что означают символы
x, y, z ∈ ℝⁿвекторы вместо скаляровТеперь каждая переменная — вектор из n чисел. Это реальный случай для нейросетей: входы и выходы слоя — батчи или векторы признаков.
J = ∂y/∂xякобиан слояМатрица частных производных: Jᵢⱼ = ∂yᵢ/∂xⱼ. Если y имеет m компонент, а x — n, то J — матрица m×n. Описывает, как каждый выход зависит от каждого входа.
v = ∂z/∂yupstream gradient (вектор)Градиент z по y — вектор той же размерности, что y. Это «сигнал», пришедший из следующего узла при backprop.
Jᵀ · vVector-Jacobian Product (VJP)Транспонированный якобиан умножается на upstream вектор v. Результат — вектор той же размерности, что x. Это и есть градиент z по x.
(∂y/∂x)ᵀтранспонирование якобианаВ forward pass: y = J·x (матрица × вектор). В backward pass: градиент = Jᵀ·v (транспонированная матрица × вектор). Транспонирование «разворачивает» операцию.
Интуиция: Полный якобиан J может быть огромным (миллионы × миллионы). Backprop никогда не строит его явно — только вычисляет произведение Jᵀv, которое имеет размер входа. Это на порядки эффективнее хранения полного J.
Аналогия: Зная таблицу умножения (якобиан), не нужно хранить её всю в памяти. Достаточно уметь умножить нужную строку на входной вектор — это и есть VJP.
Числовой пример: y = Wx, W — матрица 3×2. J = W (3×2). v = [0.5, 0.3, 0.1] (3D upstream). VJP = Wᵀ·v — матрица (2×3) умноженная на вектор (3) = вектор (2). Никакой явной якобиановой матрицы 3×2 хранить не нужно.

На практике нам не нужен полный якобиан J — только произведение Jᵀv. Это VJP (Vector-Jacobian Product) — именно это вычисляет autograd.

Граф и дельты

Обозначение: δˡ = «локальный градиент» = ∂L/∂Zˡ

Forward: Zˡ = Aˡ⁻¹·(Wˡ)ᵀ + bˡ Aˡ = f(Zˡ) Backward: δˡ = (∂L/∂Aˡ) ⊙ f′(Zˡ) ← поэлементно ∂L/∂Wˡ = (δˡ)ᵀ · Aˡ⁻¹ ∂L/∂bˡ = sum(δˡ, axis=0) ∂L/∂Aˡ⁻¹ = δˡ · Wˡ ← передаём ниже
Что означают символы
lномер слояИндекс текущего слоя в сети. l=1 — первый слой после входа, l=L — последний (выходной). Backprop идёт от L к 1.
Zˡpre-activation слоя lРезультат линейного преобразования до функции активации: Zˡ = Aˡ⁻¹·(Wˡ)ᵀ + bˡ. Матрица формы (B, n_out).
Aˡактивации слоя lВыход слоя l после активации: Aˡ = f(Zˡ). Подаётся как вход в следующий слой l+1.
Aˡ⁻¹активации предыдущего слояВход в слой l — это выход слоя l−1. В формуле ∂L/∂Wˡ = (δˡ)ᵀ·Aˡ⁻¹ нужны именно эти значения из forward pass.
Wˡ, bˡвеса и bias слоя lОбучаемые параметры. Wˡ — матрица (n_out, n_in), bˡ — вектор (n_out,). Именно для них вычисляем градиенты ∂L/∂Wˡ и ∂L/∂bˡ.
δˡлокальный градиент (дельта)∂L/∂Zˡ — насколько loss чувствителен к pre-activation. Ключевой промежуточный результат: из δˡ выводятся все три градиента (по W, b и A_prev).
∂L/∂Aˡupstream gradientГрадиент, пришедший из слоя l+1. Для последнего слоя это ∂L/∂ŷ (например 2(ŷ−y)/B для MSE). Для промежуточных — результат предыдущего шага backward.
f′(Zˡ)производная активацииПоэлементная производная функции активации, вычисленная в точках Zˡ из forward pass. Умножение на неё — это цепное правило через узел активации.
⊙поэлементное умножение (Hadamard)Умножение матриц одинакового размера поэлементно. Отличается от матричного умножения ·: здесь каждый элемент умножается на соответствующий, размер не меняется.
sum(δˡ, axis=0)суммирование по батчуBias b один для всех B примеров в батче, поэтому его gradient — сумма δˡ по всем примерам (строкам). Результат: вектор (n_out,).
Интуиция: Три уравнения backward — полный backprop одного слоя. δˡ вычисляется один раз и используется трижды: для ∂L/∂W (обновить веса), ∂L/∂b (обновить bias) и ∂L/∂Aˡ⁻¹ (передать сигнал в следующий слой назад).
Аналогия: δˡ — как «счёт претензий» отдела к предыдущему звену. Из него выводится: сколько каждый сотрудник (вес W) виноват, сколько виновата базовая ставка (bias b), и сколько виноват отдел целиком (передаём дальше).
Числовой пример: δˡ = [[0.3, −0.1]], Aˡ⁻¹ = [[1.0, 2.0]]. ∂L/∂Wˡ = (δˡ)ᵀ·Aˡ⁻¹ = [[0.3],[−0.1]]·[[1.0, 2.0]] = [[0.3, 0.6],[−0.1, −0.2]]. ∂L/∂bˡ = sum([[0.3, −0.1]], axis=0) = [0.3, −0.1].
💡 Три уравнения backward — вот и весь backprop для одного слоя. Применяешь их от последнего слоя к первому.
Теория глубже

VJP важнее полного Якобиана в реальных сетях

Полный Jacobian может быть огромным: выходов и входов у слоя тысячи или миллионы. Backprop почти никогда не строит его явно. Вместо этого он умножает incoming gradient на Jacobian локальной операции: это vector-Jacobian product.

Такой подход экономит память и вычисления. Autograd хранит граф операций forward pass, а затем вызывает для каждой операции локальное правило backward, которое возвращает нужные VJP.

  • Jacobian описывает все локальные чувствительности.
  • VJP берёт только то направление, которое нужно текущему loss.
  • Reverse-mode autodiff подходит для моделей с миллионами параметров и скалярным loss.

⚡ Градиенты через функции активации

ReLU: forward: A = max(0, Z) backward: δ_prev = δ ⊙ 𝟙[Z > 0] → просто «маска»: пропускает grad там, где Z>0 Sigmoid: forward: A = σ(Z) backward: δ_prev = δ ⊙ A⊙(1−A) → умножается на σ′ = σ(1−σ) ≤ 0.25 Tanh: forward: A = tanh(Z) backward: δ_prev = δ ⊙ (1−A²) → умножается на tanh′ = 1−tanh² ≤ 1
Что означают символы
Zpre-activation (матрица)Входные значения функции активации для всего батча. Форма: (B, n_out). Сохраняется из forward pass — нужен при backward для вычисления производной.
Aактивации (выход f)Результат применения активации к Z: A = f(Z). Также сохраняется из forward pass — используется в backward для sigmoid (A⊙(1−A)) и tanh (1−A²).
δupstream gradientГрадиент ∂L/∂A — сигнал, пришедший из следующего слоя. Умножается на локальную производную активации для получения δ_prev.
δ_prevgradient для предыдущего слоя∂L/∂Z — то, что передаётся дальше назад по сети. Именно из него следующий слой вычислит свои ∂L/∂W и ∂L/∂b.
⊙поэлементное умножениеГрадиент умножается на производную активации поэлементно — каждый нейрон независимо масштабирует свой сигнал. Не матричное умножение.
𝟙[Z > 0]маска ReLUМатрица из 0 и 1: 1 там, где Z>0 (нейрон был активен в forward), 0 там, где Z≤0. Gradient проходит только через активные нейроны.
A⊙(1−A)производная sigmoidσ′(Z) = σ(Z)·(1−σ(Z)) = A·(1−A). Переиспользует A из forward pass. Максимум = 0.25 при Z=0 — именно это ограничение вызывает vanishing gradient.
(1−A²)производная tanhtanh′(Z) = 1−tanh²(Z) = 1−A². Переиспользует A из forward pass. Максимум = 1 при Z=0 — вдвое лучше sigmoid, но тоже насыщается.
Интуиция: Каждая активация превращается в «ворота» для gradient flow. ReLU — бинарные ворота (открыты/закрыты). Sigmoid и tanh — аналоговые ворота (открыты частично), причём тем меньше, чем дальше от нуля. Поэтому ReLU так популярна для глубоких сетей.
Числовой пример: Z=[[2, −1, 0.5]], δ=[[0.8, 0.6, 0.4]]. ReLU: маска=[[1,0,1]], δ_prev=[[0.8, 0.0, 0.4]]. Sigmoid (A≈[[0.88, 0.27, 0.62]]): A⊙(1−A)≈[[0.106, 0.197, 0.236]], δ_prev≈[[0.085, 0.118, 0.094]] — значительно меньше.

Теория глубже

Активация фильтрует не только forward-сигнал, но и gradient flow

Если ReLU в forward pass обнулила отрицательный z, то в backward pass она также блокирует градиент через этот путь. Поэтому mask активации фактически решает, какие нейроны участвовали в обучении на данном batch.

Для гладких активаций вроде GELU и SiLU блокировка мягче. Они могут передавать небольшой сигнал даже в отрицательной области, что иногда делает обучение стабильнее на глубоких сетях.

  • ReLU создаёт жёсткую бинарную маску.
  • Sigmoid и tanh могут гасить градиент через saturation.
  • GELU и SiLU дают более плавный gradient gate.

🗂 Матричные формулы backprop — шпаргалка

Слой l: Z = A_prev · Wᵀ + b, A = f(Z) FORWARD: Z^l = A^(l-1) · (W^l)ᵀ + b^l shape: (B, n_out) A^l = f(Z^l) shape: (B, n_out) BACKWARD (дано δ^l = ∂L/∂A^l, shape (B, n_out)): ∂L/∂Z^l = δ^l ⊙ f′(Z^l) shape: (B, n_out) ∂L/∂W^l = (∂L/∂Z^l)ᵀ · A^(l-1) shape: (n_out, n_in) ∂L/∂b^l = Σ_B ∂L/∂Z^l shape: (n_out,) δ^(l-1) = ∂L/∂A^(l-1) = ∂L/∂Z^l · W^l shape: (B, n_in) Стартовый градиент (MSE): ∂L/∂A^L = 2(A^L − y)/B Стартовый градиент (CE): ∂L/∂Z^L = (p̂ − y)/B ← softmax+CE вместе
Что означают символы
Bразмер батчаЧисло примеров в одном батче. Все матрицы имеют B строк — по одной на каждый пример. Градиент усредняется или суммируется по B.
n_in, n_outразмерности слояn_in — число входных нейронов, n_out — выходных. Матрица весов W имеет форму (n_out, n_in), что определяет форму всех градиентов.
shape: (B, n_out)форма тензораРазмерность матрицы: B строк (примеры в батче) × n_out столбцов (нейроны слоя). Важно отслеживать формы — несовпадение формы = ошибка в реализации.
∂L/∂Z^lградиент по pre-activationКлючевой промежуточный gradient. Вычисляется как δ^l ⊙ f′(Z^l) и используется для всех трёх производных: по W, b и A_prev.
(∂L/∂Z^l)ᵀ · A^(l-1)градиент по весам WВнешнее произведение (транспонированных) дельт и входных активаций. Форма: (n_out, B) · (B, n_in) = (n_out, n_in) — совпадает с формой W ✓
Σ_B ∂L/∂Z^lградиент по bias bСуммируем дельты по всем B примерам батча. Bias один на все примеры → его gradient = сумма вкладов всех примеров. Форма: (n_out,).
∂L/∂Z^l · W^lgradient для предыдущего слояПередаём backward-сигнал в слой l−1. Форма: (B, n_out) · (n_out, n_in) = (B, n_in). Обратите внимание: W без транспонирования (в отличие от ∂L/∂W).
2(A^L − y)/Bстартовый gradient MSEПроизводная MSE loss по выходу последнего слоя A^L. Это точка входа в backprop — отсюда начинается распространение градиента назад.
(p̂ − y)/Bстартовый gradient Softmax+CEАналитически упрощённый gradient softmax + cross-entropy вместе. p̂ — предсказанные вероятности, y — one-hot вектор истинного класса. Деление на B — усреднение по батчу.
Интуиция: Эта таблица — полный backprop одного слоя в матричной форме. Всего 4 формулы. Применяешь их от последнего слоя к первому, и получаешь градиенты для всех параметров сети. Формы тензоров — лучший способ проверить правильность реализации.
Числовой пример: Слой: n_in=3, n_out=2, B=4. ∂L/∂Z^l: форма (4,2). ∂L/∂W^l = (∂L/∂Z^l)ᵀ·A^(l-1): (2,4)·(4,3) = (2,3) ✓ совпадает с W. ∂L/∂b: sum по axis=0 → (2,). δ^(l-1) = ∂L/∂Z^l·W^l: (4,2)·(2,3) = (4,3) ✓ совпадает с входом.
💡 Ключевое наблюдение: ∂L/∂W = δᵀ·X имеет shape (n_out, n_in) — то же, что у W. Каждый параметр получает свой градиент автоматически из одного матричного умножения.

Теория глубже

Backprop нужно отлаживать по нормам градиентов

Если loss не учится, смотреть только на значение loss недостаточно. Нужно проверить, доходят ли градиенты до ранних слоёв, не стали ли они нулевыми, не взорвались ли они и не блокирует ли их случайный detach.

Практический debugging начинается с gradient norms по слоям, проверки requires_grad, отсутствия in-place ошибок и сравнения train/eval режимов. Это быстрее, чем менять архитектуру вслепую.

  • Нулевые gradient norms часто означают detach, saturation или dead ReLU.
  • Очень большие norms указывают на exploding gradients или слишком высокий lr.
  • Разные нормы по слоям помогают увидеть, где именно ломается gradient flow.
Теоретический слой

Backpropagation как эффективное применение chain rule

Backpropagation — это способ быстро посчитать, как каждый параметр сети влияет на итоговый loss. Он не является отдельной магией нейросетей: это reverse-mode automatic differentiation, применённый к computational graph.

Главная идея в локальности. Каждый узел графа знает только свою локальную производную и входящий сверху градиент. Перемножая эти локальные чувствительности в обратном порядке, сеть получает градиенты для миллионов параметров за один backward pass.

Практически backprop важен не только для вывода формул. Он объясняет vanishing gradients, exploding gradients, detach, in-place operation bugs, необходимость сохранения intermediate activations и стоимость памяти при обучении.

Геометрическая интуиция

  • Loss можно представить как поверхность над пространством параметров, а градиент — как направление самого быстрого роста этой поверхности.
  • Backprop переносит чувствительность от выхода к ранним слоям через цепочку локальных преобразований.
  • Jacobian показывает, как маленькое движение входа слоя меняет выход слоя; VJP умножает этот Jacobian на градиент сверху.

Вероятностная интерпретация

  • Если loss является negative log-likelihood, градиент показывает, как изменить параметры, чтобы повысить вероятность наблюдаемых ответов.
  • Mini-batch gradient — шумная оценка полного градиента по распределению данных.
  • Слишком маленький или слишком большой gradient signal искажает обучение: модель либо почти не обновляет ранние слои, либо делает нестабильные шаги.

Оптимизационный смысл

  • Reverse-mode AD эффективен, когда параметров много, а loss один: именно это типично для нейросетей.
  • Backward pass переиспользует промежуточные значения forward pass, поэтому memory footprint растёт с глубиной и размером batch.
  • Градиент сам по себе не обучает модель; optimizer решает, как превратить градиент в шаг параметров.

Практика и типичные ошибки

  • Проверяй shapes градиентов и параметров, особенно вокруг transpose, broadcasting и batch dimension.
  • Не используй detach, no_grad или in-place операции там, где нужен gradient flow.
  • Если обучение не идёт, смотри norm градиентов по слоям: нули, NaN и резкие пики сразу показывают проблему.

Мини-пример: Для z = w*x + b и loss L градиент по w равен dL/dz * x. Если x большой, тот же upstream gradient даст больший шаг по w, поэтому масштаб входов напрямую влияет на обучение.

Type A — Derivation Foundational

Выведи dL/dW и dL/db для одного линейного слоя z = xW + b и MSE loss. Обязательно укажи shapes для x, W, z и y.

Показать решение

Для batch X формы (n,d), W формы (d,k), Z=XW+b формы (n,k). Если L=mean((Z-Y)^2), то dL/dZ имеет форму (n,k), dL/dW = X^T dL/dZ формы (d,k), dL/db = sum по batch формы (k,).

Type A — Derivation Hard

Покажи, почему для softmax + cross-entropy градиент по logits упрощается до p - y_onehot.

Показать решение

Производная log-softmax даёт матрицу Jacobian, где diagonal часть p_i(1-p_i), off-diagonal -p_i p_j. После свёртки с one-hot target все члены упрощаются до p_j - y_j. Это причина, почему CrossEntropyLoss удобно принимать logits напрямую.

Type B — Coding Intermediate

Реализуй finite-difference gradient check для маленького линейного слоя и сравни с ручным backward.

Показать решение

Finite differences должен совпасть с analytic gradient с малой погрешностью. Если не совпадает, обычно ошибка в transpose, усреднении batch или знаке.

# What it does: finite-difference check for a scalar loss
# Input shape: w=(d,)
# Output shape: grad=(d,)
import numpy as np

def loss(w):
    x = np.array([1.0, -2.0, 3.0])
    y = 2.0
    pred = w @ x
    return (pred - y) ** 2

def finite_diff(w, eps=1e-5):
    g = np.zeros_like(w)
    for i in range(w.size):
        step = np.zeros_like(w)
        step[i] = eps
        g[i] = (loss(w + step) - loss(w - step)) / (2 * eps)
    return g

w = np.array([0.2, -0.1, 0.3])
print(finite_diff(w))
# Gotchas: compare with analytic/autograd gradients at the same parameter values.

Type C — Conceptual Foundational

Почему backprop требует один forward и один backward pass, а не отдельный проход для каждого параметра?

Показать решение

Reverse-mode AD переиспользует локальные производные и upstream gradients в computational graph. Для одного scalar loss это эффективнее, чем perturb каждый параметр отдельно.

Type C — Conceptual Intermediate

Что происходит с градиентами в глубокой сети с sigmoid activations?

Показать решение

Производная sigmoid максимум 0.25. При многократном перемножении такие множители быстро уменьшают сигнал, поэтому ранние слои получают почти нулевой gradient.

Проверь себя

Вопросы для самопроверки

  1. Что вычисляет обратный проход (backpropagation) и почему он эффективен?

    Backpropagation вычисляет градиент функции потерь по всем параметрам за O(p) операций, применяя цепное правило от выхода к входу. Без него нужно было бы численно оценивать каждый из p параметров отдельно — O(p²). Это делает обучение многомиллионных сетей практически возможным.

  2. Что такое «ванишинг/взрывной» градиент и как он влияет на обучение?

    При backprop градиент умножается на производные активаций и веса на каждом слое. Если |произведение| < 1 на каждом шаге — градиент экспоненциально убывает (vanishing), ранние слои не учатся. Если > 1 — градиент взрывается (exploding). Решения: skip-connections, нормализация, клиппинг градиентов.

  3. Почему gradient checkpointing (реkomputation) используется при обучении больших моделей?

    Обычно все активации кешируются для backward pass — это O(depth × batch) памяти. Gradient checkpointing сохраняет только каждый k-й слой, пересчитывая промежуточные активации при необходимости. Это снижает память в √depth раз за счёт ~33% замедления.

  4. Как automatic differentiation отличается от символьного и численного дифференцирования?

    Символьное дифференцирование строит аналитическую формулу (может быть очень длинной). Численное считает (f(x+ε)-f(x))/ε — неточно, O(p) вызовов. Autodiff (как в PyTorch/JAX) применяет цепное правило к примитивным операциям в графе вычислений — точно, O(p) за один проход.

Как закрепить тему: Backpropagation

Этот блок связывает теорию с практическим решением: где применять идею, какие ошибки проверять и какие ручки менять в эксперименте.

Когда применять

  • Используй тему «Backpropagation», когда модель уже построена, но обучение нестабильно, медленно или не улучшается.
  • Обучение — это управление динамикой: важны не только формулы optimizer-а, но и масштаб градиента, scheduler и численная точность.
  • Смотри на train/validation curves вместе: одна линия редко объясняет проблему полностью.

Типичные ошибки

  • Менять сразу много ручек: learning rate, batch size, optimizer и regularization нужно проверять постепенно.
  • Игнорировать первые сотни шагов: именно там видны плохая инициализация, слишком большой lr и exploding gradients.
  • Лечить NaN уменьшением модели, не проверив scale loss, precision, clipping и входные данные.

Что тюнить

  • Главные ручки: learning rate, scheduler, optimizer, weight decay, clipping и batch size.
  • Для mixed precision проверяй loss scaling, BF16/FP16 режим и операции, чувствительные к overflow.
  • Для регуляризации сравни train-validation gap, а не только финальную accuracy.

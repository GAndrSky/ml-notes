4.4 Регуляризация

Dropout, BatchNorm, LayerNorm, Weight Decay — почему переобучение, как бороться

🎯 Почему возникает переобучение?

Нейронная сеть с P >> N параметров способна запомнить весь датасет наизусть. Задача регуляризации — не дать ей это сделать.

Переобучение: train loss ↓↓ test loss ↑ Underfitting: train loss ↑ test loss ↑ Золото: train loss ↓ test loss ↓ (обобщение)
💡 Bias-variance tradeoff:
Переобучение = высокая variance (чуткость к шуму обучающей выборки).
Регуляризация добавляет bias, уменьшает variance.

⚖️ Weight Decay — L2 регуляризация

L_reg = L_data + (λ/2)·Σᵢ θᵢ² ∂L_reg/∂θ = ∂L_data/∂θ + λ·θ Обновление SGD: θ ← θ − η·(g + λ·θ) = (1 − ηλ)·θ − η·g ← «сжатие» весов

При каждом шаге веса умножаются на (1−ηλ) < 1 — «тянутся к нулю». Это эквивалентно MAP-оценке с гауссовым приором θ ~ N(0, 1/λ).

💡 Большие веса → сложная функция → переобучение.
Маленькие веса → гладкая функция → лучше обобщение.
Связь с блоком 1.3: MAP с гауссовым приором = L2.

Эффект λ на веса

🎲 Dropout

Train: mask ~ Bernoulli(1−p) ã = a ⊙ mask / (1−p) ← inverted dropout (деление на (1−p) сохраняет E[ã] = E[a]) Test: ã = a ← без маски, без масштаба

При каждом forward pass случайно обнуляем долю (1−p) нейронов. Сеть не может полагаться на любой конкретный нейрон.

💡 Байесовская интерпретация:
Dropout ≈ обучение ансамбля из 2^H сетей (H нейронов скрытого слоя) с общими весами. Inference = геометрическое среднее ансамбля.

MC Dropout: оставь dropout на inference, запусти N раз → приблизит posterior uncertainty.

Интерактивный Dropout

p (dropout rate): 0.50

Почему inverted dropout, а не масштабирование на test?

Вариант 1 (наивный): train без масштабирования, test × (1−p) Проблема: нужно менять все forward-вызовы на inference Вариант 2 (inverted dropout, стандарт): train: делим на (1−p) → E[output] сохраняется test: ничего не меняем → чистый forward pass

📊 Batch Normalization (Ioffe & Szegedy, 2015)

Train (батч X ∈ ℝ^(B×H)): μ_B = (1/B)·Σᵢ xᵢ ← среднее по батчу σ²_B = (1/B)·Σᵢ (xᵢ−μ_B)² ← дисперсия по батчу x̂ᵢ = (xᵢ − μ_B) / √(σ²_B+ε) ← нормировка yᵢ = γ·x̂ᵢ + β ← масштаб+сдвиг (learnable)

γ, β — обучаемые параметры. Нужны, чтобы BN не ограничивал выразительность: сеть может «отменить» нормировку, выставив γ=σ, β=μ.

⚠️ Test (inference): используются running mean/var, накопленные за обучение — не статистики батча!
running_mean ← (1−m)·running_mean + m·μ_B

Почему BN помогает обучению?

💡 Internal Covariate Shift:
Без BN: распределение активаций постоянно меняется при обновлении весов предыдущего слоя. Каждый слой вынужден адаптироваться к «движущейся цели».
С BN: активации всегда нормированы → следующий слой получает стабильное распределение.
💡 Более гладкая loss surface:
Lipschitz-константа loss и градиентов уменьшается. Можно использовать больший lr. Менее чувствительна к инициализации.
BN → lr в 10× больше → быстрее сходимость.

Backward pass BatchNorm

∂L/∂γ = Σᵢ ∂L/∂yᵢ · x̂ᵢ ∂L/∂β = Σᵢ ∂L/∂yᵢ ∂L/∂x̂ᵢ = ∂L/∂yᵢ · γ ∂L/∂σ² = Σᵢ ∂L/∂x̂ᵢ · (xᵢ−μ) · (−½)(σ²+ε)^(−3/2) ∂L/∂μ = Σᵢ ∂L/∂x̂ᵢ · (−1/√(σ²+ε)) + ∂L/∂σ² · (−2/B)·Σ(xᵢ−μ) ∂L/∂xᵢ = ∂L/∂x̂ᵢ/√(σ²+ε) + ∂L/∂σ²·2(xᵢ−μ)/B + ∂L/∂μ/B

🔲 Layer Normalization (Ba et al., 2016)

x ∈ ℝ^H (один пример, H признаков): μ = (1/H)·Σⱼ xⱼ ← среднее по признакам σ² = (1/H)·Σⱼ (xⱼ−μ)² x̂ⱼ = (xⱼ − μ) / √(σ²+ε) yⱼ = γⱼ·x̂ⱼ + βⱼ ← γ,β shape (H,)

BatchNorm vs LayerNorm

BatchNorm: нормирует по батчу (ось B) → зависит от B, проблемы при B=1 LayerNorm: нормирует по признакам (ось H) → независим от размера батча → одинаково на train и inference

💡 Почему Transformer использует LayerNorm:
• Sequence длина переменная → батч из последовательностей разной длины
• Авторегрессивное обучение: нельзя смотреть «вперёд» по батчу
• При inference генерируем по одному токену (B=1) → BatchNorm сломается
📌 Pre-LN vs Post-LN:
Original Transformer (2017): Post-LN (после Attention)
GPT-2+, современные LLM: Pre-LN (перед Attention)
Pre-LN стабильнее, не нужна lr warmup такой длинный.

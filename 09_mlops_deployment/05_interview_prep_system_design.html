9.5 Interview Prep — ML System Design

На system design интервью оценивают не знание модной модели, а умение структурировать неопределённую задачу: уточнить цель, оценить масштаб, выбрать данные, построить pipeline, serving и monitoring.

Framework ответа

clarify -> metrics -> scale -> data -> model -> serving -> monitoring -> risks

Хороший ответ всегда начинается с уточнений. Нельзя проектировать fraud detection, recommender или RAG одинаково: стоимость ошибки, latency, labels и feedback loop разные.

1. Recommendation system

Clarify: что рекомендуем, кому, где показываем, оптимизируем CTR, watch time или retention. Data: impressions, clicks, dwell time, user/item metadata. Model: candidate generation + ranking. Serving: precompute candidates, online ranker. Monitoring: CTR, diversity, freshness, popularity bias.

2. Fraud detection

Clarify: real-time или batch, цена false positive и false negative. Data: transactions, user history, device, merchant, velocity features. Model: gradient boosting baseline, calibrated threshold. Serving: low-latency online scoring. Monitoring: drift, delayed labels, alert fatigue.

3. Semantic document search

Clarify: enterprise docs, web search или support knowledge base. Data: documents, chunks, metadata, query logs. Model: embedding retrieval + optional cross-encoder reranker. Serving: vector index, hybrid search, caching. Monitoring: zero-result rate, click satisfaction, stale documents.

4. Image moderation

Clarify: policy categories, human review, latency and false negative tolerance. Data: labeled images, hard negatives, policy updates. Model: classifier/detector plus threshold per class. Serving: async pipeline for heavy models, fallback rules. Monitoring: appeal rate, drift by content source, reviewer disagreement.

5. LLM chat assistant with RAG

Clarify: domain, answer style, hallucination tolerance, privacy. Data: documents, chunks, metadata, eval questions. Model: retriever + reranker + generator. Serving: query rewrite, retrieve, generate, cite. Monitoring: groundedness, retrieval hit rate, latency, user feedback.

Как звучать сильно на интервью

  • Всегда называй primary metric и guardrail metrics.
  • Отделяй offline metric от online business impact.
  • Обсуждай feedback loop и delayed labels.
  • Показывай fallback, rollback и human-in-the-loop там, где ошибка дорогая.
  • Не прыгай к Transformer до baseline и data pipeline.

Exercises

Foundational

Clarifying questions

Напиши 10 уточняющих вопросов для “design a fraud detection system”.

Show solution

Latency, volume, labels, fraud type, cost matrix, manual review, regions, feature freshness, compliance, acceptable false positives.

Intermediate

Metric choice

Для image moderation выбери primary и guardrail metrics.

Show solution

Primary может быть recall for harmful content или policy-weighted F1. Guardrails: false positive appeal rate, latency, reviewer load, segment fairness.

Hard

Full answer

За 20 минут спроектируй document search system по framework.

Show solution

Ответ должен пройти clarify, scale, ingestion, chunking, embeddings, vector DB, hybrid retrieval, reranking, serving, eval, monitoring and failure modes.

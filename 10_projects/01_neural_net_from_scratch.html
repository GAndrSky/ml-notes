10.1 Project: Neural Net from Scratch

Build a small MLP without PyTorch. The point is not performance; the point is to prove that you understand forward pass, loss, backpropagation, and optimization deeply enough to reproduce them.

TL;DR

You will implement a two-layer neural network using only NumPy: initialize weights, run forward pass, compute cross-entropy, derive gradients manually, verify them with finite differences, train on toy data, then extend the same structure to MNIST.

Architecture

X -> Linear(W1,b1) -> ReLU -> Linear(W2,b2) -> Softmax -> CrossEntropy
What the symbols mean
Xinput batchMatrix of shape (B, D), where B is batch size and D is input dimension.
W1first layer weightsMatrix of shape (D, H). It maps raw features into hidden features.
b1hidden biasVector of shape (H,), added to every object in the batch.
W2classifier weightsMatrix of shape (H, C). It maps hidden features into class logits.
CclassesNumber of target classes.
Intuition: the first layer learns useful features, ReLU keeps positive evidence, and the second layer turns features into class scores.
Analogy: first summarize raw notes into concepts, then use those concepts to choose an answer.
logits = ReLU(X W1 + b1) W2 + b2
Dimensions check
XW1(B,D)(D,H)Result shape is (B,H). Each object becomes a hidden vector.
A1W2(B,H)(H,C)Result shape is (B,C). Each row contains class logits.
b1,b2broadcast biasBias vectors are added to every row in the batch.
Intuition: every layer is a learned coordinate transformation followed by a nonlinearity.
Numeric example: if B=32, D=784, H=128, C=10, then logits has shape (32,10).

Milestone 1 - forward pass

# What it does: creates toy data and runs a 2-layer MLP forward pass.
# Input shapes: X=(B,D), y=(B,), W1=(D,H), W2=(H,C)
# Output shapes: logits=(B,C), probs=(B,C), loss=scalar
import numpy as np

rng = np.random.default_rng(0)
B, D, H, C = 64, 2, 16, 3

X = rng.normal(size=(B, D))
y = ((X[:, 0] > 0).astype(int) + (X[:, 1] > 0).astype(int)) % C

W1 = rng.normal(scale=np.sqrt(2 / D), size=(D, H))
b1 = np.zeros(H)
W2 = rng.normal(scale=np.sqrt(2 / H), size=(H, C))
b2 = np.zeros(C)

def relu(z):
    return np.maximum(0.0, z)

def softmax_stable(logits):
    shifted = logits - logits.max(axis=1, keepdims=True)
    exp = np.exp(shifted)
    return exp / exp.sum(axis=1, keepdims=True)

Z1 = X @ W1 + b1
A1 = relu(Z1)
logits = A1 @ W2 + b2
probs = softmax_stable(logits)
loss = -np.log(probs[np.arange(B), y] + 1e-12).mean()

print(logits.shape, probs.shape, float(loss))

# Gotchas:
# 1. Do not apply softmax before subtracting max(logits), or exp can overflow.
# 2. Check that y indexes classes and has shape (B,), not one-hot shape (B,C).

Milestone 2 - manual backprop

dlogits = (probs - one_hot(y)) / B
What the symbols mean
probspredicted distributionModel probabilities after softmax, shape (B,C).
one_hot(y)target distributionCorrect class is 1, all other classes are 0.
Bbatch sizeDivides gradients because the loss is averaged over the batch.
Intuition: each logit is pushed by the difference between what the model believed and what the target says.
Numeric example: if the correct class prob is 0.2, its gradient contribution is 0.2 - 1 = -0.8, so training raises that logit.
# What it does: computes manual gradients for the 2-layer MLP.
# Input shapes: X=(B,D), Z1=(B,H), A1=(B,H), probs=(B,C), y=(B,)
# Output shapes: dW1=(D,H), db1=(H,), dW2=(H,C), db2=(C,)
one_hot = np.zeros_like(probs)
one_hot[np.arange(B), y] = 1.0

dlogits = (probs - one_hot) / B
dW2 = A1.T @ dlogits
db2 = dlogits.sum(axis=0)

dA1 = dlogits @ W2.T
dZ1 = dA1 * (Z1 > 0)
dW1 = X.T @ dZ1
db1 = dZ1.sum(axis=0)

for name, grad in {"dW1": dW1, "db1": db1, "dW2": dW2, "db2": db2}.items():
    print(name, grad.shape, np.linalg.norm(grad))

# Gotchas:
# 1. ReLU derivative uses Z1 > 0, not A1 > 0 if you want the exact cached pre-activation.
# 2. Transposes are determined by shape algebra, not by trial and error.

Milestone 3 - train loop

# What it does: trains the network with full-batch gradient descent.
# Input shapes: parameters from previous snippets
# Output shapes: updated parameters and printed loss
lr = 0.1

for step in range(1000):
    Z1 = X @ W1 + b1
    A1 = np.maximum(0.0, Z1)
    logits = A1 @ W2 + b2
    probs = softmax_stable(logits)
    loss = -np.log(probs[np.arange(B), y] + 1e-12).mean()

    one_hot = np.zeros_like(probs)
    one_hot[np.arange(B), y] = 1.0
    dlogits = (probs - one_hot) / B
    dW2 = A1.T @ dlogits
    db2 = dlogits.sum(axis=0)
    dA1 = dlogits @ W2.T
    dZ1 = dA1 * (Z1 > 0)
    dW1 = X.T @ dZ1
    db1 = dZ1.sum(axis=0)

    W1 -= lr * dW1
    b1 -= lr * db1
    W2 -= lr * dW2
    b2 -= lr * db2

    if step % 200 == 0:
        acc = (probs.argmax(axis=1) == y).mean()
        print(step, float(loss), float(acc))

# Gotchas:
# 1. If loss does not decrease, first inspect shapes and gradient norms.
# 2. Try to overfit 16 examples before training on larger data.

Milestone 4 - gradient check

Gradient checking is slow but essential. You compare the analytic gradient from backprop with a finite-difference estimate. Run it on a tiny model only.

grad_numeric(theta_i) = (L(theta_i + eps) - L(theta_i - eps)) / (2 eps)
What the symbols mean
theta_ione parameterA single scalar weight or bias being tested.
epssmall perturbationUsually around 1e-5 for float64 checks.
LlossScalar objective before and after nudging the parameter.
Intuition: if backprop is right, a tiny manual nudge predicts the same slope as the analytic gradient.
Analogy: check a speedometer by moving a tiny distance and measuring change directly.

Exercises

Do not skip exercises. The ability to reproduce this project from scratch is the test of understanding.

Foundational - derivation

Derive dW2 and db2

Starting from logits = A1 W2 + b2 and cross-entropy loss, derive dW2 = A1.T @ dlogits and db2 = sum(dlogits).

Show solution

The derivative of each output logit with respect to a weight W2[h,c] is A1[:,h]. Accumulating across the batch gives A1.T @ dlogits. Bias affects each row equally, so its gradient is the sum over batch.

Intermediate - coding

Add mini-batch SGD

Modify the training loop to sample mini-batches and train for multiple epochs.

Show solution

Shuffle indices each epoch, slice X and y by batch indices, compute gradients on the batch, and update parameters. Keep validation data separate.

Intermediate - debugging

Overfit 16 examples

Reduce training data to 16 examples and make the model reach near-perfect training accuracy.

Show solution

If the network cannot overfit a tiny set, the implementation is wrong or the model is too constrained. Increase hidden size, lower regularization, and inspect gradients.

Hard - gradient check

Check dW1 numerically

Pick five random entries of W1 and compare finite-difference gradients against manual backprop.

Show solution

Use float64, eps around 1e-5, and relative error abs(a-b)/max(1,abs(a),abs(b)). Errors around 1e-7 to 1e-5 are usually acceptable for tiny networks.

Hard - extension

Move to MNIST

Use the same MLP structure for flattened 28x28 images and report validation accuracy.

Show solution

Normalize pixels to [0,1], use D=784 and C=10, train with mini-batches, and compare against a PyTorch baseline only after your NumPy model learns.

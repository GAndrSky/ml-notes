10.4 Project: RAG Application

Build a document question-answering system with retrieval, citations, evaluation, and failure analysis.

TL;DR

A RAG app is not "LLM plus vector database". It is an information retrieval system plus a generation layer. Most quality problems come from chunking, retrieval recall, reranking, context assembly, and evaluation.

Deliverables

  • Evaluation set with retrieval and answer-quality metrics.

Pipeline

documents -> chunks -> embeddings -> vector index -> top-k retrieval -> context -> LLM answer + citations

What to notice: if the answer is wrong, first check whether the correct chunk was retrieved. Generation cannot fix missing evidence.

Retrieval objective

score(q, d_i) = cosine(embed(q), embed(d_i))
What the symbols mean
qqueryUser question or rewritten search query.
d_ichunkOne candidate document fragment.
embedembedding modelMaps text into a vector space where similarity approximates relevance.
cosinesimilarityCompares vector directions, usually after normalization.
Intuition: retrieval finds chunks whose meaning points in the same direction as the question.
Analogy: library search returns pages most semantically aligned with your question.

Runnable toy retrieval

# What it does: toy vector retrieval with cosine similarity
# Input shapes: query=(d,), docs=(N,d)
# Output shape: top indices=(k,)
import numpy as np

def normalize(x):
    return x / (np.linalg.norm(x, axis=-1, keepdims=True) + 1e-9)

docs = np.array([[1,0], [0,1], [0.8,0.2]], dtype=float)
query = np.array([1,0.1], dtype=float)
scores = normalize(docs) @ normalize(query[None, :]).T
top = np.argsort(scores[:, 0])[::-1][:2]
print(top, scores[top, 0])
# Gotchas: semantic embeddings are not magic; evaluate recall on real questions.

Evaluation

LayerMetricQuestion answered
RetrievalRecall@kDid we retrieve evidence?
RankingMRR / nDCGIs evidence near the top?
GenerationGroundednessIs the answer supported by context?
ProductHuman ratingIs the answer useful?

Common mistakes

  • Chunking by fixed length without preserving section boundaries.
  • Using only answer quality while ignoring retrieval recall.
  • Stuffing too much context and reducing answer focus.
  • Not storing metadata, making citations impossible.

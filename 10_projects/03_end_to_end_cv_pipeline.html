Пошагово Building a CV pipeline
1 /
Шаг 01

Data audit

Check class balance, image quality, and label noise before training anything. Fix class imbalance with weighted sampling or oversampling. Identify and remove mislabeled examples. A bad dataset produces a bad model regardless of architecture.

Шаг 02

Baseline

Train a simple ResNet-18 for 5 epochs to get a number to beat. This establishes your floor and validates the data loading pipeline. Do not add augmentation or custom architecture until the baseline runs cleanly.

Шаг 03

Error analysis

Look at worst predictions. Identify patterns: blurry images, edge cases, mislabels. Sort false positives by confidence. Error analysis tells you what to fix next — augmentation, more data for specific classes, or architecture change.

Шаг 04

Iterate

Add augmentation, tune architecture, and adjust LR schedule based on error analysis findings. Change one thing at a time. Track per-class metrics, not just overall accuracy — improvements on hard classes may not show up in aggregate scores.

Шаг 05

Export

Export to ONNX with dynamic batch axis: torch.onnx.export(..., dynamic_axes={'input': {0: 'batch'}}). Verify accuracy matches PyTorch within 0.5%. Test with batch size 1 at serving latency target before declaring done.

Проверь себя

End-to-end CV pipeline

  1. Why should you do error analysis before choosing a better architecture?

    A better architecture fixes model capacity, but most CV failures come from data issues: class imbalance, mislabels, distribution shift, or preprocessing bugs. Error analysis reveals whether you need more data, better augmentation, or a different architecture. Swapping to a larger model on bad data wastes compute without improving production metrics.

  2. What is the most common cause of train-test accuracy gap in CV?

    Distribution shift between training and test data is the most common cause. This includes different lighting conditions, camera angles, resolutions, or image preprocessing. Class imbalance in training but not test data is another frequent cause. Models that learn shortcuts (background texture instead of object shape) also show large train-test gaps.

  3. Why export to ONNX instead of serving the PyTorch model directly?

    ONNX enables deployment without a Python/PyTorch runtime, reducing container size and startup time. ONNX Runtime typically achieves 2-4x latency improvement over PyTorch inference through operator fusion and hardware-specific optimizations. It also enables deployment on edge devices and CPUs that do not support PyTorch natively.

Что ломается без этого

Skipping the data audit and training on imbalanced data produces a model that gets high accuracy by ignoring minority classes — misleading metrics hide the failure. A model predicting the majority class 95% of the time achieves 95% accuracy on a 95/5 split while being completely useless. Always check per-class metrics before reporting results.

Проверь себя

Вопросы для самопроверки

  1. Из каких этапов состоит end-to-end CV pipeline?

    (1) Data collection & annotation (labelme, CVAT). (2) EDA — распределение классов, качество разметки. (3) Preprocessing & augmentation (albumentations). (4) Model selection (pretrained backbone + fine-tuning). (5) Training + validation monitoring. (6) Evaluation (mAP, confusion matrix). (7) Serving (ONNX export, TorchServe).

  2. Почему аугментация данных критична для CV?

    Аугментация: random crop, flip, rotation, color jitter, cutout — искусственно увеличивает размер и разнообразие датасета. Модель не «видит» одно и то же изображение дважды, что снижает переобучение. Для медицинских изображений — domain-specific augmentations (elastic deformation).

  3. Что такое transfer learning для CV и как выбрать backbone?

    Обучаем только «голову» (classifier/detector) поверх замороженного ImageNet-pretrained backbone. Или fine-tune весь backbone с маленьким lr. Выбор: ViT лучше при больших данных, EfficientNet/ConvNeXt — universal backbone. Начинай с предобученного — экономит данные и время.

  4. Как оценить, достаточно ли данных для обучения CV-модели?

    Правило большого пальца: 1000+ примеров на класс для fine-tuning, 10k+ для обучения с нуля. Признаки нехватки: большой gap train/val, высокая variance результатов. Стратегии при нехватке: transfer learning, data augmentation, self-supervised pretraining (DINO, MAE).

Как закрепить тему: End-to-End CV Pipeline

Блок связывает теорию с практикой: где применять идею, какие ошибки проверять и как развиваться дальше.

Когда применять

  • Transfer learning от ImageNet — стандарт, даже при малом объёме данных.
  • Начинай с аугментации после того, как baseline работает — не до.
  • Confusion matrix и per-class metrics — обязательны, accuracy одной недостаточно.

Типичные ошибки

  • Не делать EDA данных перед обучением — дисбаланс и ошибки разметки сразу видны.
  • Применять агрессивную аугментацию без проверки качества аугментированных примеров.
  • Fine-tune весь backbone с тем же lr, что и голову — backbone нужен меньший lr.

Что тюнить / проверить

  • Learning rate для backbone vs head: обычно backbone_lr = head_lr / 10.
  • Augmentation strength: albumentations HorizontalFlip + ColorJitter как baseline.
  • Image resolution: выше = точнее, дороже; начинай с 224, увеличивай при необходимости.

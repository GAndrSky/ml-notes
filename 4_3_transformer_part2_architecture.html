5.3 — Transformer (Часть 2): Архитектура

Positional Encoding · Encoder/Decoder · FFN · Layer Norm · обучение

1. Positional Encoding

Attention — перестановочно-инвариантная операция: порядок токенов не имеет значения без явной информации о позиции. PE вводит её.

Синусоидальный PE (Vaswani et al., 2017)

PE[pos, 2i] = sin(pos / 10000^(2i/d_model)) PE[pos, 2i+1] = cos(pos / 10000^(2i/d_model))

pos — позиция в последовательности, i — индекс измерения (0..d_model/2-1)

Свойства синусоидального PE

1.Каждая позиция имеет уникальный паттерн — однозначная кодировка
2.PE[pos+k] — линейная функция PE[pos] → модель может учить относительные расстояния
3.Нет параметров — детерминировано, обобщается на любую длину
4.Частоты: от 2π (i=0) до 20000π (i=d/2-1) — как спектр
Вход в Transformer: x_emb + PE, где x_emb ∈ ℝL×d — обученные эмбеддинги токенов, PE — фиксированные синусоиды одного размера.

Learnable PE (BERT, GPT)

PE ∈ ℝmax_len × d_model — обучаемая матрица эмбеддингов позиций

Гибче, но не обобщается за пределы max_len. BERT: max_len=512, GPT-2: max_len=1024.

RoPE (Rotary Position Embedding, GPT-NeoX, LLaMA)

Применяется к Q и K через вращение в 2D-подпространствах: q'·k' = f(q,pos_q)·f(k,pos_k) зависит только от |pos_q - pos_k|

Кодирует относительные позиции прямо в attention, обобщается за max_len.

3. Feed-Forward Network (FFN)

FFN(x) = ReLU(x·W₁ + b₁) · W₂ + b₂ W₁ ∈ ℝd_model × d_ff, W₂ ∈ ℝd_ff × d_model d_ff = 4·d_model (стандарт)
Зачем FFN? MHA — линейная операция над значениями (V взвешиваются, не трансформируются). FFN добавляет нелинейность внутри каждой позиции. Интерпретация: FFN хранит «фактические знания» (Geva et al., 2021 — «Transformer Feed-Forward Layers Are Key-Value Memories»).

Современные вариации FFN

SwiGLU (LLaMA):FFN(x) = (W₁x ⊙ σ(W₃x)) · W₂ — вентиль на входе
GeGLU (PaLM):аналогично, но с GELU вместо sigmoid
MoE (Mixtral):Mixture of Experts — выбирает Top-k из N FFN через gating network

4. Layer Normalization (Pre-LN)

LN из блока 3.4 — напоминание в контексте Transformer:

LN(x) = γ ⊙ (x - μ) / √(σ² + ε) + β μ = mean(x, dim=-1) # по оси features σ² = var(x, dim=-1) γ, β ∈ ℝd_model — обучаемые

Residual + Pre-LN в формулах

Encoder слой l: x' = x + MHA(LN(x)) ← residual + Pre-LN x'' = x' + FFN(LN(x')) ← residual + Pre-LN
Почему residual stream важен? Residual connection создаёт «прямой путь» для градиента от loss до эмбеддингов — аналог LSTM cell state. Можно думать об архитектуре как о последовательном уточнении residual stream через добавление информации от attention и FFN.

5. Обучение Transformer

Loss: Cross-Entropy на токенах

L = -1/T · Σ_{t=1}^{T} log p(x_t | x_{<t};θ)

В seq2seq: суммируем CE по всем целевым токенам. В языковом моделировании (GPT): предсказываем следующий токен авторегрессивно.

Label Smoothing (Vaswani et al., 2017)

y_smooth = (1-ε)·y_onehot + ε/V ε = 0.1, V = vocab size

Регуляризует модель, предотвращает сверхуверенные предсказания. Связь с блоком 1.4 (label smoothing через KL).

Warmup + Cosine Schedule (Noam Schedule)

lr(step) = d_model^(-0.5) · min(step^(-0.5), step · warmup^(-1.5))
warmup steps4000
d_model512

Optimizer: Adam с β₁=0.9, β₂=0.98, ε=10⁻⁹

β₂=0.98 (не 0.999!) — более агрессивное отслеживание второго момента ε=10⁻⁹ (не 10⁻⁸) — меньший шум при малых градиентах на старте

Dropout в Transformer

1.После каждого Sublayer перед сложением с residual: p=0.1
2.На attention weights: Dropout(softmax(·))
3.На embeddings + PE

Inference: авторегрессивная генерация

Greedy:x_t = argmax p(x|x_{<t}) — детерминировано, но скучно
Top-k:сэмплируем из top-k токенов по вероятности
Top-p (nucleus):сэмплируем из минимального множества с Σp ≥ p (динамический k)
Temperature:p ← softmax(logits/T); T<1 — острее, T>1 — разнообразнее
KV-Cache:при каждом шаге пересчитываем только K,V для нового токена, остальные кешируем → O(L) вместо O(L²) на шаг

Grouped Query Attention (GQA)

MHA: h голов Q, h голов K, h голов V GQA: h голов Q, g голов K, g голов V (g << h) MQA: h голов Q, 1 голова K, 1 голова V

GQA в LLaMA-3 (h=32, g=8): K,V в 4× меньше — KV-Cache в 4× меньше памяти.

RMSNorm (LLaMA вместо LayerNorm)

RMSNorm(x) = γ ⊙ x / RMS(x) RMS(x) = √(1/d · Σ xᵢ²) ← без вычитания среднего

Нет bias — на 7% быстрее LN, аналогичное качество.

Сравнение: Encoder-only vs Decoder-only vs Enc-Dec

Encoder-only (BERT)

Задачи:классификация, NER, QA (extractive)
Обучение:MLM (маскируем 15% токенов)
Attn:bidirectional — видит всё

Decoder-only (GPT)

Задачи:генерация, CLM
Обучение:next-token prediction
Attn:causal — только прошлое

7. Полный Transformer на PyTorch

Positional Encoding

import torch, torch.nn as nn, math class SinusoidalPE(nn.Module): def __init__(self, d_model, max_len=5000, dropout=0.1): super().__init__() self.drop = nn.Dropout(dropout) pe = torch.zeros(max_len, d_model) pos = torch.arange(max_len).unsqueeze(1).float() # (max_len,1) div = torch.exp(torch.arange(0, d_model, 2).float() * (-math.log(10000) / d_model)) # (d/2,) pe[:, 0::2] = torch.sin(pos * div) # чётные измерения pe[:, 1::2] = torch.cos(pos * div) # нечётные self.register_buffer('pe', pe.unsqueeze(0)) # (1,max_len,d) def forward(self, x): # x: (B, L, d) return self.drop(x + self.pe[:, :x.size(1)])

Encoder Block

class EncoderBlock(nn.Module): def __init__(self, d_model, num_heads, d_ff, dropout=0.1): super().__init__() self.attn = nn.MultiheadAttention(d_model, num_heads, dropout=dropout, batch_first=True) self.ffn = nn.Sequential( nn.Linear(d_model, d_ff), nn.ReLU(), nn.Dropout(dropout), nn.Linear(d_ff, d_model) ) self.ln1 = nn.LayerNorm(d_model) self.ln2 = nn.LayerNorm(d_model) self.drop = nn.Dropout(dropout) def forward(self, x, src_key_padding_mask=None): # Pre-LN Self-Attention + residual x2 = self.ln1(x) attn_out, _ = self.attn(x2, x2, x2, key_padding_mask=src_key_padding_mask) x = x + self.drop(attn_out) # Pre-LN FFN + residual x = x + self.drop(self.ffn(self.ln2(x))) return x

Decoder Block

class DecoderBlock(nn.Module): def __init__(self, d_model, num_heads, d_ff, dropout=0.1): super().__init__() self.self_attn = nn.MultiheadAttention(d_model, num_heads, dropout=dropout, batch_first=True) self.cross_attn = nn.MultiheadAttention(d_model, num_heads, dropout=dropout, batch_first=True) self.ffn = nn.Sequential(nn.Linear(d_model, d_ff), nn.ReLU(), nn.Dropout(dropout), nn.Linear(d_ff, d_model)) self.ln1 = nn.LayerNorm(d_model) self.ln2 = nn.LayerNorm(d_model) self.ln3 = nn.LayerNorm(d_model) self.drop = nn.Dropout(dropout) def forward(self, x, memory, tgt_mask=None, memory_key_padding_mask=None): # Masked Self-Attention x2 = self.ln1(x) sa, _ = self.self_attn(x2, x2, x2, attn_mask=tgt_mask) x = x + self.drop(sa) # Cross-Attention x2 = self.ln2(x) ca, _ = self.cross_attn(x2, memory, memory, key_padding_mask=memory_key_padding_mask) x = x + self.drop(ca) # FFN x = x + self.drop(self.ffn(self.ln3(x))) return x

Полный Transformer

class Transformer(nn.Module): def __init__(self, src_vocab, tgt_vocab, d_model=512, N=6, num_heads=8, d_ff=2048, dropout=0.1, max_len=512): super().__init__() self.src_emb = nn.Embedding(src_vocab, d_model) self.tgt_emb = nn.Embedding(tgt_vocab, d_model) self.pe = SinusoidalPE(d_model, max_len, dropout) self.encoder = nn.ModuleList([EncoderBlock(d_model,num_heads,d_ff,dropout) for _ in range(N)]) self.decoder = nn.ModuleList([DecoderBlock(d_model,num_heads,d_ff,dropout) for _ in range(N)]) self.ln_out = nn.LayerNorm(d_model) self.proj = nn.Linear(d_model, tgt_vocab) self.scale = math.sqrt(d_model) self._init_weights() def _init_weights(self): for p in self.parameters(): if p.dim() > 1: nn.init.xavier_uniform_(p) # рекомендация оригинальной статьи def forward(self, src, tgt, src_pad_mask=None, tgt_causal_mask=None): # Encoder e = self.pe(self.src_emb(src) * self.scale) for layer in self.encoder: e = layer(e, src_key_padding_mask=src_pad_mask) # Decoder d = self.pe(self.tgt_emb(tgt) * self.scale) for layer in self.decoder: d = layer(d, e, tgt_mask=tgt_causal_mask, memory_key_padding_mask=src_pad_mask) return self.proj(self.ln_out(d)) # (B, T, tgt_vocab) # Пример использования model = Transformer(src_vocab=32000, tgt_vocab=32000) print(f"Параметры: {sum(p.numel() for p in model.parameters()):,}") # → Параметры: ~44M при d=512, N=6

Noam LR Schedule

class NoamScheduler: def __init__(self, optimizer, d_model, warmup_steps=4000): self.opt = optimizer self.d = d_model self.ws = warmup_steps self.step_num = 0 def step(self): self.step_num += 1 lr = self.d**(-0.5) * min(self.step_num**(-0.5), self.step_num * self.ws**(-1.5)) for pg in self.opt.param_groups: pg['lr'] = lr optimizer = torch.optim.Adam(model.parameters(), lr=1, betas=(0.9,0.98), eps=1e-9) scheduler = NoamScheduler(optimizer, d_model=512, warmup_steps=4000)

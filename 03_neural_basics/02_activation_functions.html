3.2 Функции активации

ReLU, GELU, SiLU, Sigmoid, Tanh, Softmax — формулы, производные, почему именно эти

🤔 Зачем вообще нужна нелинейность?

Без функции активации стек линейных слоёв коллапсирует в одну матрицу:

A₂(A₁x + b₁) + b₂ = (A₂A₁)x + (A₂b₁+b₂) = Ax + b

Сколько бы слоёв мы ни добавили — получаем одну линейную операцию. Нелинейная активация f разрывает этот коллапс.

💡 Три требования к хорошей активации:
1. Нелинейность — иначе вся сеть линейная
2. Дифференцируемость (почти везде) — нужна для backprop
3. Ненасыщение — чтобы градиент не «умирал» при обратном распространении

💀 Проблема затухающего градиента (Vanishing Gradient)

Почему Sigmoid и Tanh опасны в глубоких сетях

При backprop градиент умножается на производную f′(z) в каждом слое:

∂L/∂x = f′(z) · (вышестоящий градиент)

У sigmoid: max(σ′) = 0.25. За 10 слоёв:

0.25¹⁰ ≈ 9.5 × 10⁻⁷ → градиент мертв
⚠️ Это главная причина, почему до 2010-х глубокие сети не обучались. Решение пришло с ReLU.

☠️ Dying ReLU — «мёртвые нейроны»

ReLU(z) = 0 при z < 0. Производная ReLU′(z) = 0 при z < 0.

Если из-за большого learning rate или неудачной инициализации нейрон попадает в зону z < 0 для всех обучающих примеров — он «умирает»: вес никогда не обновляется.

∂L/∂w = 0 · δ = 0   ∀ примеров → нейрон мёртв навсегда

Решения: Leaky ReLU, ELU, инициализация He, маленький lr, batch norm.

🎯 Softmax: вектор → распределение вероятностей

softmax(z)ᵢ = eᶻⁱ / Σⱼ eᶻʲ

Числовая стабильность: при больших z ваш e^z переполняется. Трюк:

softmax(z) = softmax(z - max(z)) (вычитание константы не меняет результат)

Якобиан softmax

∂pᵢ/∂zⱼ = pᵢ(δᵢⱼ - pⱼ) = diag(p) - p·pᵀ

Матрица (K×K), не вектор — мы уже считали это в блоке 1.2 (Якобиан).

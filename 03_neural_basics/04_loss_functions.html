3.4 Loss-функции

MSE, BCE, CrossEntropy, Focal Loss — выводы из MLE, геометрия, градиенты

🎯 Единый фрейм: всё из MLE

Любая loss-функция — это отрицательное log-правдоподобие при выборе распределения для шума/меток:

L(θ) = −log p(y | x, θ) ← NLL = negative log-likelihood Минимизировать L(θ) ⟺ максимизировать log p(y|x,θ)
Распределение y|xNLL →LossЗадача
Гауссово N(f_θ(x), σ²)−log NMSEРегрессия
Бернулли Ber(σ(f_θ(x)))−log BerBCEБинарная классификация
Категориальное Cat(softmax(f_θ))−log CatCrossEntropyМногоклассовая
💡 Меняешь предположение о распределении — получаешь другую loss. Архитектура сети не меняется, меняется только интерпретация выходного слоя.

📊 Интерактивный атлас: выбери функцию потерь


📐 Вывод MSE из MLE (гауссовский шум)

Предполагаем: метки зашумлены гауссовски — y = f_θ(x) + ε, ε ~ N(0, σ²)


Шаг 1: p(y|x,θ) = (1/√(2πσ²)) · exp(−(y − f_θ(x))² / (2σ²))
Шаг 2: log p = −½log(2πσ²) − (y − f_θ(x))² / (2σ²)
Шаг 3: NLL = −log p = const + (y − f_θ(x))² / (2σ²)
Шаг 4: L_MSE = (1/N)·Σ (yᵢ − ŷᵢ)² ← минимизация NLL ≡ MSE
Градиент: ∂L/∂ŷ = −2(y − ŷ)/N = 2(ŷ − y)/N
💡 σ² — просто константа при оптимизации. MSE неявно предполагает, что ошибки распределены нормально с постоянной дисперсией (гомоскедастичность). Выбросы сильно штрафуются из-за квадрата.

📐 Вывод BCE из MLE (распределение Бернулли)

Предполагаем: y ∈ {0,1}, вероятность класса 1: p̂ = σ(f_θ(x))


Шаг 1: p(y|x,θ) = p̂ʸ · (1−p̂)^(1−y) ← PMF Бернулли
Шаг 2: log p = y·log(p̂) + (1−y)·log(1−p̂)
Шаг 3: L_BCE = −(1/N)·Σ [yᵢ·log(p̂ᵢ) + (1−yᵢ)·log(1−p̂ᵢ)]
Градиент (через sigmoid): ∂L/∂z = p̂ − y ← элегантно!

💡 Если y=1: loss = −log(p̂). Чем увереннее модель (p̂→1), тем меньше loss. Если p̂→0 при y=1: −log(0)→∞.

Числовой пример

p̂ (предсказание): 0.70

📐 Вывод CrossEntropy из MLE (категориальное)

Предполагаем: y — one-hot вектор, вероятности: p̂ = softmax(f_θ(x))


Шаг 1: p(y|x,θ) = Πₖ p̂ₖ^(yₖ) ← PMF категориального
Шаг 2: log p = Σₖ yₖ · log(p̂ₖ) = log(p̂_c), c = истинный класс
Шаг 3: L_CE = −(1/N)·Σᵢ log(p̂ᵢ_cᵢ) = −(1/N)·Σᵢ Σₖ yᵢₖ·log(p̂ᵢₖ)
Связь с информационной теорией: L_CE = H(y, p̂) = H(y) + KL(y‖p̂) = KL(y‖p̂) (т.к. H(y)=0 для one-hot)
Градиент (softmax+CE): ∂L/∂zₖ = p̂ₖ − yₖ ← идентично BCE!

🔍 Focal Loss — для несбалансированных классов

Стандартный BCE одинаково штрафует лёгкие и сложные примеры. При дисбалансе классов (99% фона, 1% объектов) модель «ленится».


FL(p̂, y) = −αₜ · (1 − pₜ)^γ · log(pₜ) pₜ = p̂ if y=1 1−p̂ if y=0 α — балансировочный вес класса γ — focusing parameter (обычно 2)
💡 (1−pₜ)^γ — это «модулирующий фактор».
Лёгкий пример (p̂→1): (1−0.9)²=0.01 → loss почти 0
Сложный пример (p̂=0.5): (1−0.5)²=0.25 → полная loss
Фокус на трудных примерах!

⚡ Сравнение градиентов loss-функций

Градиент по логиту z (ключевой факт)

MSE: ∂L/∂ŷ = 2(ŷ − y)/N (линейный, симметричный) BCE+sigmoid: ∂L/∂z = (p̂ − y)/N (не зависит от σ′!) CE+softmax: ∂L/∂zₖ = (p̂ₖ − yₖ)/N (элегантно: pred − true)
💡 Градиент BCE через z (не через p̂!) отменяет σ′(z) знаменателя — именно поэтому sigmoid+BCE не имеет проблемы vanishing gradient в выходном слое.

📐 MAE и Huber Loss — робастная регрессия

MAE из MLE (распределение Лапласа)

Предполагаем: y = f_θ(x) + ε, ε ~ Laplace(0, b)

p(y|x,θ) = (1/2b)·exp(−|y − f_θ(x)| / b) log p = −log(2b) − |y − f_θ(x)| / b NLL = const + |y − ŷ| / b L_MAE = (1/N)·Σ |yᵢ − ŷᵢ| ∂L/∂ŷ = sign(ŷ − y) ← субградиент!
⚠️ MAE негладкий в 0 — субградиент, оптимизаторы хуже сходятся. Зато робастен к выбросам (линейный штраф, не квадратный).

Huber Loss — компромисс MSE+MAE

Lδ(y, ŷ) = { ½(y−ŷ)² если |y−ŷ| ≤ δ ← MSE δ·|y−ŷ| − ½δ² если |y−ŷ| > δ ← MAE Градиент: ∂L/∂ŷ = (ŷ−y) если |e| ≤ δ ∂L/∂ŷ = δ·sign(ŷ−y) если |e| > δ
💡 δ — порог переключения. При δ→∞: Huber ≈ MSE. При δ→0: Huber ≈ MAE.
Гладкость MSE там, где важна точность; устойчивость MAE там, где выбросы.
δ = 1.0 — MSE   — MAE   — Huber

🔢 Числовая стабильность: log-sum-exp трюк

Наивный softmax при больших логитах даёт overflow/underflow:

Наивно (ПЛОХО): exp(1000) = ∞ → nan в softmax log-sum-exp трюк (ХОРОШО): m = max(z₁,...,zₖ) log Σ exp(zᵢ) = m + log Σ exp(zᵢ − m) → всегда exp(≤0), нет overflow Стабильный softmax: p̂ᵢ = exp(zᵢ − m) / Σⱼ exp(zⱼ − m)
💡 PyTorch F.cross_entropy и F.log_softmax применяют этот трюк автоматически. Именно поэтому передавай логиты, а не вероятности в CrossEntropyLoss.

🗂️ Шпаргалка: когда какую loss выбрать

Задача Выход сети Loss PyTorch Когда менять
Регрессия, нет выбросов ŷ ∈ ℝ MSE MSELoss() Стандарт
Регрессия, есть выбросы ŷ ∈ ℝ Huber / MAE HuberLoss() / L1Loss() Шумные данные
Бинарная классификация z ∈ ℝ (логит) BCE BCEWithLogitsLoss() Стандарт
Дисбаланс классов z ∈ ℝ (логит) Focal FocalLoss (custom) 99%/1% split
Многоклассовая (K классов) z ∈ ℝᴷ (логиты) CrossEntropy CrossEntropyLoss() Стандарт
Многометочная (multi-label) z ∈ ℝᴷ (логиты) BCE × K BCEWithLogitsLoss() Несколько меток сразу
💡 Правило: смотри на тип выхода и распределение. Чем больше хочешь штрафовать уверенные ошибки — тем больше логарифмический штраф (CE/BCE) лучше линейного (MAE) или квадратного (MSE).

Концепции

Любая loss = NLL при выборе распределения для y|x
MSE ← Гауссово, BCE ← Бернулли, CE ← Категориальное, MAE ← Лапласово
Градиент BCE/CE через логит: ∂L/∂z = p̂ − y (элегантно и стабильно)
Focal Loss подавляет лёгкие примеры через (1−pₜ)^γ — для дисбаланса классов

Практика PyTorch

CrossEntropyLoss и BCEWithLogitsLoss принимают ЛОГИТЫ — не вероятности!
log-sum-exp трюк внутри этих функций предотвращает overflow
При выбросах: Huber (δ=1) вместо MSE; при дисбалансе: Focal (γ=2)
Multi-label ≠ Multiclass: используй BCE×K, а не CrossEntropy

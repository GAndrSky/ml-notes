Теория глубже

Нейрон - локальный детектор признака

Один нейрон не понимает объект целиком. Он проверяет один вопрос: похож ли вход на тот шаблон, который записан в его весах. Если похож, pre-activation становится большой; если нет, сигнал слабый или отрицательный.

В глубокой сети такие маленькие детекторы собираются в иерархию. Ранние слои реагируют на простые паттерны, следующие слои комбинируют их в более абстрактные признаки, а последние слои превращают представление в решение.

  • Веса задают направление, на которое нейрон реагирует.
  • Bias задаёт порог, после которого реакция становится заметной.
  • Активация решает, сколько сигнала передать дальше.

w = [w₁, w₂, ..., wₙ]⊤ ∈ ℝⁿ     (веса)
b ∈ ℝ                            (bias)

Аналогия: Как итоговая оценка из нескольких факторов: важные признаки получают больший вес.

y = f(z)                                             (activation / нелинейность)

💡 Почему два шага? Линейная часть z — это «взвешенное голосование» входов. Функция активации f — это «решение» нейрона что делать с результатом. Без f вся сеть сколлапсировала бы в одну линейную операцию.

Скалярное произведение — сердце нейрона

w⊤x = Σᵢ wᵢxᵢ = |w|·|x|·cos(θ)
Что означают символы
ΣсуммаСкладывает вклад объектов, классов, признаков или моделей.
σстандартное отклонениеМера разброса вокруг среднего.
zlogitСырой линейный score до sigmoid.
σ(z)вероятностьScore, сжатый в диапазон от 0 до 1.
thresholdпорог решенияГраница, после которой вероятность переводится в класс 1.
Интуиция: Формула переводит линейный score в вероятность и затем в решение через threshold.
Аналогия: Как индикатор риска: сырой балл превращается в вероятность события.
Числовой пример: Если logit=0, sigmoid даёт 0.5; если logit=2, вероятность уже около 0.88.

Это косинусное сходство до нормировки: нейрон сильно активируется, когда вход x похож на вектор весов w (угол θ мал → cos(θ) ≈ 1).

Теория глубже

Скалярное произведение измеряет совпадение направления

Геометрически w^T x большой, когда входной вектор направлен примерно туда же, куда вектор весов. Поэтому нейрон можно читать как измеритель сходства: он усиливает входы, которые совпадают с его learned direction.

Длина вектора тоже влияет на значение. Поэтому scaling и normalization важны: один признак с большим масштабом может доминировать в dot product, даже если он не несёт больше смысла.

  • Угол между x и w отвечает за похожесть направления.
  • Длина x отвечает за силу входного сигнала.
  • Длина w отвечает за чувствительность нейрона к этому направлению.

⚖️ Роль bias b

Bias — это порог активации. Без него нейрон вынужден проводить гиперплоскость через начало координат.


Без bias: w⊤x = 0 → плоскость через (0,0)
С bias: w⊤x + b = 0 → свободная плоскость

Теория глубже

Bias сдвигает границу решения без изменения направления

Веса задают ориентацию гиперплоскости, а bias двигает её относительно начала координат. Без bias нейрон обязан проводить границу через ноль, что искусственно ограничивает модель.

Оптимизационно bias часто берёт на себя базовый уровень активации. Если признак должен срабатывать даже при небольшом входе, bias может сдвинуть threshold; если нужен строгий фильтр, bias делает нейрон менее склонным активироваться.

  • w меняет наклон разделяющей границы.
  • b меняет положение границы.
  • Большой положительный bias повышает базовую активность, отрицательный - делает нейрон более строгим.

💻 Реализация в NumPy и PyTorch

import numpy as np # Один нейрон вручную x = np.array([2.0, -1.0, 0.5]) # вход (3,) w = np.array([0.5, -0.3, 0.8]) # веса (3,) b = 0.1 # bias # Шаг 1: pre-activation z = w @ x + b # dot product + bias # z = 0.5*2 + (-0.3)*(-1) + 0.8*0.5 + 0.1 = 1.8 # Шаг 2: activation (ReLU) y = np.maximum(0, z) # ReLU: max(0, z) print(f"z = {z:.4f}") # z = 1.8000 print(f"y = {y:.4f}") # y = 1.8000 # Батч: x.shape = (batch, features) X = np.random.randn(32, 3) # 32 примера, 3 признака W = np.array([0.5, -0.3, 0.8]) Z = X @ W + b # shape: (32,) Y = np.maximum(0, Z) # shape: (32,)
import torch import torch.nn as nn # nn.Linear = один слой нейронов # in_features=3, out_features=1 → один нейрон layer = nn.Linear(in_features=3, out_features=1) # Веса инициализированы случайно (Kaiming uniform по умолчанию) print(layer.weight.shape) # (1, 3) — матрица W print(layer.bias.shape) # (1,) — вектор b # Ручная установка весов with torch.no_grad(): layer.weight.copy_(torch.tensor([[0.5, -0.3, 0.8]])) layer.bias.copy_(torch.tensor([0.1])) # Forward pass (батч из 4 примеров) x = torch.tensor([[2.0, -1.0, 0.5], [1.0, 0.0, 2.0], [-1.0, 1.0, 0.0], [0.5, -0.5, 1.0]]) # shape: (4, 3) z = layer(x) # shape: (4, 1) y = torch.relu(z) # shape: (4, 1) # Вручную: z = x @ W.T + b z_manual = x @ layer.weight.T + layer.bias # Совпадает!
import torch # Нейрон с autograd для проверки градиентов x = torch.tensor([2.0, -1.0, 0.5]) w = torch.tensor([0.5, -0.3, 0.8], requires_grad=True) b = torch.tensor(0.1, requires_grad=True) # Forward z = w @ x + b y = torch.relu(z) # Допустим, loss = y (MSE с target=0 для примера) loss = y # упрощённо loss.backward() # Градиенты: # ∂loss/∂w = ∂y/∂z · ∂z/∂w = 1 * x = x (т.к. z>0, ReLU′=1) print(w.grad) # tensor([2., -1., 0.5]) = x print(b.grad) # tensor(1.) = 1 # Геометрический смысл: ∂z/∂w = x # Градиент по весам = входной вектор x # → вес обновляется пропорционально входу!

Теория глубже

Универсальная аппроксимация не означает лёгкое обучение

Теорема об универсальной аппроксимации говорит, что достаточно широкая сеть может представить очень много функций. Но она не гарантирует, что gradient descent быстро найдёт нужные веса, что данных хватит или что модель будет хорошо обобщать.

Практический смысл другой: нелинейные нейроны дают достаточно богатый язык для построения функций. А качество уже зависит от архитектуры, initialization, loss, optimizer, regularization и данных.

  • Expressivity отвечает за то, что сеть может представить.
  • Optimization отвечает за то, получится ли это найти.
  • Generalization отвечает за то, будет ли найденная функция работать вне train.

Что нейрон делает на самом деле

Нейрон удобно понимать не как "маленький мозг", а как механизм взвешивания свидетельств. Каждый входной признак приносит свой сигнал, вес говорит насколько этому сигналу стоит доверять, а bias задаёт базовую склонность нейрона сработать даже при нулевом входе. В сумме нейрон отвечает на вопрос: хватает ли собранных аргументов, чтобы активировать конкретный паттерн?

Геометрически один нейрон задаёт гиперплоскость в пространстве признаков. По одну сторону этой границы сигнал считается "достаточно сильным", по другую — нет. Именно поэтому одиночный нейрон умеет хорошо разделять только линейно разделимые структуры. Сила нейросети появляется не в одном нейроне, а в композиции многих таких простых правил, которые постепенно превращают исходное пространство в более удобное.

Почему это важно для понимания глубины сети

Если смотреть на слой целиком, он уже не просто "решает задачу", а строит новое представление данных. Один набор нейронов может выделять края, другой — шаблоны, третий — более абстрактные комбинации. То есть глубина сети — это глубина переописания объекта через всё более полезные признаки.

Теоретический минимум: нейрон — это линейная граница плюс смещение, а вся нейросеть — композиция таких границ, которая постепенно строит удобную систему признаков для задачи.
Теоретический слой

Нейрон как обучаемая аффинная проверка признаков

Один нейрон берёт входной вектор, считает взвешенную сумму, добавляет bias и пропускает результат через активацию. В этой операции уже есть главная идея deep learning: признаки получают обучаемые веса, а сеть настраивает, какие комбинации важны.

Перцептрон без нелинейностей остаётся линейной моделью. Глубина становится полезной, когда между линейными слоями появляются активации, позволяющие строить композицию нелинейных преобразований.

Геометрическая интуиция

  • w^T x + b задаёт гиперплоскость; знак score показывает сторону пространства.
  • Bias сдвигает границу без изменения её наклона.
  • Несколько нейронов создают несколько направлений, по которым слой измеряет вход.

Вероятностная интерпретация

  • Сам score не является вероятностью; вероятность появляется после sigmoid или softmax.
  • Веса можно читать как learned evidence: положительный вклад повышает score, отрицательный снижает.
  • Калибровка вероятностей не гарантируется только архитектурой нейрона.

Оптимизационный смысл

  • Градиент показывает, как изменить каждый вес, чтобы уменьшить loss.
  • Если входы не масштабированы, веса получают градиенты разного масштаба.
  • Initialization задаёт стартовые направления, learning rate задаёт размер шагов.

Практика и типичные ошибки

  • Следи за shapes: batch dimension, input features, output units.
  • Не забывай bias: без него граница обязана проходить через начало координат.
  • Подбирай activation под задачу: sigmoid, softmax или linear output.

Мини-пример: Если нейрон оценивает риск, вес возраста +0.4 повышает score при росте возраста, а вес скидки -0.2 снижает score при наличии скидки.

Как закрепить тему: Перцептрон и нейрон

Этот блок связывает теорию с практическим решением: где применять идею, какие ошибки проверять и какие ручки менять в эксперименте.

Когда применять

  • Возвращайся к теме «Перцептрон и нейрон», когда нужно понять, как слой преобразует сигнал и почему градиент проходит или затухает.
  • Эти темы нужны перед обучением больших моделей: без них трудно отлаживать loss, активации и размерности.
  • Используй формулы как карту forward/backward pass, а не как отдельные факты.

Типичные ошибки

  • Путать logits, probabilities и labels: от этого ломается выбор loss-функции и интерпретация выхода.
  • Игнорировать scale: слишком большие logits, активации или loss быстро приводят к нестабильному обучению.
  • Считать shape второстепенной деталью: большинство багов в нейросетях начинается с неправильной оси.

Что тюнить

  • Проверяй activation/loss pair, initialization, learning rate и нормализацию.
  • Следи за распределением активаций и градиентов по слоям: нули, NaN и взрыв масштаба показывают проблему раньше метрики.
  • Для классификации отдельно настраивай threshold или calibration, если важны вероятности.

Exercises: проверь, что нейрон понятен не на уровне терминов

Не пропускай упражнения: если ты можешь вывести формулу, реализовать её и объяснить ошибку в shapes, тема действительно закреплена.

DerivationFoundational

Для нейрона \(z=w^Tx+b\) выпиши вручную \(\partial z/\partial w_i\), \(\partial z/\partial x_i\) и \(\partial z/\partial b\). Объясни, почему градиент по весу равен соответствующему входному признаку.

Show solution

\(\partial z/\partial w_i=x_i\), потому что \(z=\sum_i w_i x_i+b\). По входу \(\partial z/\partial x_i=w_i\), по bias \(\partial z/\partial b=1\). Вес получает тем больший градиент, чем активнее был его вход.

CodingFoundational

Реализуй один нейрон на NumPy: вход \(X\) shape `[batch, features]`, веса `w` shape `[features]`, bias scalar. Верни logits shape `[batch]` и проверь пример на batch из 3 объектов.

Show solution
import numpy as np

X = np.array([[1.0, 2.0], [0.0, -1.0], [3.0, 1.0]])
w = np.array([0.5, -0.25])
b = 0.1
z = X @ w + b
print(z.shape)  # (3,)
print(z)        # [ 0.1   0.35  1.35]
ConceptualIntermediate

Почему один нейрон без нелинейности не может решить XOR, даже если подобрать любые веса и bias?

Show solution

Один нейрон задаёт одну линейную границу. XOR требует разделить два диагонально расположенных positives от двух negatives, что одной прямой в 2D невозможно. Нужна композиция нескольких границ и нелинейность.

DebuggingIntermediate

У тебя `X.shape = [32, 10]`, `w.shape = [32]`. Почему `X @ w` не является корректным нейроном и какой shape должен быть у `w`?

Show solution

`32` - это batch dimension, а не число признаков. Вес должен соответствовать признакам: `w.shape = [10]`, тогда каждый объект из batch получает свой scalar logit.

Rubber duckHard

Объясни вслух, почему bias не является "лишним весом", а сдвигает границу решения независимо от наклона.

Show solution

Веса задают ориентацию гиперплоскости, а bias двигает её относительно начала координат. Без bias граница обязана проходить через ноль, что часто искусственно ограничивает модель.

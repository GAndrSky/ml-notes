Зачем Attention? Проблемы RNN

RNN — узкое место

1.Информация о шаге t=1 должна пройти через все T hidden states до шага T → затухание
2.Последовательная обработка — нельзя распараллелить
3.Фиксированный вектор контекста в seq2seq — «бутылочное горлышко»

Attention — решение

1.Прямое соединение между любыми двумя позициями — O(1) путь
2.Весь вход обрабатывается параллельно
3.Каждый токен «смотрит» на все остальные с разными весами

Ключевая идея: запрос, ключ, значение

Аналогия: поиск по базе данных.

Query Q — «что я ищу?» (запрос от текущего токена)
Key K — «что у меня есть?» (метка каждого токена в контексте)
Value V — «что вернуть?» (контент каждого токена)
Softmax(Q·Kᵀ/√dₖ) даёт веса совместимости Q с каждым K. Эти веса используются для взвешенной суммы V.

Scaled Dot-Product Attention — полный вывод

Attention(Q, K, V) = softmax( Q·Kᵀ / √dₖ ) · V

Q ∈ ℝL×dₖ, K ∈ ℝS×dₖ, V ∈ ℝS×dᵥ → выход ∈ ℝL×dᵥ

Почему делим на √dₖ?

Шаг 1.Q и K инициализированы из N(0,1). Скалярное произведение qᵢᵀkⱼ = Σₖ qₖkₖ — сумма dₖ независимых слагаемых.
Шаг 2.Var[qᵢᵀkⱼ] = dₖ → std = √dₖ
Шаг 3.Без нормировки при большом dₖ логиты огромны → softmax насыщается → градиент ≈ 0
Шаг 4.Делим на √dₖ → Var[qᵢᵀkⱼ/√dₖ] = 1 → softmax в рабочем диапазоне ✓

Additive vs Dot-Product Attention

Dot-Product (Luong)
score(q,k) = qᵀk / √dₖ
Быстрее (matmul), лучше при больших dₖ
Additive (Bahdanau)
score(q,k) = vᵀ·tanh(Wq·q + Wk·k)
Медленнее, иногда лучше при малых dₖ

Пошаговое вычисление (числовой пример)

L=3 токена, dₖ=2, dᵥ=2

Шаг 1.Q = [[1,0],[0,1],[1,1]], K = [[1,0],[0,1],[1,1]], V = [[1,0],[0,1],[1,1]]
Шаг 2.QKᵀ = [[1,0,1],[0,1,1],[1,1,2]] (матрица совместимости 3×3)
Шаг 3./ √2 = [[0.71,0,0.71],[0,0.71,0.71],[0.71,0.71,1.41]]
Шаг 4.softmax по строкам → A[0] = [0.42, 0.21, 0.37] (сумма=1)
Шаг 5.A·V → выходной вектор для токена 0: 0.42·[1,0]+0.21·[0,1]+0.37·[1,1] = [0.79, 0.58]

Multi-Head Attention

Один head ограничен — учит одну «проекцию» совместимости. MHA запускает h параллельных head, каждый в своём подпространстве.

headi = Attention(Q·WiQ, K·WiK, V·WiV) MHA(Q,K,V) = Concat(head1, …, headh) · WO

Проекции: WiQ ∈ ℝd_model×d_k, WiK ∈ ℝd_model×d_k, WiV ∈ ℝd_model×d_v

Стандартно: dₖ = dᵥ = d_model / h. Для d_model=512, h=8: dₖ = 64

Параметры MHA

h·(d_model·dₖ + d_model·dₖ + d_model·dᵥ) + d_model·d_model = h·d_model·(2dₖ + dᵥ) + d_model² = 4·d_model² (при dₖ=dᵥ=d_model/h)
Зачем несколько голов?
head 1 может учить синтаксические связи (подлежащее↔сказуемое),
head 2 — кореференцию (местоимение↔антецедент),
head 3 — позиционные паттерны (соседние токены), и т.д.

Маски в Attention

Padding Mask

Батч содержит последовательности разной длины — дополняем PAD-токенами. Маскируем, чтобы attention не смотрел на паддинг:

score[i][j] = -∞, если j — PAD-токен → softmax: exp(-∞)=0 → вес=0 ✓

Causal Mask (Look-ahead Mask)

В декодере (авторегрессивная генерация) токен позиции t не должен видеть t+1, t+2, …:

M[i][j] = -∞ при j > i M[i][j] = 0 при j ≤ i

Это нижнетреугольная матрица (lower triangular mask).

PyTorch: создание масок

import torch import torch.nn.functional as F # Causal mask (look-ahead) для декодера def causal_mask(L, device='cpu'): # True = маскировать (запрещено смотреть) mask = torch.triu(torch.ones(L, L, device=device), diagonal=1).bool() return mask # (L, L) upper triangular # Padding mask def padding_mask(lengths, max_len): # lengths: (B,) — реальные длины последовательностей idx = torch.arange(max_len).unsqueeze(0) # (1, max_len) return idx >= lengths.unsqueeze(1) # (B, max_len) True=PAD # Применение в scaled_dot_product_attention scores = Q @ K.transpose(-2,-1) / (dk**0.5) if mask is not None: scores = scores.masked_fill(mask, float('-inf')) attn_weights = F.softmax(scores, dim=-1)

Self-Attention vs Cross-Attention

Self-Attention
Q = K = V = X (один вход)
Используется в энкодере и в self-attn декодера
Cross-Attention
Q из декодера, K=V из энкодера
Позволяет декодеру «смотреть» на вход

Вычислительная сложность

Self-Attention: O(L²·d) — квадратично по длине! FFN: O(L·d²) — линейно по длине

При L=1024, d=512: L²·d = 536M, L·d² = 268M — сопоставимо

При L=8192: L²·d = 34G — attention становится узким местом → Flash Attention, Sparse Attention

Производная Attention по Q

Пусть A = softmax(QKᵀ/√dₖ), выход O = A·V ∂L/∂Q = dA/dQ · (∂A_soft/∂logits) · V dA/dQ: каждая строка Aᵢ = softmax(qᵢKᵀ/√dₖ) ∂Aᵢ/∂qᵢ = (diag(Aᵢ) - Aᵢ·Aᵢᵀ) · Kᵀ/√dₖ ← Якобиан softmax из 1.2

Flash Attention — зачем нужен

Проблема:Матрица QKᵀ при L=4096 занимает 4096²×4 байт = 64 МБ на голову × 32 головы = 2 GB — не влезает в SRAM GPU
Flash Attention:Вычисляет attention блоками (tiling) без материализации полной матрицы QKᵀ в HBM. Сложность памяти: O(L) вместо O(L²)
Связь с Info Theory (блок 1.4): Attention — это взвешенная сумма, где веса — категориальное распределение. Можно интерпретировать как H(attention) — энтропию внимания: низкая энтропия → «острый» фокус на одном токене, высокая → равномерное рассеивание.

PyTorch — реализация с нуля

Scaled Dot-Product Attention

import torch, torch.nn as nn, math import torch.nn.functional as F def scaled_dot_product_attention(Q, K, V, mask=None): """ Q: (B, H, L, dₖ) K: (B, H, S, dₖ) V: (B, H, S, dᵥ) mask: (B, 1, L, S) или (L, S) — True = маскировать """ dk = Q.shape[-1] scores = Q @ K.transpose(-2, -1) / math.sqrt(dk) # (B,H,L,S) if mask is not None: scores = scores.masked_fill(mask, float('-inf')) weights = F.softmax(scores, dim=-1) # (B,H,L,S) return weights @ V, weights # (B,H,L,dᵥ), weights

Multi-Head Attention

class MultiHeadAttention(nn.Module): def __init__(self, d_model, num_heads, dropout=0.1): super().__init__() assert d_model % num_heads == 0 self.h = num_heads self.dk = d_model // num_heads # Объединённые проекции (эффективнее отдельных) self.W_q = nn.Linear(d_model, d_model, bias=False) self.W_k = nn.Linear(d_model, d_model, bias=False) self.W_v = nn.Linear(d_model, d_model, bias=False) self.W_o = nn.Linear(d_model, d_model, bias=False) self.drop = nn.Dropout(dropout) def split_heads(self, x): B, L, D = x.shape return x.view(B, L, self.h, self.dk).transpose(1, 2) # (B,H,L,dk) def forward(self, Q, K, V, mask=None): B = Q.shape[0] Q = self.split_heads(self.W_q(Q)) # (B,H,L,dk) K = self.split_heads(self.W_k(K)) V = self.split_heads(self.W_v(V)) out, weights = scaled_dot_product_attention(Q, K, V, mask) out = out.transpose(1, 2).contiguous().view(B, -1, self.h * self.dk) return self.W_o(out), weights # (B,L,d_model)

Использование PyTorch встроенного MHA

mha = nn.MultiheadAttention(embed_dim=512, num_heads=8, dropout=0.1, batch_first=True) x = torch.randn(4, 32, 512) # (batch, seq, d_model) # Self-attention out, attn_w = mha(x, x, x) # (4,32,512) # Cross-attention (декодер смотрит на энкодер) memory = torch.randn(4, 20, 512) out2, _ = mha(query=x, key=memory, value=memory) # С causal mask causal = torch.triu(torch.ones(32,32), diagonal=1).bool() out3, _ = mha(x, x, x, attn_mask=causal)

1. Degradation Problem

До ResNet: добавление слоёв ухудшало train loss. Это не overfitting — деградация оптимизации.

He et al. 2015: 56-слойная plain сеть хуже 20-слойной на CIFAR-10 — и train, и test. Деградация!
Проблема.Если оптимум — identity H(x)=x, слою надо W→0. Трудно при случайной инициализации.
Решение He.Учим остаток F(x)=H(x)−x, тогда H(x)=F(x)+x. Identity → F(x)=0 — легко.
Ансамбль (Veit 2016).ResNet c N блоков = ансамбль из 2ᴺ путей. Удаление блока ≈ не меняет ответ.

3. Градиент через skip connection

Стек N блоков: x_L = x_l + Σ F(xᵢ). Backprop:

Backward BatchNorm — полный вывод

1. ∂L/∂γ = Σ_b ∂L/∂y_b · x̂_b    ∂L/∂β = Σ_b ∂L/∂y_b
2. ∂L/∂x̂_b = ∂L/∂y_b · γ
3. ∂L/∂σ² = Σ_b ∂L/∂x̂_b · (x_b−μ) · (−½)(σ²+ε)^(−3/2)
4. ∂L/∂μ = Σ_b ∂L/∂x̂_b · (−1/√(σ²+ε)) + ∂L/∂σ² · (−2/B)·Σ(x_b−μ)
5. ∂L/∂x_b = ∂L/∂x̂_b/√(σ²+ε) + ∂L/∂σ²·2(x_b−μ)/B + ∂L/∂μ/B
Градиент ∂L/∂x_b зависит от всех примеров батча через μ,σ² — неявная регуляризация.

7. PyTorch — реализация

Basic ResBlock

class BasicBlock(nn.Module): expansion = 1 def __init__(self, in_ch, out_ch, stride=1): super().__init__() self.conv1 = nn.Conv2d(in_ch, out_ch, 3, stride=stride, padding=1, bias=False) self.bn1 = nn.BatchNorm2d(out_ch) self.conv2 = nn.Conv2d(out_ch, out_ch, 3, padding=1, bias=False) self.bn2 = nn.BatchNorm2d(out_ch) self.skip = nn.Identity() if stride != 1 or in_ch != out_ch: self.skip = nn.Sequential( nn.Conv2d(in_ch, out_ch, 1, stride=stride, bias=False), nn.BatchNorm2d(out_ch)) def forward(self, x): out = F.relu(self.bn1(self.conv1(x))) out = self.bn2(self.conv2(out)) return F.relu(out + self.skip(x)) # ← skip!

Bottleneck

class Bottleneck(nn.Module): expansion = 4 def __init__(self, in_ch, planes, stride=1): super().__init__() self.cv1 = nn.Conv2d(in_ch, planes, 1, bias=False); self.bn1 = nn.BatchNorm2d(planes) self.cv2 = nn.Conv2d(planes, planes, 3, stride=stride, padding=1, bias=False); self.bn2 = nn.BatchNorm2d(planes) self.cv3 = nn.Conv2d(planes, planes*4, 1, bias=False); self.bn3 = nn.BatchNorm2d(planes*4) self.skip = nn.Identity() if stride != 1 or in_ch != planes*4: self.skip = nn.Sequential(nn.Conv2d(in_ch, planes*4, 1, stride=stride, bias=False), nn.BatchNorm2d(planes*4)) def forward(self, x): out = F.relu(self.bn1(self.cv1(x))) out = F.relu(self.bn2(self.cv2(out))) return F.relu(self.bn3(self.cv3(out)) + self.skip(x))

ManualBatchNorm (PyTorch)

class ManualBN(nn.Module): def __init__(self, C, eps=1e-5, momentum=0.1): super().__init__() self.gamma = nn.Parameter(torch.ones(C)) self.beta = nn.Parameter(torch.zeros(C)) self.eps = eps; self.m = momentum self.register_buffer('rm', torch.zeros(C)) self.register_buffer('rv', torch.ones(C)) def forward(self, x): # x: (B, C, H, W) if self.training: mu = x.mean([0,2,3]); var = x.var([0,2,3], unbiased=False) self.rm.mul_(1-self.m).add_(mu.detach()*self.m) self.rv.mul_(1-self.m).add_(var.detach()*self.m) else: mu, var = self.rm, self.rv x = (x - mu[:,None,None]) / (var[:,None,None] + self.eps).sqrt() return self.gamma[:,None,None] * x + self.beta[:,None,None]

BatchNorm NumPy forward + backward

def bn_forward(x, gamma, beta, eps=1e-5): # x: (B, C) mu = x.mean(0); var = x.var(0) xhat = (x - mu) / np.sqrt(var + eps) return gamma * xhat + beta, (x, xhat, mu, var, gamma, eps) def bn_backward(dy, cache): x, xhat, mu, var, gamma, eps = cache; B = x.shape[0] dgamma = (dy * xhat).sum(0); dbeta = dy.sum(0) dxhat = dy * gamma dvar = (dxhat*(x-mu)*(-0.5)*(var+eps)**(-1.5)).sum(0) dmu = (dxhat/(-np.sqrt(var+eps))).sum(0) + dvar*(-2*(x-mu)/B).sum(0) dx = dxhat/np.sqrt(var+eps) + dvar*2*(x-mu)/B + dmu/B return dx, dgamma, dbeta

GroupNorm и RMSNorm

gn = nn.GroupNorm(num_groups=32, num_channels=256) class RMSNorm(nn.Module): def __init__(self, d, eps=1e-8): super().__init__() self.gamma = nn.Parameter(torch.ones(d)); self.eps = eps def forward(self, x): rms = x.pow(2).mean(-1, keepdim=True).add(self.eps).sqrt() return x / rms * self.gamma

Как закрепить тему: ResNet и Normalization

Этот блок связывает теорию с практическим решением: где применять идею, какие ошибки проверять и какие ручки менять в эксперименте.

Когда применять

  • Используй тему «ResNet и Normalization», когда выбираешь inductive bias: локальность, последовательность, attention, residual flow или нормализацию.
  • Архитектура должна соответствовать структуре данных: изображение, текст, последовательность и табличные признаки требуют разных предположений.
  • Перед усложнением архитектуры проверь, что bottleneck действительно в модели, а не в данных, loss или обучении.

Типичные ошибки

  • Копировать архитектуру без учёта размера данных, бюджета памяти и целевой метрики.
  • Сравнивать модели с разным compute budget и делать вывод, что одна архитектура всегда лучше.
  • Игнорировать shape flow: stride, padding, heads, channels и sequence length напрямую определяют память и скорость.

Что тюнить

  • Настраивай depth, width, normalization, dropout, kernel/head sizes и residual structure.
  • Следи за memory footprint: иногда ограничение sequence length или feature map важнее числа параметров.
  • Для Transformer-подобных моделей отдельно проверяй attention mask, positional encoding и scaling.

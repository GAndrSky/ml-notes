1. Degradation Problem

Теория глубже

Residual block учит поправку, а не всю функцию заново

Если оптимальное преобразование близко к identity, plain network всё равно должна выучить identity через несколько нелинейных слоёв. ResNet делает identity route явным, а блок учит только residual correction F(x).

Это снижает сложность оптимизации: блоку легче научиться маленькой поправке, чем полностью реконструировать вход. Поэтому глубина становится полезной, а не только опасной.

  • Skip path переносит исходный сигнал.
  • Residual branch добавляет поправку.
  • Если поправка не нужна, сеть может приблизиться к identity.

До ResNet: добавление слоёв ухудшало train loss. Это не overfitting — деградация оптимизации.

He et al. 2015: 56-слойная plain сеть хуже 20-слойной на CIFAR-10 — и train, и test. Деградация!
Проблема.Если оптимум — identity H(x)=x, слою надо W→0. Трудно при случайной инициализации.
Решение He.Учим остаток F(x)=H(x)−x, тогда H(x)=F(x)+x. Identity → F(x)=0 — легко.
Ансамбль (Veit 2016).ResNet c N блоков = ансамбль из 2ᴺ путей. Удаление блока ≈ не меняет ответ.

2. Residual Block — математика

Basic Block (ResNet-18/34)

y = F(x) + x F(x) = W₂ · σ(BN(W₁·x)) ← 2× Conv 3×3 y = σ(BN(F(x)) + x) ← ReLU после Add Параметров: 2·9·C² = 18C² Projection shortcut (когда размеры ≠): skip(x) = BN(Conv1×1(x, stride=s))
Что означают символы
yвыход блокаy = σ(BN(F(x)) + x) — сумма residual преобразования и оригинального входа, затем ReLU. Блок учит поправку, а не полное преобразование.
xвход / skip connectionисходный сигнал, проходящий по shortcut пути напрямую. Если размерности совпадают — identity; иначе — projection: BN(Conv1×1(x, stride=s)).
F(x)residual функцияW₂·σ(BN(W₁·x)) — два свёрточных слоя 3×3. Блок учит только «разницу» между выходом и входом: F(x) = y − x. При инициализации F≈0 → y≈x.
W₁, W₂веса свёрточных слоёвдва Conv 3×3 с C каналами. Параметров: 2·9·C² = 18C². При C=256: 18·256²=1.18M параметров на один Basic Block.
σReLU (нелинейность)здесь σ — это ReLU, не sigmoid. Применяется после BN в середине блока и после сложения на выходе. Вводит нелинейность между слоями.
BNBatchNormнормализует по батчу и пространственным измерениям. Стабилизирует масштаб активаций между слоями. В projection shortcut: BN(Conv1×1) выравнивает размерность skip пути.
Интуиция: блок не учит преобразование целиком — он учит лишь поправку F(x) к уже хорошему приближению x. В начале обучения W≈0 → F≈0 → y≈x: блок начинает как identity и постепенно учится добавлять полезную информацию.
Аналогия: редактор правит черновик: основа (x) остаётся, добавляются только правки (F(x)). Легче исправить несколько слов, чем переписать абзац с нуля.
Числовой пример: x=[1.0, 0.5, −0.3]. F(x)=[0.1, −0.05, 0.2]. До ReLU: [1.1, 0.45, −0.1]. После ReLU: y=[1.1, 0.45, 0]. Блок внёс небольшую поправку, не «стёр» входной сигнал.

Bottleneck (ResNet-50/101/152)

W₁: 1×1 C→C/4 squeeze W₂: 3×3 C/4→C/4 process W₃: 1×1 C/4→C expand Параметров ≈ 3.5C²/4 (в ~3× меньше Basic)
Что означают символы
Cчисло каналовширина residual stream — число feature map каналов на входе и выходе блока. Bottleneck сохраняет C, но сжимает внутри до C/4.
C/4bottleneck widthвнутренняя ширина после 1×1 squeeze. При C=256: C/4=64. Именно здесь происходит дорогая 3×3 свёртка — в пространстве вчетверо меньшего числа каналов.
W₁, W₂, W₃три свёрточных слояW₁: 1×1 (C→C/4, squeeze), W₂: 3×3 (C/4→C/4, spatial), W₃: 1×1 (C/4→C, expand). Параметры: 1·C·C/4 + 9·(C/4)² + 1·C/4·C = C²/4+9C²/16+C²/4 ≈ 3.5C²/4.
3.5C²/4число параметров Bottleneckvs Basic Block: 18C². При C=256: Bottleneck≈57K, Basic≈1.18M — в ~20× меньше! Это позволяет строить ResNet-50/101/152 без взрывного роста параметров.
Интуиция: squeeze-expand структура выносит дорогую 3×3 свёртку в узкое пространство каналов. 1×1 слои — это channel mixing без пространственной обработки; они дёшевы, но позволяют контролировать ширину для 3×3.
Аналогия: перед решением сложной задачи сжать условие до ключевых переменных, решить в компактной записи, затем развернуть ответ обратно.
Числовой пример: C=256. Basic Block: 2×9×256²=1.18M. Bottleneck: W₁=256×64=16K, W₂=9×64²=37K, W₃=64×256=16K → итого ≈69K — в 17× меньше при той же глубине residual преобразования.

Pre-activation (He 2016)

Post-act 2015: y = σ(BN(F(x)+x)) Pre-act 2016: y = x + F(BN(σ(x))) Skip path чист → градиент без BN и ReLU
Что означают символы
yвыход блокаPost-act (2015): y=σ(BN(F(x)+x)) — ReLU после сложения. Pre-act (2016): y=x+F(BN(σ(x))) — ReLU и BN перед свёрткой. Skip path в Pre-act проходит без изменений.
xвход / skip pathв Pre-act: x проходит через + без BN и ReLU — «чистый» gradient highway. Градиент ∂L/∂x проходит без умножения на σ′(BN(...)) — нет риска затухания от насыщения ReLU на skip пути.
F(BN(σ(x)))residual с Pre-LNBN и ReLU применяются ДО свёртки, а не после. Это аналог Pre-LN в Transformer. Градиент через F тоже стабилизируется: BN нормирует входы свёртки, σ′ не насыщает их на skip.
σReLUнелинейность (ReLU, не sigmoid). В Post-act применяется после сложения x+F(x) — значит градиент skip пути тоже проходит через σ′. В Pre-act skip чист — σ применяется только внутри F.
Интуиция: в Post-act ReLU «зажимает» и skip путь — если (F(x)+x) < 0, градиент через skip тоже обнуляется. Pre-act убирает ReLU с skip пути, давая градиенту чистую дорогу от loss до ранних слоёв.
Аналогия: Post-act — турникет стоит в конце коридора и блокирует всех; Pre-act — турникет только на боковом пути, главный проход всегда открыт.
Числовой пример: F(x)+x = −0.1 (отрицательное). Post-act: σ(−0.1)=0, grad=0 — skip заблокирован. Pre-act: y=x+F(...)= x + σ(BN(x'))*W — skip x всегда добавляется, даже если F отрицательно.
Зачем Pre-act лучше: при Post-act добавление проходит через BN+ReLU — skip не чистый. Pre-act: residual stream = чистая сумма слагаемых.
Теория глубже

Bottleneck экономит compute через 1x1 channel projection

Bottleneck block сначала сжимает channels через 1x1 convolution, затем делает 3x3 convolution в меньшем пространстве, а потом расширяет channels обратно. Это снижает стоимость spatial convolution.

1x1 convolution не видит соседние пиксели, но эффективно смешивает каналы. Поэтому bottleneck разделяет channel mixing и spatial processing, похожим образом к идее эффективных CNN-блоков.

  • 1x1 reduce уменьшает channel dimension.
  • 3x3 работает в сжатом пространстве.
  • 1x1 expand возвращает нужную ширину residual stream.

Skip connection создаёт прямой путь для градиента

В residual block градиент может пройти через branch F(x), но также имеет identity path. Это уменьшает риск, что все градиенты будут проходить через длинную цепочку нестабильных Jacobian.

Именно поэтому residual connections стали базовым элементом не только CNN, но и Transformer, diffusion U-Net и многих современных архитектур.

  • Identity path стабилизирует backward.
  • Residual branch может учить сложную поправку.
  • Projection shortcut нужен, когда меняется размерность.

3. Градиент через skip connection

Стек N блоков: x_L = x_l + Σ F(xᵢ). Backprop:

∂L/∂x_l = ∂L/∂x_L · (1 + ∂ΣF/∂x_l) Слагаемое «1» = прямой вклад skip. Даже если ∂F/∂x → 0, градиент ≥ ∂L/∂x_L. Plain: ∂L/∂x_l = ∂L/∂x_L · ΠWᵢ·σ′ᵢ → затухание
Что означают символы
∂L/∂x_lградиент на слое lто, что нужно вычислить при backprop. Для ResNet: ∂L/∂x_l = ∂L/∂x_L · (1 + ∂ΣF/∂x_l) — слагаемое «1» от skip пути всегда присутствует.
∂L/∂x_Lградиент от верхнего слоя Lприходит с более глубокого слоя L. Умножается на (1 + ∂ΣF/∂x_l) — слагаемое 1 гарантирует, что даже при ∂F/∂x→0 градиент не исчезает.
1 + ∂ΣF/∂x_lполный Якобиан с skip«1» — вклад skip connection. ∂ΣF/∂x_l — вклад residual ветки. Даже если ∂F→0 (блок не активен), градиент ≥ ∂L/∂x_L — без затухания от этого слоя.
ΠWᵢ·σ′ᵢпроизведение Якобианов (Plain)в сети без skip градиент — это произведение весов и производных активаций по всем N слоям. При ‖W‖·σ′<1: произведение → 0 (затухание); при >1: → ∞ (взрыв). ResNet убирает этот риск для skip пути.
Интуиция: skip connection добавляет «1» к Якобиану. Это значит, что градиент от любого слоя доходит до любого раннего слоя хотя бы через skip путь — без умножения на цепочку матриц. ResNet из 152 слоёв обучается, а Plain-152 нет.
Аналогия: лифт в небоскрёбе: можно идти по лестнице (через все слои, теряя силы) или сесть в лифт (skip) и доехать напрямую.
Числовой пример: N=5, ‖W‖·σ′=0.7. Plain: 0.7⁵≈0.168 (теряем 83% градиента). ResNet: ≈1+∂F/∂x ≈ 1.05 — градиент почти не затухает через skip, даже если ∂F мало.
Пример N=5, ‖W‖·σ′=0.7. Plain: 0.7⁵=0.168. ResNet: ≈1.05.
Аналогия с LSTM: ∂cₜ/∂cₜ₋₁ = fₜ — тот же gradient highway без матричных умножений.

DenseNet

x_l = H([x₀,x₁,…,x_{l-1}]) Concat вместо Add
Что означают символы
x_lвыход слоя lв DenseNet — функция от конкатенации ВСЕХ предыдущих слоёв [x₀,…,x_{l-1}], а не только x_{l-1}. Каждый слой получает градиенты напрямую от всех последующих.
H([x₀,…,x_{l-1}])composite функция от конкатенацииH — это BN+ReLU+Conv, применённые к конкатенации feature maps всех предыдущих слоёв. Конкатенация (не сложение) сохраняет все предыдущие признаки в явном виде.
Concat vs Addключевое отличие от ResNetResNet: x_l = x_{l-1} + F(x_{l-1}) — сложение «стирает» предыдущее в один вектор. DenseNet: [x₀,…,x_{l-1}] — все слои видны явно. Память: O(L²) vs O(L), но recognition улучшается.
Интуиция: DenseNet — extreme version of residual: каждый слой видит все предыдущие как прямые входы. Это максимизирует gradient flow и feature reuse, но резко растёт ширина конкатенации с глубиной.
Аналогия: группа студентов, где каждый новый читает все предыдущие конспекты, а не только последний. ResNet — читает только вчерашний и добавляет свои заметки поверх.
Числовой пример: growth rate k=32 (каждый слой добавляет 32 канала). Слой 5 получает [x₀,x₁,x₂,x₃,x₄] с шириной 4×32=128 каналов (плюс начальные). Слой 10: 9×32=288 каналов на входе — растёт линейно.

Transformer

x = x + MHA(LN(x)) x = x + FFN(LN(x)) — тот же residual, Pre-LN
Что означают символы
xresidual streamвектор состояния токена, который проходит через все слои как «магистраль». Каждый sublayer добавляет к нему поправку — точно как ResNet добавляет F(x) к x.
x + MHA(LN(x))residual после attentionструктурный аналог ResNet Basic Block: x — skip path, MHA(LN(x)) — residual branch (F(x)). LN перед sublayer — это Pre-LN, аналог Pre-activation ResNet (He 2016).
LNLayerNorm (Pre-LN)применяется ДО sublayer, а не после. Аналог Pre-act ResNet: нормировка на входе F, skip чист. Стабилизирует обучение глубоких Transformer так же, как Pre-act стабилизировал ResNet-1000+.
Интуиция: Transformer позаимствовал residual connection из ResNet — и по той же причине: gradient highway через сотни слоёв. Skip path в Transformer (x +) позволяет обучать модели с 96+ слоями (GPT-3) без взрыва/затухания градиентов.
Аналогия: одна и та же инженерная идея — «обходной путь» — используется и в CNN (ResNet), и в Transformer. Разница только в том, что делает F(x): свёртка vs attention+FFN.
Числовой пример: GPT-3: 96 слоёв. Gradient через skip проходит 96 сложений (+) вместо умножений. Даже если каждый sublayer вносит ≈0 (в начале обучения), signal течёт от loss до embeddings без затухания.
Теория глубже

Выбор normalization зависит от оси статистики и режима inference

BatchNorm использует batch statistics во время training и running statistics во время eval. Если batch маленький или distribution inference отличается, эти статистики могут стать источником ошибки.

LayerNorm и GroupNorm не зависят от batch size таким же образом. Поэтому LN доминирует в Transformer, GN полезен при маленьких batch в vision, а BN остаётся сильным default для классических CNN с нормальным batch size.

  • BatchNorm: хорошо для CNN и больших batch.
  • LayerNorm: хорошо для sequence models и Transformer.
  • GroupNorm: устойчивее BatchNorm при маленьких batch.

5. Normalization — единая формула

x̂ = (x − μ) / √(σ² + ε) y = γ · x̂ + β Разница — только ОСИ для μ и σ²
Что означают символы
xвходной сигналисходное значение до нормировки. Один из элементов тензора активаций. Формула одинакова для BN/LN/IN/GN — разница только в том, по каким элементам считать μ и σ².
μсреднее по оси нормировкивычитается для центрирования. Ось зависит от метода: BN — по (B,H,W) для каждого канала c; LN — по (C,H,W) для каждого примера b; IN — по (H,W) для каждого (b,c).
σ²дисперсия по той же осимера разброса. x̂ = (x−μ)/√(σ²+ε) имеет нулевое среднее и единичную дисперсию по выбранной оси. Вычисляется по той же группе элементов, что и μ.
εчисловая стабильность~1e-5 в знаменателе √(σ²+ε). Предотвращает деление на ноль при σ²=0 (все элементы группы одинаковы).
γ, βобучаемые scale и shiftγ·x̂+β восстанавливает любой нужный масштаб и сдвиг. Инициализируются γ=1, β=0 (identity). Без них нормировка принудительно фиксировала бы N(0,1).
Интуиция: формула одна для всех методов нормализации — разница только в том, какие элементы группируются для вычисления μ и σ². BN объединяет по батчу, LN — по признакам, IN — по пространству, GN — по группам каналов.
Аналогия: стандартизация оценок: нормировать класс целиком (BN), нормировать одного студента по всем предметам (LN) или нормировать по отдельному предмету каждого студента (IN).
Числовой пример: x=[2,4,6,8]. μ=5, σ²=5. x̂≈[−1.34,−0.45,0.45,1.34]. γ=2, β=1: y=[−1.68, 0.10, 1.90, 3.68]. Если γ=1,β=0: выход центрирован с дисперсией 1.

Batch Norm (Ioffe & Szegedy, 2015)

μ_c = 1/(B·H·W) · Σ_{b,h,w} x_{b,c,h,w} ← per channel Train: batch stats Test: running EMA (momentum=0.1)
Что означают символы
μ_cсреднее по каналу c1/(B·H·W) · Σ x_{b,c,h,w} — усредняем по всем примерам в батче и по пространству (H×W) для каждого канала отдельно. Один μ_c на каждый из C каналов.
B, H, Wразмеры батча и пространстваB — batch size, H×W — пространственные размеры feature map. Нормировка по B·H·W элементам на канал. При B=1 (inference одного примера) статистика ненадёжна — BN нестабилен.
running EMAскользящая средняя для inferenceпри training: μ и σ² вычисляются по батчу. При inference: используется EMA, накопленная за training: μ_run ← 0.9·μ_run + 0.1·μ_batch. model.eval() переключает на running stats.
momentum=0.1скорость обновления EMAвес нового батча при обновлении running stats. 0.1 означает: 10% из нового батча, 90% из истории. Малый momentum → медленнее адаптируется, зато стабильнее при inference.
Интуиция: BatchNorm нормирует каждый канал по всему батчу — это делает активации стабильными и ускоряет обучение. Но создаёт train/test разрыв: на training μ из батча, на test — из EMA. При маленьком батче (B<8) статистика шумная → BN работает хуже.
Аналогия: нормировать оценки по классу: у каждого предмета (канала) своя средняя по всем студентам (батчу) и пространству. На финальном экзамене (inference) используется историческое среднее, а не текущей группы.
Числовой пример: канал c=3, B=4, H=W=2 → 4×2×2=16 элементов. x_{b,3,h,w}=[1,2,3,4,5,6,7,8,2,3,4,5,6,7,8,9]. μ_3=5.0, σ²_3=5.25. x̂=нормированные. После γ=1,β=0: выход центрирован.

Layer Norm (Ba et al., 2016)

μ_b = 1/(C·H·W) · Σ_{c,h,w} x_{b,c,h,w} ← per sample Train ≡ Test. Стандарт Transformer/NLP.
Что означают символы
μ_bсреднее по примеру b1/(C·H·W) · Σ_{c,h,w} x_{b,c,h,w} — усредняем по всем каналам и пространству одного примера. Один μ_b на каждый пример в батче.
C, H, Wканалы и пространствонормировка по C·H·W элементам одного примера. В NLP: C — число признаков токена, H=W=1 → нормировка по всем C признакам токена. Стандарт Transformer.
Train ≡ Testодинаковое поведениеLN не использует статистику батча — μ и σ² вычисляются по признакам одного примера. Поэтому нет разрыва между training и inference, нет running stats, нет model.train()/eval() влияния.
Интуиция: LN нормирует каждый пример независимо по его признакам. Это делает LN идеальным для sequence models: длина последовательности и batch size не влияют на статистику. Именно поэтому LN доминирует в Transformer.
Аналогия: нормировать одного студента по всем его оценкам (а не по классу). Каждый студент оценивается относительно себя — независимо от других.
Числовой пример: токен с признаками x=[1,2,3,4,5,6], C=6. μ=3.5, σ²=2.92. x̂≈[−1.46,−0.88,−0.29,0.29,0.88,1.46]. Это одно и то же при B=1 и B=32 — LN не зависит от батча.

Instance Norm (Ulyanov 2016)

μ_{b,c} = 1/(H·W) · Σ_{h,w} x_{b,c,h,w} ← per (B,C) Style transfer — убирает стиль, держит структуру
Что означают символы
μ_{b,c}среднее по (b, c) — per-example-per-channel1/(H·W) · Σ_{h,w} x_{b,c,h,w} — усредняем только по пространству H×W для каждой пары (пример b, канал c). Самый узкий масштаб нормировки.
1/(H·W)коэффициент усреднения по пространствуДелит сумму пикселей H·W на число позиций. Статистика считается только по пространству — без батча, без других каналов.
Σ_{h,w}сумма по всем пространственным позициямСуммирует пиксели по высоте h (0..H-1) и ширине w (0..W-1). Для канала c примера b — ровно H·W значений.
H, Wпространственные размеры feature mapВысота и ширина карты признаков. Нормировка только по H·W пикселям одного канала одного примера — независима для каждого (b,c).
Style transferключевое применение Instance Normμ_{b,c} и σ_{b,c} кодируют «стиль» изображения (яркость, контраст канала). Замена этих статистик на статистику стилевого изображения переносит его стиль при сохранении структуры контента.
Интуиция: IN нормирует каждый канал каждого изображения по его собственному пространственному среднему, «стирая» стиль. Для классификации плохо (теряем информацию о яркости и контрасте), для style transfer — идеально.
Аналогия: перевести фото в оттенки серого и выровнять гистограмму — убрать «стиль» освещения, оставив только форму объектов.
Числовой пример: канал c=0 картинки b=0: x=[100,120,80,110], H·W=4. μ=102.5. x̂≈[−0.19,1.32,−1.70,0.57]. Заменяем μ,σ на статистику стилевого изображения → пиксели принимают его «тон».

Group Norm (Wu & He, 2018)

Делим C на G групп, нормируем внутри: G=1 → LayerNorm G=C → InstanceNorm G=32 стандарт, не зависит от B → detection
Что означают символы
Cчисло каналовОбщее количество каналов признаков, которые нужно нормализовать.
Gчисло группКаналы делятся на G равных групп. G=32 — стандарт. G=1 даёт LayerNorm; G=C даёт InstanceNorm.
C/Gканалов в группеНормализация вычисляется отдельно внутри каждой группы из C/G каналов. Пример: C=256, G=32 → 8 каналов на группу.
γ, βlearned scale/shiftОбучаемые параметры (по C штук), восстанавливающие нужный масштаб после нормализации.
Интуиция: GroupNorm нормирует по пространству H×W и подгруппе каналов — не зависит от batch size, поэтому стабильно при B=1.
Аналогия: вместо нормализации всех студентов класса (BN) нормализуем по небольшим группам — работает даже с одним учеником.
Числовой пример: C=256, G=32, H=W=7: статистики считаются по 8×7×7=392 значений — достаточно для стабильных оценок даже при B=1.

RMSNorm (Zhang & Sennrich, 2019)

RMS(x) = √(1/d · Σ xᵢ²) y = γ · x / RMS(x) ← нет μ, нет β! ~7% быстрее LN. LLaMA, Mistral, Gemma.
Что означают символы
xвходной векторВектор активаций размерности d, который нужно нормализовать.
RMS(x)среднеквадратичное значение√(1/d · Σ xᵢ²) — корень из среднего квадрата элементов. Нет вычитания среднего (нет μ).
dразмерностьЧисло элементов в x. Пример: d=4096 в LLaMA-3.
γlearned scaleОбучаемый вектор масштаба размером d. Только γ — β отсутствует (нет сдвига).
Интуиция: RMSNorm убирает масштаб (делит на RMS), но не центрирует (нет вычитания μ) и не сдвигает (нет β) — меньше операций, та же стабильность.
Аналогия: LayerNorm — вычесть среднее и поделить на std; RMSNorm — просто поделить на «размер» вектора.
Числовой пример: x=[1,2,−1,2], d=4 → RMS=√((1+4+1+4)/4)=√2.5≈1.58. y=γ·x/1.58 ≈ γ·[0.63, 1.27, −0.63, 1.27]. Никакого вычитания среднего.

Backward BatchNorm — полный вывод

1. ∂L/∂γ = Σ_b ∂L/∂y_b · x̂_b    ∂L/∂β = Σ_b ∂L/∂y_b
2. ∂L/∂x̂_b = ∂L/∂y_b · γ
3. ∂L/∂σ² = Σ_b ∂L/∂x̂_b · (x_b−μ) · (−½)(σ²+ε)^(−3/2)
4. ∂L/∂μ = Σ_b ∂L/∂x̂_b · (−1/√(σ²+ε)) + ∂L/∂σ² · (−2/B)·Σ(x_b−μ)
5. ∂L/∂x_b = ∂L/∂x̂_b/√(σ²+ε) + ∂L/∂σ²·2(x_b−μ)/B + ∂L/∂μ/B
Градиент ∂L/∂x_b зависит от всех примеров батча через μ,σ² — неявная регуляризация.

7. PyTorch — реализация

Basic ResBlock

class BasicBlock(nn.Module): expansion = 1 def __init__(self, in_ch, out_ch, stride=1): super().__init__() self.conv1 = nn.Conv2d(in_ch, out_ch, 3, stride=stride, padding=1, bias=False) self.bn1 = nn.BatchNorm2d(out_ch) self.conv2 = nn.Conv2d(out_ch, out_ch, 3, padding=1, bias=False) self.bn2 = nn.BatchNorm2d(out_ch) self.skip = nn.Identity() if stride != 1 or in_ch != out_ch: self.skip = nn.Sequential( nn.Conv2d(in_ch, out_ch, 1, stride=stride, bias=False), nn.BatchNorm2d(out_ch)) def forward(self, x): out = F.relu(self.bn1(self.conv1(x))) out = self.bn2(self.conv2(out)) return F.relu(out + self.skip(x)) # ← skip!

Bottleneck

class Bottleneck(nn.Module): expansion = 4 def __init__(self, in_ch, planes, stride=1): super().__init__() self.cv1 = nn.Conv2d(in_ch, planes, 1, bias=False); self.bn1 = nn.BatchNorm2d(planes) self.cv2 = nn.Conv2d(planes, planes, 3, stride=stride, padding=1, bias=False); self.bn2 = nn.BatchNorm2d(planes) self.cv3 = nn.Conv2d(planes, planes*4, 1, bias=False); self.bn3 = nn.BatchNorm2d(planes*4) self.skip = nn.Identity() if stride != 1 or in_ch != planes*4: self.skip = nn.Sequential(nn.Conv2d(in_ch, planes*4, 1, stride=stride, bias=False), nn.BatchNorm2d(planes*4)) def forward(self, x): out = F.relu(self.bn1(self.cv1(x))) out = F.relu(self.bn2(self.cv2(out))) return F.relu(self.bn3(self.cv3(out)) + self.skip(x))

ManualBatchNorm (PyTorch)

class ManualBN(nn.Module): def __init__(self, C, eps=1e-5, momentum=0.1): super().__init__() self.gamma = nn.Parameter(torch.ones(C)) self.beta = nn.Parameter(torch.zeros(C)) self.eps = eps; self.m = momentum self.register_buffer('rm', torch.zeros(C)) self.register_buffer('rv', torch.ones(C)) def forward(self, x): # x: (B, C, H, W) if self.training: mu = x.mean([0,2,3]); var = x.var([0,2,3], unbiased=False) self.rm.mul_(1-self.m).add_(mu.detach()*self.m) self.rv.mul_(1-self.m).add_(var.detach()*self.m) else: mu, var = self.rm, self.rv x = (x - mu[:,None,None]) / (var[:,None,None] + self.eps).sqrt() return self.gamma[:,None,None] * x + self.beta[:,None,None]

BatchNorm NumPy forward + backward

def bn_forward(x, gamma, beta, eps=1e-5): # x: (B, C) mu = x.mean(0); var = x.var(0) xhat = (x - mu) / np.sqrt(var + eps) return gamma * xhat + beta, (x, xhat, mu, var, gamma, eps) def bn_backward(dy, cache): x, xhat, mu, var, gamma, eps = cache; B = x.shape[0] dgamma = (dy * xhat).sum(0); dbeta = dy.sum(0) dxhat = dy * gamma dvar = (dxhat*(x-mu)*(-0.5)*(var+eps)**(-1.5)).sum(0) dmu = (dxhat/(-np.sqrt(var+eps))).sum(0) + dvar*(-2*(x-mu)/B).sum(0) dx = dxhat/np.sqrt(var+eps) + dvar*2*(x-mu)/B + dmu/B return dx, dgamma, dbeta

GroupNorm и RMSNorm

gn = nn.GroupNorm(num_groups=32, num_channels=256) class RMSNorm(nn.Module): def __init__(self, d, eps=1e-8): super().__init__() self.gamma = nn.Parameter(torch.ones(d)); self.eps = eps def forward(self, x): rms = x.pow(2).mean(-1, keepdim=True).add(self.eps).sqrt() return x / rms * self.gamma

Как закрепить тему: ResNet и Normalization

Этот блок связывает теорию с практическим решением: где применять идею, какие ошибки проверять и какие ручки менять в эксперименте.

Когда применять

  • Используй тему «ResNet и Normalization», когда выбираешь inductive bias: локальность, последовательность, attention, residual flow или нормализацию.
  • Архитектура должна соответствовать структуре данных: изображение, текст, последовательность и табличные признаки требуют разных предположений.
  • Перед усложнением архитектуры проверь, что bottleneck действительно в модели, а не в данных, loss или обучении.

Типичные ошибки

  • Копировать архитектуру без учёта размера данных, бюджета памяти и целевой метрики.
  • Сравнивать модели с разным compute budget и делать вывод, что одна архитектура всегда лучше.
  • Игнорировать shape flow: stride, padding, heads, channels и sequence length напрямую определяют память и скорость.

Что тюнить

  • Настраивай depth, width, normalization, dropout, kernel/head sizes и residual structure.
  • Следи за memory footprint: иногда ограничение sequence length или feature map важнее числа параметров.
  • Для Transformer-подобных моделей отдельно проверяй attention mask, positional encoding и scaling.

Упражнения: ResNet и Normalization

Проверка темы - понимать, как residual path и normalization помогают глубокой сети обучаться.

[Foundational] Residual formula

Что означает \(y=x+F(x)\)? Решение: блок учит поправку \(F(x)\), сохраняя прямой путь для \(x\).

[Foundational] BatchNorm vs LayerNorm

Главное отличие? Решение: BatchNorm использует статистики batch/channel, LayerNorm нормирует признаки внутри объекта/токена.

[Intermediate] Почему residual помогает градиенту?

Решение: gradient может пройти через identity path, не умножаясь на длинную цепочку нелинейностей.

[Intermediate] Train/eval BatchNorm

Почему BatchNorm имеет train и eval режимы? Решение: train использует batch statistics, eval - накопленные running statistics.

[Hard] Normalization placement

Почему placement norm до/после блока влияет на stability? Решение: оно меняет путь градиента и масштаб сигнала в residual branch.

Проверь себя

Вопросы для самопроверки

  1. Как residual блок решает проблему деградации в очень глубоких сетях?

    Residual блок: y = F(x) + x, где F — несколько conv-слоёв. Вместо аппроксимации желаемого mapping H(x) напрямую, сеть учит «остаток» F(x) = H(x) - x. Если слой бесполезен — проще обнулить F(x)≈0, чем научить y≈x через сложные преобразования.

  2. Почему Group Normalization предпочтительнее Batch Normalization при малом размере батча?

    BatchNorm вычисляет статистику по всему батчу — при batch_size=1–2 оценки μ и σ нестабильны и шумны. GroupNorm нормирует по группам каналов внутри одного примера, не зависит от batch size. В object detection и segmentation с малыми батчами GroupNorm стандартен.

  3. Что такое pre-activation ResNet (pre-norm) и почему он лучше оригинального?

    Оригинальный ResNet: Conv → BN → ReLU → Conv → BN, затем Add → ReLU. Pre-activation: BN → ReLU → Conv → BN → ReLU → Conv, затем Add. Pre-norm позволяет identity shortcut передавать неактивированный сигнал, что улучшает поток градиентов и даёт лучшую оптимизируемость.

  4. Зачем в ResNet-50 используется bottleneck block (1×1 → 3×3 → 1×1)?

    Bottleneck снижает вычислительную стоимость: 1×1 conv уменьшает число каналов (bottleneck), 3×3 conv работает в сжатом пространстве, 1×1 conv восстанавливает число каналов. Это позволяет построить более глубокую сеть (50 вместо 34 слоёв) при сопоставимой вычислительной стоимости.

5.9 Object Detection

Object detection answers two questions at once: what objects are present, and where each object is located.

TL;DR

Detection is classification plus localization. The model predicts bounding boxes, class probabilities, and confidence scores. The hard parts are matching predictions to ground truth, suppressing duplicate boxes, and evaluating localization quality with IoU and mAP.

Why this matters

Detection appears in autonomous driving, retail analytics, medical imaging, OCR pipelines, manufacturing inspection, moderation, sports analytics, and robotics. It is usually more production-sensitive than image classification because boxes must be spatially accurate, stable across frames, and fast enough for real-time inference.

Bounding boxes and IoU

A detector usually predicts boxes as coordinates. The standard quality measure for localization is Intersection over Union.

IoU(A,B) = area(intersection) / area(union)
What the symbols mean
Apredicted boxBox proposed by the model.
Bground-truth boxHuman-labeled correct object location.
A cap BoverlapArea where predicted and true boxes overlap.
A cup BunionTotal area covered by either box.
Intuition: IoU is high only when the predicted box covers the object and does not include too much extra area.
Analogy: two stickers on the same object should overlap almost perfectly.
Numeric example: if overlap is 40 pixels and union is 80 pixels, IoU = 0.5.

Anchors, YOLO, and one-stage detection

Anchor-based detectors start with predefined reference boxes. The network predicts offsets from anchors and class/objectness scores. YOLO-style models divide the image into prediction locations and output many candidate boxes in one forward pass.

b_x = sigma(t_x) + c_x, b_w = p_w exp(t_w)
What the symbols mean
t_x,t_wraw predictionsNetwork outputs before box decoding.
c_xcell coordinateGrid location where the box is predicted.
p_wanchor widthPrior width of the reference box.
sigmasigmoidConstrains center offset into a stable range.
Intuition: the detector does not invent boxes from scratch; it adjusts reference boxes.
Analogy: resize and move a template frame until it fits the object.

YOLO tradeoff: very fast and production-friendly, but dense candidate boxes require careful confidence thresholds and NMS.

Non-Maximum Suppression

NMS removes duplicate detections. Sort boxes by confidence, keep the strongest box, and suppress nearby boxes with high IoU.

keep box_i if IoU(box_i, selected_box) < threshold

Typical thresholds: confidence 0.25-0.5, NMS IoU 0.45-0.7. Lower NMS threshold removes more duplicates but may delete nearby objects.

DETR: detection as set prediction

DETR removes anchors and NMS. It uses transformer object queries and bipartite matching between predicted boxes and ground-truth objects.

loss = class_loss + lambda_box L1(box, box*) + lambda_iou GIoU(box, box*)

The key idea is that the model predicts a fixed-size set of objects, and Hungarian matching decides which prediction corresponds to which ground-truth object.

Metrics: mAP

Mean Average Precision summarizes precision-recall curves across classes and IoU thresholds. COCO-style mAP averages from IoU 0.50 to 0.95, which strongly rewards precise boxes.

Runnable code: IoU and NMS

# What it does: compute IoU and run greedy NMS
# Input shapes: boxes=(N,4) in xyxy format, scores=(N,)
# Output shape: kept indices=(K,)
import numpy as np

def iou(a, b):
    x1 = max(a[0], b[0]); y1 = max(a[1], b[1])
    x2 = min(a[2], b[2]); y2 = min(a[3], b[3])
    inter = max(0, x2 - x1) * max(0, y2 - y1)
    area_a = max(0, a[2]-a[0]) * max(0, a[3]-a[1])
    area_b = max(0, b[2]-b[0]) * max(0, b[3]-b[1])
    return inter / (area_a + area_b - inter + 1e-9)

def nms(boxes, scores, threshold=0.5):
    order = scores.argsort()[::-1]
    keep = []
    while order.size:
        i = order[0]
        keep.append(i)
        rest = order[1:]
        order = rest[np.array([iou(boxes[i], boxes[j]) < threshold for j in rest])]
    return keep

boxes = np.array([[0,0,10,10], [1,1,11,11], [30,30,40,40]], dtype=float)
scores = np.array([0.9, 0.8, 0.7])
print(nms(boxes, scores, threshold=0.5))
# Gotchas: use consistent box format; tune NMS threshold per dataset density.

Common mistakes

  • Reporting classification accuracy instead of detection metrics.
  • Training on boxes with inconsistent coordinate formats.
  • Using one confidence threshold for all classes when class imbalance is strong.
  • Ignoring small-object performance and latency constraints.

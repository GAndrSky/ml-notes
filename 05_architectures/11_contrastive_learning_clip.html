5.11 Contrastive Learning and CLIP

Contrastive learning trains representations by pulling matching pairs together and pushing non-matching pairs apart.

TL;DR

CLIP learns a shared embedding space for images and text. Matching image-caption pairs get high cosine similarity; mismatched pairs get low similarity. This makes zero-shot classification possible by comparing an image to text prompts.

Why this matters

Contrastive learning is the backbone of modern retrieval, embedding models, multimodal systems, self-supervised visual pretraining, and CLIP-like zero-shot classifiers. It turns "labels" into relationships between examples.

InfoNCE loss

For one anchor, InfoNCE makes the positive example more similar than all negatives in the batch.

L_i = -log exp(s_i,i / tau) / sum_j exp(s_i,j / tau)
What the symbols mean
s_i,jsimilarity scoreUsually cosine similarity between anchor i and candidate j.
i,ipositive pairThe correct matching pair, such as an image and its caption.
jcandidatesAll possible matches in the batch: one positive and many negatives.
tautemperatureControls how sharp the softmax distribution is.
Intuition: the correct pair must win a classification problem against all other batch items.
Analogy: match a photo to the correct caption among many captions.
Numeric example: if the positive similarity is much larger than negatives, its softmax probability approaches 1 and loss approaches 0.

SimCLR: self-supervised contrastive vision

SimCLR creates two augmented views of the same image. These two views are positives. Other images in the batch are negatives. The model learns invariances: crop, color jitter, blur, and small geometric changes should not destroy identity.

  • Encoder: CNN or ViT maps image to representation.
  • Projection head: MLP maps representation to contrastive space.
  • Augmentations: define what information should be ignored.

CLIP architecture

CLIP uses two encoders: one for images and one for text. It trains on image-caption pairs and optimizes contrastive loss in both directions: image-to-text and text-to-image.

logits = normalize(image_emb) @ normalize(text_emb)^T / tau
What the symbols mean
image_embimage embeddingVector produced by image encoder.
text_embtext embeddingVector produced by text encoder.
@matrix productComputes all image-text similarities in the batch.
normalizeunit vectorsMakes dot product equal cosine similarity.
Intuition: CLIP aligns image and language vectors so matching concepts point in similar directions.
Analogy: put images and sentences on the same map, then search by nearest neighbor.

Zero-shot classification

To classify an image, encode prompts like "a photo of a cat", "a photo of a dog", "a photo of a car". Pick the text embedding most similar to the image embedding.

Prompt wording matters. CLIP is not only recognizing labels; it compares the image to natural-language descriptions.

Runnable code: contrastive logits

# What it does: compute CLIP-style contrastive logits
# Input shapes: image_emb=(B,d), text_emb=(B,d)
# Output shape: logits=(B,B)
import numpy as np

def normalize(x, eps=1e-9):
    return x / (np.linalg.norm(x, axis=1, keepdims=True) + eps)

image_emb = np.array([[1.0, 0.0], [0.2, 0.8]])
text_emb = np.array([[0.9, 0.1], [0.0, 1.0]])
tau = 0.07
logits = normalize(image_emb) @ normalize(text_emb).T / tau
print(logits)
# Gotchas: embeddings must be normalized; temperature strongly changes confidence.

Common mistakes

  • Thinking negatives are always true negatives. In large batches, false negatives can exist.
  • Ignoring augmentation design in self-supervised contrastive learning.
  • Assuming CLIP explanations are causal because text similarity looks intuitive.
  • Using cosine similarity without checking embedding normalization.

Connections

Contrastive learning connects ViT, RAG retrieval, embedding models, image search, and multimodal LLMs.

5.11 Contrastive Learning and CLIP

Contrastive learning trains representations by pulling matching pairs together and pushing non-matching pairs apart.

Почему важно

CLIP научил модель понимать связь между изображениями и текстом без явных меток. Contrastive learning — подход, извлекающий универсальные представления из пар «похожее–непохожее».

TL;DR

CLIP learns a shared embedding space for images and text. Matching image-caption pairs get high cosine similarity; mismatched pairs get low similarity. This makes zero-shot classification possible by comparing an image to text prompts.

Why this matters

Contrastive learning is the backbone of modern retrieval, embedding models, multimodal systems, self-supervised visual pretraining, and CLIP-like zero-shot classifiers. It turns "labels" into relationships between examples.

InfoNCE loss

For one anchor, InfoNCE makes the positive example more similar than all negatives in the batch.

L_i = -log exp(s_i,i / tau) / sum_j exp(s_i,j / tau)
What the symbols mean
s_i,jsimilarity scoreUsually cosine similarity between anchor i and candidate j.
i,ipositive pairThe correct matching pair, such as an image and its caption.
jcandidatesAll possible matches in the batch: one positive and many negatives.
tautemperatureControls how sharp the softmax distribution is.
Intuition: the correct pair must win a classification problem against all other batch items.
Analogy: match a photo to the correct caption among many captions.
Numeric example: if the positive similarity is much larger than negatives, its softmax probability approaches 1 and loss approaches 0.

SimCLR: self-supervised contrastive vision

SimCLR creates two augmented views of the same image. These two views are positives. Other images in the batch are negatives. The model learns invariances: crop, color jitter, blur, and small geometric changes should not destroy identity.

  • Encoder: CNN or ViT maps image to representation.
  • Projection head: MLP maps representation to contrastive space.
  • Augmentations: define what information should be ignored.

CLIP architecture

CLIP uses two encoders: one for images and one for text. It trains on image-caption pairs and optimizes contrastive loss in both directions: image-to-text and text-to-image.

logits = normalize(image_emb) @ normalize(text_emb)^T / tau
What the symbols mean
image_embimage embeddingVector produced by the image encoder (e.g. ViT). Shape: N×d, one row per image in the batch.
text_embtext embeddingVector produced by the text encoder (Transformer). Shape: N×d, one row per caption.
normalizeL2 normalizationDivides each vector by its norm so all embeddings lie on the unit sphere. Ensures dot product equals cosine similarity.
^TtransposeTurns text_emb from N×d into d×N. After @, result is N×N: entry (i,j) is the cosine similarity between image i and text j.
@matrix productComputes all N² image-text similarities at once. The diagonal holds the correct (matched) pairs; off-diagonal are negatives.
tautemperatureLearnable scalar (typically ~0.07). Low tau sharpens the softmax — correct pairs must be clearly more similar than all negatives.
Intuition: the result is an N×N logit matrix. Training pushes diagonal entries (matched pairs) high and off-diagonal (mismatches) low — simultaneously in both image-to-text and text-to-image directions.
Analogy: put images and sentences on the same map so matching pairs land near each other; all others are pushed apart.
Numeric example: batch N=4. image_emb @ text_emb^T gives a 4×4 matrix. Divide by tau=0.07 → cross-entropy on rows (image→text) and columns (text→image). Loss is 0 only when each image ranks its caption first across all N candidates.

Zero-shot classification

To classify an image, encode prompts like "a photo of a cat", "a photo of a dog", "a photo of a car". Pick the text embedding most similar to the image embedding.

Prompt wording matters. CLIP is not only recognizing labels; it compares the image to natural-language descriptions.

Runnable code: contrastive logits

# What it does: compute CLIP-style contrastive logits
# Input shapes: image_emb=(B,d), text_emb=(B,d)
# Output shape: logits=(B,B)
import numpy as np

def normalize(x, eps=1e-9):
    return x / (np.linalg.norm(x, axis=1, keepdims=True) + eps)

image_emb = np.array([[1.0, 0.0], [0.2, 0.8]])
text_emb = np.array([[0.9, 0.1], [0.0, 1.0]])
tau = 0.07
logits = normalize(image_emb) @ normalize(text_emb).T / tau
print(logits)
# Gotchas: embeddings must be normalized; temperature strongly changes confidence.

Common mistakes

  • Thinking negatives are always true negatives. In large batches, false negatives can exist.
  • Ignoring augmentation design in self-supervised contrastive learning.
  • Assuming CLIP explanations are causal because text similarity looks intuitive.
  • Using cosine similarity without checking embedding normalization.

Проверь себя

Вопросы для самопроверки

  1. Что такое contrastive learning и в чём его основная идея?

    Contrastive learning обучает представления, делая «похожие» примеры (позитивные пары) близкими в embedding space, а «разные» (негативные) — далёкими. Не нужны метки — позитивные пары создаются через augmentations (SimCLR) или разные модальности (CLIP: изображение + текст).

  2. Как CLIP обучается и что позволяет делать в inference?

    CLIP обучается на 400M пар (изображение, текст) из интернета. Цель: максимизировать dot-product между эмбеддингами совпадающих пар и минимизировать для несовпадающих (InfoNCE loss). При inference: вычислить сходство изображения с текстовыми описаниями классов — zero-shot классификация.

  3. Что такое zero-shot transfer в контексте CLIP?

    Zero-shot transfer: модель применяется к задаче, для которой не было примеров при обучении. Для CLIP: описываем классы текстом («a photo of a dog»), получаем текстовые эмбеддинги, сравниваем с эмбеддингом тестового изображения. CLIP достигает точности ResNet-50 на ImageNet без единого обучающего примера.

  4. Почему InfoNCE loss работает хуже с маленьким батчем?

    InfoNCE рассматривает все другие примеры в батче как негативные. Маленький батч → мало негативов → задача слишком проста, representation learning неэффективен. SimCLR требует batch_size=4096+; MoCo решает это через memory bank с тысячами негативов, независимо от батча.

Как закрепить тему: Contrastive Learning & CLIP

Блок связывает теорию с практикой: где применять идею, какие ошибки проверять и как развиваться дальше.

Когда применять

  • Используй CLIP embeddings для zero-shot классификации и image search.
  • Contrastive pretraining с аугментациями — мощный подход для self-supervised CV.
  • CLIP features хорошо работают как frozen backbone для downstream задач.

Типичные ошибки

  • Не масштабировать батч при contrastive learning — маленький батч = мало негативов.
  • Использовать CLIP для задач вне distribution обучения без проверки zero-shot accuracy.
  • Забывать о temperature τ — критический гиперпараметр для InfoNCE loss.

Что тюнить / проверить

  • Температура τ в InfoNCE: слишком низкая → collapse, слишком высокая → медленная сходимость.
  • Batch size: больше = лучше, но дороже.
  • Augmentation strategy: сила аугментаций влияет на «сложность» positives.

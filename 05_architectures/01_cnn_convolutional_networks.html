1. Дискретная свёртка — математика

Теория глубже

CNN вшивает prior локальности и повторяемости паттернов

Свёртка предполагает, что важные признаки часто локальны: край, угол, текстура или маленький объект можно распознать по соседним пикселям. Второе предположение - один и тот же detector полезен в разных местах изображения.

Именно parameter sharing делает CNN эффективной. Один фильтр скользит по всей карте и ищет один паттерн везде, вместо того чтобы учить отдельный набор весов для каждой позиции.

  • Локальность уменьшает число возможных взаимодействий.
  • Shared weights уменьшают число параметров.
  • Translation equivariance означает: сдвиг входа сдвигает feature map.

Свёртка в DL — дискретная кросс-корреляция (по историческим причинам называется свёрткой):

(I ★ K)[i,j] = Σₘ Σₙ K[m,n] · I[i+m, j+n]
Что означают символы
Iвходное изображениеДвумерный (или многоканальный) массив пикселей. I[i+m, j+n] — пиксель в рецептивном окне.
Kядро (kernel)Обучаемый фильтр размером k×k. K[m,n] — вес при смещении (m,n) внутри окна.
i, jпозиция выходаКоординаты ячейки выходной feature map, в которую записывается результат.
m, nсмещение внутри ядраПробегают от 0 до k−1. Задают, какой пиксель входа участвует в этой позиции ядра.
Σₘ Σₙсуммирование по окнуDot product ядра с соответствующим патчем входа — k² умножений и сложений.
★кросс-корреляцияВ DL так называют "свёртку" (ядро не переворачивается). Истинная свёртка → K[-m,-n], но разница несущественна при обучении.
Интуиция: Ядро скользит по изображению и в каждой позиции вычисляет скалярное произведение с локальным патчем — один "ответ" детектора на каждое место.
Аналогия: как трафарет: прикладываешь его к каждому месту картинки и записываешь, насколько хорошо совпало.
Числовой пример: K=[[1,0],[0,-1]], I-патч=[[2,3],[4,5]] → (I★K)[i,j] = 1·2+0·3+0·4+(−1)·5 = −3.

где I — входное изображение (H×W), K — ядро (kernel) размером k×k.

Размер выхода при valid padding:

H_out = ⌊(H_in + 2P - k) / S⌋ + 1
Что означают символы
H_inвысота входаЧисло строк входной feature map (или изображения).
PpaddingЧисло нулевых строк/столбцов, добавленных по краям. P=1 для 3×3 сохраняет размер при S=1.
kразмер ядраСторона квадратного ядра (например, k=3). Уменьшает выход на k−1 при valid padding.
Sstride (шаг)На сколько позиций ядро сдвигается за один шаг. S=2 вдвое уменьшает выход.
⌊ ⌋floor (округление вниз)Дробная часть отбрасывается — крайний патч, в который ядро не вмещается полностью, пропускается.
Интуиция: сколько раз ядро помещается в дополненный вход с данным шагом — это и есть размер выхода.
Аналогия: как нарезать батон кусками k см с шагом S — число кусков даётся этой же формулой.
Числовой пример: H_in=224, P=1, k=3, S=2 → ⌊(224+2−3)/2⌋+1 = ⌊223/2⌋+1 = 111+1 = 112.

P — padding, S — stride (шаг). Аналогично для W.

Истинная свёртка vs кросс-корреляция: при свёртке ядро поворачивается на 180°: (I * K)[i,j] = Σ K[-m,-n]·I[i+m,j+n]. В DL ядро обучается — поворот не имеет значения, поэтому используют кросс-корреляцию, называя её "свёрткой".

Подсчёт параметров

#params = k × k × C_in × C_out + C_out (bias)
Что означают символы
k × kпространственный размер ядраk² весов на каждую пару каналов. Для k=3: 9 весов; для k=1: 1 вес (pointwise conv).
C_inвходные каналыГлубина входной feature map. Каждый фильтр соединён со всеми C_in каналами.
C_outвыходные каналыЧисло фильтров. Каждый фильтр даёт одну выходную feature map. Bias: один скаляр на фильтр.
C_out (bias)смещениеОпциональный bias — C_out дополнительных параметров. Часто отключают вместе с BatchNorm (bias=False).
Интуиция: число параметров не зависит от H и W — один и тот же набор весов используется на всех позициях (parameter sharing).
Аналогия: один трафарет для всей картины — независимо от размера холста трафарет один и тот же.
Числовой пример: Conv2d(3→64, 3×3): 3×3×3×64 + 64 = 1728 + 64 = 1792 параметров.

Для сверхтонкого слоя Conv2d(3→64, 3×3): 3×3×3×64 + 64 = 1792 параметров

Теория глубже

Теоретическое RF больше, чем реально используемое RF

На бумаге receptive field показывает, какая область входа может повлиять на конкретную активацию. Но фактическое влияние внутри этой области обычно распределено неравномерно: центральные пиксели часто влияют сильнее, чем дальние.

Поэтому простое увеличение глубины не всегда означает, что модель хорошо использует глобальный контекст. Иногда нужны dilation, pooling, attention или архитектурные блоки, которые явно расширяют обмен информацией.

  • Малые ядра дают больше нелинейностей на ту же область.
  • Dilation расширяет RF без роста числа параметров.
  • Stride и pooling быстро уменьшают spatial resolution.

2. Рецептивное поле (Receptive Field)

Рецептивное поле нейрона — размер области входного изображения, влияющей на его активацию.

RFL = RFL-1 + (k-1) · Πi=1L-1 si
Что означают символы
RF_Lрецептивное поле слоя LРазмер области входного изображения, влияющей на одну активацию слоя L.
RF_{L-1}RF предыдущего слояРецептивное поле, накопленное до слоя L. RF растёт рекурсивно: RF_1 = k.
kразмер ядра слоя LКаждый нейрон слоя L смотрит на k соседей слоя L−1, расширяя RF на (k−1) · масштаб.
Π s_iпроизведение страйдовНакопленный масштаб: stride каждого предыдущего слоя умножает, насколько «далеко» сдвиг на одну позицию.
Интуиция: каждый новый слой расширяет RF на (k−1), умноженное на накопленный шаг всех предыдущих страйдов.
Аналогия: смотришь в бинокль через несколько линз — каждая увеличивает угол обзора пропорционально своему масштабу.
Числовой пример: 3 слоя Conv(k=3, s=1): RF=1+(3−1)·1+(3−1)·1+(3−1)·1=7. Один Conv(k=7, s=1): RF=7 — одинаково, но 3 слоя имеют нелинейности и меньше параметров.

k — kernel size слоя L, si — stride слоёв до L

Зачем важно? Для классификации объекта нейрон должен "видеть" весь объект. Глубокие сети с малыми ядрами достигают большого RF без избыточного числа параметров.

Сравнение: одно большое ядро vs стек малых

7×7conv, 1 слой
Параметры: 7·7·C·C = 49C²
RF = 7
3×3conv × 3 слоя
Параметры: 3·(3·3·C·C) = 27C²
RF = 1+2+2+2 = 7

Три слоя 3×3 дают тот же RF, что один 7×7, но параметров в 1.8× меньше и между слоями — нелинейности!

Теория глубже

Pooling покупает устойчивость ценой точной позиции

Max pooling оставляет сильнейший локальный сигнал и делает модель менее чувствительной к небольшому сдвигу объекта. Average pooling сглаживает область и больше похож на локальное усреднение энергии признака.

Потеря spatial detail может вредить segmentation, detection и задачам, где важна точная геометрия. Поэтому современные сети часто заменяют грубый pooling на strided convolution, skip connections или multi-scale features.

  • Max pooling хорошо ловит наличие признака.
  • Average pooling лучше сохраняет общий фон сигнала.
  • Global average pooling превращает feature maps в class-level summary.

3. Pooling — субдискретизация

Pooling уменьшает пространственный размер карты признаков, сохраняя важные признаки.

Max Pooling

y[i,j] = maxm,n ∈ window x[i·s+m, j·s+n]
Что означают символы
y[i,j]выходное значениеРезультат max pooling в позиции (i,j) выходной карты.
maxмаксимум по окнуВыбирает наибольшее значение среди k² элементов окна. Нет параметров — чисто выборочная операция.
x[i·s+m, j·s+n]элемент входаЗначение входной feature map в позиции, определяемой центром окна (i,j) и смещением (m,n) внутри него.
sstrideШаг скольжения окна. s=2 → выход вдвое меньше входа по каждому измерению.
m, nсмещение внутри окнаПробегают все позиции окна k×k. m,n ∈ {0,…,k−1}.
Интуиция: max pooling сохраняет только самый сильный сигнал в каждом окне — "есть ли признак здесь?" вместо "насколько он силён в среднем?".
Аналогия: из каждого квартала города выбираешь самый высокий дом — убираешь детали, сохраняешь доминантный сигнал.
Числовой пример: окно 2×2: [[1,3],[4,2]] → max=4. При s=2 карта 4×4 становится 2×2.

Average Pooling

y[i,j] = 1/(k²) · Σm,n ∈ window x[i·s+m, j·s+n]
Что означают символы
y[i,j]выходное значениеСреднее арифметическое k² элементов окна в позиции (i,j).
1/(k²)нормировкаДелим на число элементов в окне k×k. Для k=2: 1/4; для k=3: 1/9.
Σсумма по окнуСуммирует все k² значений входа в текущем окне.
x[i·s+m, j·s+n]элемент входаЗначение входной карты в позиции, смещённой на (m,n) от левого верхнего угла окна (i,j).
sstrideШаг скольжения окна. Обычно s=k (непересекающиеся окна) или s=1 (скользящее среднее).
Интуиция: average pooling усредняет сигнал по окну — сохраняет общий «уровень энергии» признака, но менее чувствителен к точной позиции максимума.
Аналогия: вместо лучшего студента в группе берём средний балл по группе.
Числовой пример: окно 2×2: [[1,3],[4,2]] → avg=(1+3+4+2)/4=2.5. MaxPool дал бы 4.

Global Average Pooling (GAP)

yc = 1/(H·W) · Σi,j xc[i,j]
Что означают символы
y_cвыход для канала cОдно скалярное значение на канал — среднее всей пространственной карты. Форма результата: (C,).
cиндекс каналаGAP работает независимо для каждого из C каналов. Пространство H×W «схлопывается» отдельно для каждого.
1/(H·W)нормировкаДелим на полное число позиций. GAP — это average pooling по всей карте сразу, без параметров.
Σ_{i,j}сумма по пространствуСуммирует все H×W значений канала c. Пример: H=W=7, C=512 → суммируем 49 чисел, получаем одно.
x_c[i,j]элемент feature mapЗначение в позиции (i,j) канала c входной карты (C, H, W).
Интуиция: GAP превращает «где есть признак» в «насколько признак присутствует вообще» — удаляет пространственную информацию, оставляя общий отклик канала.
Аналогия: вместо карты температур по городу — одно среднее значение температуры за день.
Числовой пример: карта (512, 7, 7): GAP → вектор (512,). Заменяет FC-слой с 512·7·7=25088 входами — нет параметров!

GAP сворачивает карту (C, H, W) → (C,). Заменяет FC-слои в ResNet, MobileNet — нет параметров!

Зачем pooling? (1) Трансляционная инвариантность к небольшим сдвигам. (2) Уменьшение объёма вычислений и памяти. (3) Неявная регуляризация.

Градиент через Max Pooling

∂L/∂x[i,j] = ∂L/∂y · 𝟙[x[i,j] = max]
Что означают символы
∂L/∂x[i,j]градиент по входуНасколько потеря чувствительна к элементу входа x[i,j]. Нужен для передачи градиента в предыдущий слой.
∂L/∂yвходящий градиентГрадиент от следующего слоя — насколько изменилась потеря при изменении выхода max pool.
𝟙[x[i,j] = max]индикаторная функцияРавна 1, если x[i,j] был максимумом в своём окне; иначе 0. Градиент проходит только через «победителя».
Интуиция: max pooling — кусочно-постоянная функция. Её производная равна 0 везде, кроме максимального элемента, где она равна 1.
Аналогия: только лучший студент в группе получает обратную связь; все остальные — нулевой градиент.
Числовой пример: окно [[1,3],[4,2]], max=4 на позиции (1,0). ∂L/∂y=0.5 → ∂L/∂x: только позиция (1,0) получает 0.5, остальные 0.

Градиент проходит только через максимальный элемент окна ("маршрутизация градиента").

⚠ Stride в conv vs Pooling: Современные архитектуры (ResNet, EfficientNet) часто заменяют MaxPool слоями Conv с stride=2 — это обучаемое прореживание с сохранением информации.

4. Equivariance и Invariance

Определения

Equivariant: f(T(x)) = T'(f(x))
Invariant: f(T(x)) = f(x)
Что означают символы
fфункция (слой/сеть)Отображение, которое проверяется на equivariance. Например, свёрточный слой или GAP.
Tпреобразование входаНапример, сдвиг изображения на δ пикселей. T(x) — преобразованный вход.
T'соответствующее преобразование выходаEquivariance: преобразование входа ведёт к предсказуемому преобразованию выхода. Для сдвига T'=T (feature map тоже сдвигается).
xвходное изображениеИсходный вход до преобразования.
Интуиция: equivariance = преобразование входа предсказуемо отражается в выходе; invariance = преобразование входа вообще не меняет выход.
Аналогия: equivariant — переставил мебель, карта квартиры тоже изменилась; invariant — как ни переставляй, площадь квартиры та же.
Числовой пример: свёртка equivariant к сдвигу: сдвинь кошку на 5 пикселей → feature map тоже сдвинется на 5. GAP invariant: среднее по всей карте не зависит от позиции объекта.
Equivariance (свёртка)

Сдвиг входа → сдвиг карты признаков. Одни и те же признаки обнаруживаются в любой позиции одним и тем же ядром.
Invariance (MaxPool, GAP)

Небольшой сдвиг объекта не меняет итоговую классификацию — pooling "поглощает" смещение.

Математическое доказательство equivariance

Шаг 1.Пусть T_δ — оператор сдвига: (T_δ I)[i,j] = I[i-δ, j]
Шаг 2.(T_δ I ★ K)[i,j] = Σ K[m,n]·I[i+m-δ, j+n] = (I★K)[i-δ, j]
Шаг 3.Т.е. f(T_δ I) = T_δ(f(I)) → свёртка equivariant к сдвигу ✓
Свёртка не equivariant к: ротациям, масштабированию, отражениям. Для этого: увеличение данных (data augmentation), группо-equivariant свёртки (G-CNN), или пространственный трансформер (STN).

Parameter Sharing — суть экономии

FC-слой (28²→128): 784 × 128 = 100352 параметров
Conv(1→1, 3×3) на том же входе: 9 параметров
Что означают символы
784 × 128параметры FC-слояПолносвязный слой: отдельный вес для каждой пары вход-выход. 28²=784 входных нейрона × 128 выходных = 100 352 параметра.
9параметры Conv(1→1, 3×3)Одно ядро 3×3 = 9 весов. Оно скользит по всем 784 позициям, используя одни и те же 9 чисел везде.
parameter sharingобщие весаКлючевое свойство свёртки: одно ядро применяется ко всем пространственным позициям. Число параметров не зависит от H и W.
Интуиция: FC видит все пиксели вместе и учит уникальные связи; Conv учит один детектор и применяет его везде — в 11 000× меньше параметров для этого примера.
Аналогия: FC — отдельный охранник у каждого кресла в кинотеатре; Conv — один охранник с фонариком, который обходит все ряды.
Числовой пример: 100352 / 9 ≈ 11150× экономия на параметрах при том же детекторе признаков.

Одно ядро сканирует все пространственные позиции — это и есть parameter sharing. Ядро учит детектор признака (например, "вертикальный край"), который работает везде в изображении.

5. Типичная CNN-архитектура

Классическая CNN — стек блоков "Conv → BN → ReLU → Pool", завершённый классификатором.

↓

↓

Теория глубже

Depthwise separable conv разделяет spatial mixing и channel mixing

Обычная convolution одновременно смешивает соседние пиксели и каналы. Depthwise separable convolution сначала применяет отдельный spatial filter к каждому каналу, а затем 1x1 convolution смешивает каналы.

Такое разделение резко снижает число параметров и FLOPs. Цена - меньше совместной выразительности в одном слое, поэтому MobileNet-подобные сети компенсируют это шириной, глубиной и аккуратным дизайном блоков.

  • Depthwise conv отвечает за spatial pattern внутри канала.
  • Pointwise 1x1 conv отвечает за mixing каналов.
  • Mobile architectures используют этот trade-off для скорости на edge devices.

Depthwise Separable Convolution (MobileNet)

Разделение свёртки на два этапа резко сокращает количество параметров:

Обычная Conv

Cost = k·k·C_in·C_out·H·W
Что означают символы
k·kумножений на позицию на пару каналовКаждый элемент ядра требует одного умножения с элементом входа. Для k=3: 9 умножений.
C_inвходные каналыКаждый выходной фильтр соединён со всеми C_in входными каналами — полная связь по глубине.
C_outвыходные каналыЧисло фильтров; каждый независим и стоит k·k·C_in операций. Итого умножаем на C_out.
H·Wпространственных позицийЯдро применяется в каждой из H·W позиций выходной карты. Пример: H=W=14 → 196 позиций.
Интуиция: обычная Conv смешивает пространство и каналы одновременно — каждый выходной нейрон зависит от k×k×C_in входных значений.
Аналогия: C_out поваров, каждый готовит блюдо, используя k×k×C_in ингредиентов из H×W точек.
Числовой пример: k=3, C_in=64, C_out=128, H=W=14: 9·64·128·196 ≈ 14.5M умножений.

Depthwise Sep.

Cost = k·k·C_in·H·W + C_in·C_out·H·W
Что означают символы
k·k·C_in·H·Wстоимость depthwiseДля каждого из C_in каналов применяется отдельный фильтр k×k — нет смешения каналов. Стоимость: k²·C_in·H·W.
C_in·C_out·H·Wстоимость pointwise1×1 свёртка смешивает C_in каналов в C_out. Нет пространственного ядра (k=1), но перебирает все C_in×C_out пары.
+два независимых шагаРазделение пространственной и канальной обработки: сначала depthwise (где?), затем pointwise (что смешать?).
Интуиция: вместо одного дорогого шага — два дешёвых: spatial mixing (depthwise) отдельно от channel mixing (pointwise).
Аналогия: сначала каждый повар обрабатывает только свои ингредиенты по рецепту, потом все смешивается в общем котле.
Числовой пример: k=3, C_in=64, C_out=128, H=W=14: depthwise = 9·64·196 ≈ 113K; pointwise = 64·128·196 ≈ 1.6M; итого ≈ 1.7M vs обычной ≈ 14.5M.
Ratio = 1/C_out + 1/k² ≈ 1/k² при большом C_out
Что означают символы
1/C_outвклад pointwise-шагаPointwise стоит C_in·C_out·H·W против обычного k²·C_in·C_out·H·W — экономия в k² раз по сравнению с обычным pointwise...
1/k²вклад depthwise-шагаDepthwise стоит k²·C_in·H·W — в C_out раз меньше, чем у обычного Conv. При больших C_out это слагаемое мало.
≈ 1/k²асимптотика при C_out→∞При C_out≫k² слагаемое 1/C_out пренебрежимо мало и основная экономия определяется размером ядра k.
kразмер ядраДля k=3: Ratio≈1/9, т.е. ~9× экономия. Именно поэтому depthwise separable conv так эффективна при k=3.
Интуиция: экономия растёт с k² — большое ядро даёт бо́льшую выгоду от разделения spatial и channel обработки.
Аналогия: разделить работу на два специализированных этапа выгоднее тем больше, чем сложнее исходная задача (большее k).
Числовой пример: k=3, C_out=128: Ratio = 1/128 + 1/9 ≈ 0.008 + 0.111 = 0.119 → ~8.4× экономия FLOPs.

Для k=3: экономия в ~9× при сохранении аналогичной выразительности.

6. Backprop в свёрточном слое

Прямой проход: Z = I ★ K + b, затем A = f(Z)

Пусть δ = ∂L/∂Z (incoming gradient, shape = Z).

Градиент по ядру ∂L/∂K

∂L/∂K[m,n] = Σi,j δ[i,j] · I[i+m, j+n]
Что означают символы
∂L/∂K[m,n]градиент по весу ядраНасколько нужно изменить вес K[m,n] для уменьшения потери. Форма — та же, что у ядра: k×k.
δ[i,j]входящий градиент = ∂L/∂Z[i,j]Градиент потери по выходу свёртки Z в позиции (i,j). Приходит из следующего слоя.
I[i+m, j+n]элемент входаЗначение входного изображения в позиции, которая взаимодействовала с весом K[m,n] при вычислении Z[i,j].
Σ_{i,j}суммирование по позициям выходаВес K[m,n] использовался во всех H_out×W_out позициях — все их вклады суммируются.
Интуиция: градиент по ядру — это кросс-корреляция входа I с градиентом δ. Та же структура, что и прямой проход, но роли переставлены.
Аналогия: "насколько каждый элемент трафарета виноват в ошибке?" — складываешь вклад всех мест, где этот элемент применялся.
Числовой пример: δ=[[1,1],[1,1]], I-патч: K[0,0] использовался с пикселями I[0,0]=1, I[0,1]=2, I[1,0]=4, I[1,1]=5 → ∂L/∂K[0,0]=1+2+4+5=12.
Суть: ∂L/∂K — это свёртка входа I с градиентом δ (кросс-корреляция δ по I)

Градиент по входу ∂L/∂I (для предыдущего слоя)

∂L/∂I[i,j] = Σm,n δ[i-m, j-n] · K[m,n]
Что означают символы
∂L/∂I[i,j]градиент по входуНасколько потеря чувствительна к пикселю I[i,j]. Нужен для обновления весов предыдущего слоя.
δ[i-m, j-n]входящий градиент со сдвигомПиксель I[i,j] вошёл в Z[i-m, j-n] через позицию ядра K[m,n]. Отсюда сдвиг (i-m, j-n) в аргументе δ.
K[m,n]вес ядраТот же обученный фильтр, что и в прямом проходе. Использование сдвинутого δ эквивалентно свёртке с перевёрнутым ядром.
Σ_{m,n}суммирование по ядруI[i,j] участвовал во всех позициях выхода, где ядро его "накрывало" — все эти вклады складываются.
Интуиция: backprop через свёртку — тоже свёртка, но с перевёрнутым ядром (transposed convolution / full convolution).
Аналогия: если прямой проход — "размазать" ядро по входу, то обратный — "собрать" ошибки обратно по тем же путям.
Числовой пример: K=[[1,0],[-1,0]], δ=[[1,1],[1,1]] → ∂L/∂I[0,0] = δ[0,0]·K[0,0]=1·1=1; ∂L/∂I[1,0] = δ[1,0]·K[0,0]+δ[0,0]·K[1,0]=1+(−1)=0.
Суть: это полная свёртка δ с повёрнутым на 180° ядром K — или traspозиционная свёртка

Градиент по bias

∂L/∂bc = Σi,j δc[i,j]
Что означают символы
∂L/∂b_cградиент по bias канала cBias b_c прибавляется к каждой из H_out×W_out позиций выходной карты канала c. Поэтому его вклад суммируется по всем позициям.
cиндекс канала/фильтраУ каждого из C_out фильтров свой scalar bias. Градиент считается независимо для каждого c.
δ_c[i,j]входящий градиент канала c∂L/∂Z_c[i,j] — насколько потеря чувствительна к активации Z_c в позиции (i,j).
Σ_{i,j}сумма по пространствуBias одинаков для всех H_out×W_out позиций → по правилу chain rule все их градиенты складываются.
Интуиция: один bias shared по всему пространству — значит, его градиент = сумма всех пространственных позиций (broadcast в прямом проходе → reduce в обратном).
Аналогия: если одна константа прибавляется к 196 позициям, то её вклад в ошибку — это сумма ошибок со всех 196 мест.
Числовой пример: H_out=W_out=4, δ_c = все единицы → ∂L/∂b_c = 16·1 = 16. Если δ_c неоднородный: просто сложить все 16 значений.

Суммируем градиент по всей пространственной карте — каждая позиция получает один и тот же bias, поэтому вклады складываются.

Ключевое наблюдение: backprop через свёртку — тоже свёртка (или её транспозиция). Это делает реализацию элегантной: те же примитивы CUDA/cuDNN.

Пошаговый числовой пример

Дано:I = [[1,2,3],[4,5,6],[7,8,9]], K = [[1,0],[-1,0]], stride=1, pad=0
Forward:Z[0,0] = 1·1+2·0+4·(-1)+5·0 = -3; Z[0,1] = 2·1+3·0+5·(-1)+6·0 = -3 и т.д.
Пусть δ = [[1,1],[1,1]] (upstream gradient).
∂L/∂K[0,0]= δ[0,0]·I[0,0] + δ[0,1]·I[0,1] + δ[1,0]·I[1,0] + δ[1,1]·I[1,1] = 1+2+4+5 = 12
∂L/∂K[1,0]= 1·4 + 1·5 + 1·7 + 1·8 = 24

7. Реализация на PyTorch

Базовый свёрточный блок

import torch import torch.nn as nn import torch.nn.functional as F class ConvBlock(nn.Module): """Conv → BN → ReLU (стандартный блок CNN)""" def __init__(self, in_ch, out_ch, k=3, s=1, p=1): super().__init__() self.conv = nn.Conv2d(in_ch, out_ch, k, stride=s, padding=p, bias=False) self.bn = nn.BatchNorm2d(out_ch) def forward(self, x): return F.relu(self.bn(self.conv(x)))

Простая CNN для CIFAR-10

class SimpleCNN(nn.Module): def __init__(self, num_classes=10): super().__init__() # Encoder: (3,32,32) → (256,4,4) self.features = nn.Sequential( ConvBlock(3, 64), # (64,32,32) ConvBlock(64, 128), # (128,32,32) nn.MaxPool2d(2, 2), # (128,16,16) ConvBlock(128, 256), # (256,16,16) ConvBlock(256, 256), # (256,16,16) nn.MaxPool2d(2, 2), # (256,8,8) ConvBlock(256, 512), # (512,8,8) nn.AdaptiveAvgPool2d(1), # (512,1,1) GAP ) self.classifier = nn.Linear(512, num_classes) def forward(self, x): x = self.features(x).squeeze(-1).squeeze(-1) # (B,512) return self.classifier(x) model = SimpleCNN() print(f"Параметры: {sum(p.numel() for p in model.parameters()):,}") # → Параметры: 3,742,730

Depthwise Separable Conv

class DepthwiseSeparable(nn.Module): def __init__(self, in_ch, out_ch, k=3, s=1): super().__init__() # Depthwise: каждый канал свёртывается отдельно self.dw = nn.Conv2d(in_ch, in_ch, k, stride=s, padding=k//2, groups=in_ch, bias=False) self.bn1 = nn.BatchNorm2d(in_ch) # Pointwise: смешиваем каналы 1×1 conv self.pw = nn.Conv2d(in_ch, out_ch, 1, bias=False) self.bn2 = nn.BatchNorm2d(out_ch) def forward(self, x): x = F.relu(self.bn1(self.dw(x))) return F.relu(self.bn2(self.pw(x))) # Сравнение параметров regular = nn.Conv2d(64, 128, 3, padding=1) dws = DepthwiseSeparable(64, 128) print(f"Обычная: {sum(p.numel() for p in regular.parameters()):,}") print(f"DW-Sep: {sum(p.numel() for p in dws.parameters()):,}") # → Обычная: 73,728 DW-Sep: 8,448 (экономия ~8.7×)

NumPy: свёртка вручную (2D)

import numpy as np from scipy.signal import correlate2d def conv2d_manual(img, kernel, padding=0, stride=1): """Ручная 2D кросс-корреляция (одноканальная)""" if padding > 0: img = np.pad(img, padding, mode='constant') H, W = img.shape k = kernel.shape[0] H_out = (H - k) // stride + 1 W_out = (W - k) // stride + 1 out = np.zeros((H_out, W_out)) for i in range(0, H_out): for j in range(0, W_out): out[i,j] = np.sum(img[i*stride:i*stride+k, j*stride:j*stride+k] * kernel) return out # Детектор вертикальных краёв (фильтр Собеля) sobel_x = np.array([[-1, 0, 1],[-2, 0, 2],[-1, 0, 1]]) img = np.random.randn(6, 6) result = conv2d_manual(img, sobel_x) print("Input:", img.shape, "→ Output:", result.shape) # (6,6) → (4,4)

Как закрепить тему: CNN

Этот блок связывает теорию с практическим решением: где применять идею, какие ошибки проверять и какие ручки менять в эксперименте.

Когда применять

  • Используй тему «CNN», когда выбираешь inductive bias: локальность, последовательность, attention, residual flow или нормализацию.
  • Архитектура должна соответствовать структуре данных: изображение, текст, последовательность и табличные признаки требуют разных предположений.
  • Перед усложнением архитектуры проверь, что bottleneck действительно в модели, а не в данных, loss или обучении.

Типичные ошибки

  • Копировать архитектуру без учёта размера данных, бюджета памяти и целевой метрики.
  • Сравнивать модели с разным compute budget и делать вывод, что одна архитектура всегда лучше.
  • Игнорировать shape flow: stride, padding, heads, channels и sequence length напрямую определяют память и скорость.

Что тюнить

  • Настраивай depth, width, normalization, dropout, kernel/head sizes и residual structure.
  • Следи за memory footprint: иногда ограничение sequence length или feature map важнее числа параметров.
  • Для Transformer-подобных моделей отдельно проверяй attention mask, positional encoding и scaling.

Проверка темы - уметь считать shapes и объяснять, почему convolution использует local connectivity и weight sharing.

[Foundational] Output shape

[Foundational] Parameters count

Сколько параметров у conv 3x3, 3 input channels, 64 output channels? Решение: \(3\cdot3\cdot3\cdot64=1728\) плюс 64 bias, если bias включён.

Зачем нужен 1x1 conv? Решение: он смешивает каналы без spatial mixing и меняет channel dimension.

Почему MobileNet экономит параметры? Решение: spatial filtering делается отдельно по каналам, затем 1x1 conv смешивает каналы.

Проверь себя

Вопросы для самопроверки

  1. Почему свёрточные сети эффективнее полносвязных для изображений?

    CNN используют три ключевых свойства: (1) local connectivity — каждый нейрон смотрит только на рецептивное поле, (2) weight sharing — один фильтр применяется по всему изображению (меньше параметров), (3) translation equivariance — детектор угла работает в любом месте изображения.

  2. Что делает операция pooling и зачем она нужна?

    Pooling (max или average) уменьшает пространственное разрешение, агрегируя соседние значения. Max pooling сохраняет наиболее сильный сигнал, обеспечивая инвариантность к малым сдвигам. Это снижает размерность, уменьшает вычислительную стоимость и увеличивает рецептивное поле.

  3. Что такое receptive field и как он растёт с глубиной?

    Receptive field — область входного изображения, влияющая на один нейрон. После каждого свёрточного слоя 3×3 он растёт на 2 пикселя. Pooling 2×2 удваивает рецептивное поле. Dilated convolution позволяет резко увеличить рецептивное поле без потери разрешения.

  4. В чём разница между 1×1 convolution и обычной?

    1×1 свёртка не смотрит на пространственный контекст — она просто смешивает информацию по каналам для каждого пикселя. Используется для изменения числа каналов (channel bottleneck в ResNet, Inception), вычислительно дёшево. Это по сути MLP применённый к каждому пикселю отдельно.

1. Дискретная свёртка — математика

Свёртка в DL — дискретная кросс-корреляция (по историческим причинам называется свёрткой):

(I ★ K)[i,j] = Σₘ Σₙ K[m,n] · I[i+m, j+n]
Что означают символы
ΣсуммаСобирает вклад многих элементов в один итог.
KKeyКак каждый элемент описан для сопоставления с query.
=связьПоказывает, как левая величина выражается через правую часть.
Интуиция: Формула собирает в одной записи влияния входов на итоговый результат.
Аналогия: Как таблица влияния, где видно, какой фактор за что отвечает.
Числовой пример: Если вес признака 2, значение признака 3 и bias 1, вклад будет 2·3+1=7.

где I — входное изображение (H×W), K — ядро (kernel) размером k×k.

Размер выхода при valid padding:

H_out = ⌊(H_in + 2P - k) / S⌋ + 1
Что означают символы
KKeyКак каждый элемент описан для сопоставления с query.
=связьПоказывает, как левая величина выражается через правую часть.
Интуиция: Формула связывает несколько величин и показывает, как они влияют друг на друга.
Аналогия: Как панель с несколькими ручками, каждая из которых меняет общий результат.
Числовой пример: Если один множитель равен 2, а вход увеличился с 3 до 4, вклад этой части вырос с 6 до 8.

P — padding, S — stride (шаг). Аналогично для W.

Истинная свёртка vs кросс-корреляция: при свёртке ядро поворачивается на 180°: (I * K)[i,j] = Σ K[-m,-n]·I[i+m,j+n]. В DL ядро обучается — поворот не имеет значения, поэтому используют кросс-корреляцию, называя её "свёрткой".

2. Рецептивное поле (Receptive Field)

Рецептивное поле нейрона — размер области входного изображения, влияющей на его активацию.

RFL = RFL-1 + (k-1) · Πi=1L-1 si
Что означают символы
KKeyКак каждый элемент описан для сопоставления с query.
=связьПоказывает, как левая величина выражается через правую часть.
Интуиция: Формула связывает несколько величин и показывает, как они влияют друг на друга.
Аналогия: Как панель с несколькими ручками, каждая из которых меняет общий результат.
Числовой пример: Если один множитель равен 2, а вход увеличился с 3 до 4, вклад этой части вырос с 6 до 8.

k — kernel size слоя L, si — stride слоёв до L

Зачем важно? Для классификации объекта нейрон должен "видеть" весь объект. Глубокие сети с малыми ядрами достигают большого RF без избыточного числа параметров.

Сравнение: одно большое ядро vs стек малых

7×7conv, 1 слой
Параметры: 7·7·C·C = 49C²
RF = 7
3×3conv × 3 слоя
Параметры: 3·(3·3·C·C) = 27C²
RF = 1+2+2+2 = 7

Три слоя 3×3 дают тот же RF, что один 7×7, но параметров в 1.8× меньше и между слоями — нелинейности!

4. Equivariance и Invariance

Определения

Equivariant: f(T(x)) = T'(f(x))
Invariant: f(T(x)) = f(x)
Что означают символы
x / XвходДанные или признаки, которые формула обрабатывает.
=связьПоказывает, как левая величина выражается через правую часть.
Интуиция: Формула связывает несколько величин и показывает, как они влияют друг на друга.
Аналогия: Как панель с несколькими ручками, каждая из которых меняет общий результат.
Числовой пример: Если один множитель равен 2, а вход увеличился с 3 до 4, вклад этой части вырос с 6 до 8.
Equivariance (свёртка)

Сдвиг входа → сдвиг карты признаков. Одни и те же признаки обнаруживаются в любой позиции одним и тем же ядром.
Invariance (MaxPool, GAP)

Небольшой сдвиг объекта не меняет итоговую классификацию — pooling "поглощает" смещение.

Математическое доказательство equivariance

Шаг 1.Пусть T_δ — оператор сдвига: (T_δ I)[i,j] = I[i-δ, j]
Шаг 2.(T_δ I ★ K)[i,j] = Σ K[m,n]·I[i+m-δ, j+n] = (I★K)[i-δ, j]
Шаг 3.Т.е. f(T_δ I) = T_δ(f(I)) → свёртка equivariant к сдвигу ✓
Свёртка не equivariant к: ротациям, масштабированию, отражениям. Для этого: увеличение данных (data augmentation), группо-equivariant свёртки (G-CNN), или пространственный трансформер (STN).

Parameter Sharing — суть экономии

FC-слой (28²→128): 784 × 128 = 100352 параметров
Conv(1→1, 3×3) на том же входе: 9 параметров
Что означают символы
=связьПоказывает, как левая величина выражается через правую часть.
Интуиция: Формула ищет знакомый локальный паттерн во входе.
Аналогия: Как вести лупу по изображению и искать совпадение с маленьким шаблоном.
Числовой пример: Фильтр [1, −1] на фрагменте [5, 2] даёт 1·5 + (−1)·2 = 3: сильный отклик на перепад.

Одно ядро сканирует все пространственные позиции — это и есть parameter sharing. Ядро учит детектор признака (например, "вертикальный край"), который работает везде в изображении.

5. Типичная CNN-архитектура

Классическая CNN — стек блоков "Conv → BN → ReLU → Pool", завершённый классификатором.

Input: (B, 3, H, W) — RGB изображение
↓
Conv2d(3→64, 3×3, P=1) → BN → ReLU → (B, 64, H, W)
↓
Conv2d(64→128, 3×3, P=1) → BN → ReLU → (B, 128, H, W)
↓
MaxPool(2, S=2) → (B, 128, H/2, W/2)
↓
Conv2d(128→256, 3×3, P=1) → BN → ReLU → (B, 256, H/2, W/2)
↓
GlobalAvgPool → (B, 256)
↓
Linear(256 → num_classes) → Softmax → (B, C)

6. Backprop в свёрточном слое

Прямой проход: Z = I ★ K + b, затем A = f(Z)

Пусть δ = ∂L/∂Z (incoming gradient, shape = Z).

Градиент по ядру ∂L/∂K

∂L/∂K[m,n] = Σi,j δ[i,j] · I[i+m, j+n]
Что означают символы
∂частная производнаяЧувствительность выхода к изменению одной переменной.
ΣсуммаСобирает вклад многих элементов в один итог.
KKeyКак каждый элемент описан для сопоставления с query.
=связьПоказывает, как левая величина выражается через правую часть.
Интуиция: Формула собирает в одной записи влияния входов на итоговый результат.
Аналогия: Как таблица влияния, где видно, какой фактор за что отвечает.
Числовой пример: Если вес признака 2, значение признака 3 и bias 1, вклад будет 2·3+1=7.
Суть: ∂L/∂K — это свёртка входа I с градиентом δ (кросс-корреляция δ по I)

Градиент по входу ∂L/∂I (для предыдущего слоя)

∂L/∂I[i,j] = Σm,n δ[i-m, j-n] · K[m,n]
Что означают символы
∂частная производнаяЧувствительность выхода к изменению одной переменной.
ΣсуммаСобирает вклад многих элементов в один итог.
KKeyКак каждый элемент описан для сопоставления с query.
=связьПоказывает, как левая величина выражается через правую часть.
Интуиция: Формула собирает в одной записи влияния входов на итоговый результат.
Аналогия: Как таблица влияния, где видно, какой фактор за что отвечает.
Числовой пример: Если вес признака 2, значение признака 3 и bias 1, вклад будет 2·3+1=7.
Суть: это полная свёртка δ с повёрнутым на 180° ядром K — или traspозиционная свёртка

Градиент по bias

∂L/∂bc = Σi,j δc[i,j]
Что означают символы
∂частная производнаяЧувствительность выхода к изменению одной переменной.
ΣсуммаСобирает вклад многих элементов в один итог.
bbiasСмещение, которое двигает порог или базовый уровень ответа.
=связьПоказывает, как левая величина выражается через правую часть.
Интуиция: Формула собирает в одной записи влияния входов на итоговый результат.
Аналогия: Как таблица влияния, где видно, какой фактор за что отвечает.
Числовой пример: Если вес признака 2, значение признака 3 и bias 1, вклад будет 2·3+1=7.

Суммируем градиент по всей пространственной карте — каждая позиция получает один и тот же bias, поэтому вклады складываются.

Ключевое наблюдение: backprop через свёртку — тоже свёртка (или её транспозиция). Это делает реализацию элегантной: те же примитивы CUDA/cuDNN.

Пошаговый числовой пример

Дано:I = [[1,2,3],[4,5,6],[7,8,9]], K = [[1,0],[-1,0]], stride=1, pad=0
Forward:Z[0,0] = 1·1+2·0+4·(-1)+5·0 = -3; Z[0,1] = 2·1+3·0+5·(-1)+6·0 = -3 и т.д.
Пусть δ = [[1,1],[1,1]] (upstream gradient).
∂L/∂K[0,0]= δ[0,0]·I[0,0] + δ[0,1]·I[0,1] + δ[1,0]·I[1,0] + δ[1,1]·I[1,1] = 1+2+4+5 = 12
∂L/∂K[1,0]= 1·4 + 1·5 + 1·7 + 1·8 = 24

7. Реализация на PyTorch

Базовый свёрточный блок

import torch import torch.nn as nn import torch.nn.functional as F class ConvBlock(nn.Module): """Conv → BN → ReLU (стандартный блок CNN)""" def __init__(self, in_ch, out_ch, k=3, s=1, p=1): super().__init__() self.conv = nn.Conv2d(in_ch, out_ch, k, stride=s, padding=p, bias=False) self.bn = nn.BatchNorm2d(out_ch) def forward(self, x): return F.relu(self.bn(self.conv(x)))

Простая CNN для CIFAR-10

class SimpleCNN(nn.Module): def __init__(self, num_classes=10): super().__init__() # Encoder: (3,32,32) → (256,4,4) self.features = nn.Sequential( ConvBlock(3, 64), # (64,32,32) ConvBlock(64, 128), # (128,32,32) nn.MaxPool2d(2, 2), # (128,16,16) ConvBlock(128, 256), # (256,16,16) ConvBlock(256, 256), # (256,16,16) nn.MaxPool2d(2, 2), # (256,8,8) ConvBlock(256, 512), # (512,8,8) nn.AdaptiveAvgPool2d(1), # (512,1,1) GAP ) self.classifier = nn.Linear(512, num_classes) def forward(self, x): x = self.features(x).squeeze(-1).squeeze(-1) # (B,512) return self.classifier(x) model = SimpleCNN() print(f"Параметры: {sum(p.numel() for p in model.parameters()):,}") # → Параметры: 3,742,730

Depthwise Separable Conv

class DepthwiseSeparable(nn.Module): def __init__(self, in_ch, out_ch, k=3, s=1): super().__init__() # Depthwise: каждый канал свёртывается отдельно self.dw = nn.Conv2d(in_ch, in_ch, k, stride=s, padding=k//2, groups=in_ch, bias=False) self.bn1 = nn.BatchNorm2d(in_ch) # Pointwise: смешиваем каналы 1×1 conv self.pw = nn.Conv2d(in_ch, out_ch, 1, bias=False) self.bn2 = nn.BatchNorm2d(out_ch) def forward(self, x): x = F.relu(self.bn1(self.dw(x))) return F.relu(self.bn2(self.pw(x))) # Сравнение параметров regular = nn.Conv2d(64, 128, 3, padding=1) dws = DepthwiseSeparable(64, 128) print(f"Обычная: {sum(p.numel() for p in regular.parameters()):,}") print(f"DW-Sep: {sum(p.numel() for p in dws.parameters()):,}") # → Обычная: 73,728 DW-Sep: 8,448 (экономия ~8.7×)

NumPy: свёртка вручную (2D)

import numpy as np from scipy.signal import correlate2d def conv2d_manual(img, kernel, padding=0, stride=1): """Ручная 2D кросс-корреляция (одноканальная)""" if padding > 0: img = np.pad(img, padding, mode='constant') H, W = img.shape k = kernel.shape[0] H_out = (H - k) // stride + 1 W_out = (W - k) // stride + 1 out = np.zeros((H_out, W_out)) for i in range(0, H_out): for j in range(0, W_out): out[i,j] = np.sum(img[i*stride:i*stride+k, j*stride:j*stride+k] * kernel) return out # Детектор вертикальных краёв (фильтр Собеля) sobel_x = np.array([[-1, 0, 1],[-2, 0, 2],[-1, 0, 1]]) img = np.random.randn(6, 6) result = conv2d_manual(img, sobel_x) print("Input:", img.shape, "→ Output:", result.shape) # (6,6) → (4,4)

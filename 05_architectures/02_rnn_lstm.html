RNN — нейронная сеть с памятью: скрытое состояние несёт информацию из прошлого. LSTM решил проблему vanishing gradient, сделав возможным обучение на длинных последовательностях.

1. Рекуррентная нейронная сеть — идея

Теория глубже

RNN сжимает прошлое в скрытое состояние

RNN читает последовательность по шагам и хранит историю в hidden state. Это сильное ограничение: вся полезная информация о прошлом должна поместиться в вектор фиксированного размера.

Такой inductive bias подходит для потоковых данных и коротких зависимостей. Но когда нужно помнить много деталей на больших расстояниях, fixed hidden state становится узким горлом.

  • h_t хранит summary прошлого.
  • x_t добавляет новую информацию текущего шага.
  • Один и тот же recurrent weight переиспользуется на всех позициях.

RNN обрабатывает последовательность x₁, x₂, …, xₜ, поддерживая hidden state hₜ — «память» о прошлом.

hₜ = tanh( Wₕ·hₜ₋₁ + Wₓ·xₜ + b ) yₜ = Wᵧ·hₜ + bᵧ
Что означают символы
hₜhidden state на шаге tВектор размера H — «память» сети после обработки t-го входа. hₜ₋₁ — состояние с предыдущего шага.
xₜвход на шаге tВектор размера n: токен, пиксель, временной признак или embedding на текущем шаге последовательности.
Wₕрекуррентная матрица весовРазмер H×H. Отображает прошлое состояние hₜ₋₁ в новое. Одна и та же на всех шагах — parameter sharing во времени.
Wₓматрица весов входаРазмер H×n. Отображает текущий вход xₜ в пространство hidden state.
tanhнелинейностьСжимает сумму в диапазон (−1,1). Производная tanh'(z)=1−tanh²(z) ∈ [0,1] — источник vanishing gradient.
yₜ, Wᵧвыход и его весаyₜ ∈ ℝᴼ — предсказание на шаге t. Wᵧ (O×H) — выходная матрица, применяется независимо для каждого шага.
Интуиция: новое состояние — взвешенная смесь прошлого hₜ₋₁ и текущего входа xₜ, сжатая tanh. Одни и те же веса используются на всех T шагах.
Аналогия: как человек с ограниченной памятью — на каждом шаге обновляет своё понимание, смешивая старые знания с новой информацией.
Числовой пример: H=2, n=1, t=1: hₜ = tanh(Wₕ·[0,0] + Wₓ·1 + b) = tanh(Wₓ + b). Параметров: H·H + H·n + H = 4+2+2=8 (без Wᵧ).

Параметры Wₕ, Wₓ, b — одни и те же на каждом шаге. Это и есть parameter sharing во времени.

Размерности: xₜ ∈ ℝⁿ, hₜ ∈ ℝᴴ, Wₓ ∈ ℝᴴˣⁿ, Wₕ ∈ ℝᴴˣᴴ, Wᵧ ∈ ℝᴼˣᴴ
Параметров: H·n + H·H + H + O·H + O = H(n + H + 1) + O(H+1)

x₁..xₜ → y₁..yₜ
Что означают символы

Аналогия: переводчик-синхронист: слышит слово — сразу переводит, помня контекст предыдущих слов.
Числовой пример: предложение из 5 слов → 5 шагов → 5 тегов POS (NN, VB, DT, NN, .).

Аналогия: дать художнику одно слово — и он пишет целый рассказ по нему.
Числовой пример: image captioning: x₁ = ResNet-embedding(224×224) → h₀ → генерация: y₁=«a», y₂=«dog», y₃=«running», y₄=<EOS>.

Долгая память RNN зависит от спектра recurrent dynamics

Backprop through time умножает градиент на похожие матрицы много раз. Если эффективные собственные значения меньше единицы, сигнал затухает. Если больше единицы, сигнал взрывается.

Это не только математическая проблема. На практике RNN может хорошо учить ближайшие зависимости и почти игнорировать далёкие, потому что gradient от далёкого шага просто не доходит до ранних параметров.

  • Vanishing gradient стирает дальние зависимости.
  • Exploding gradient требует clipping и осторожного lr.
  • Gates в LSTM/GRU создают более управляемые пути для памяти.

2. Проблема затухающего/взрывного градиента

При backprop через T шагов градиент умножается на Wₕ и tanh' на каждом шаге:

∂L/∂h₁ = ∂L/∂hₜ · Π_{k=2}^{T} ( diag(1-hₖ²) · Wₕᵀ )
Что означают символы
∂L/∂h₁градиент на первом шагеТо, что нужно вычислить — насколько потеря чувствительна к h₁. При затухании это стремится к 0, Wₓ и Wₕ не обновляются для дальних зависимостей.
∂L/∂hₜвходящий градиентГрадиент, пришедший с последнего шага T (от loss). Он передаётся назад через T-1 шагов.
Π_{k=2}^{T}произведение T-1 ЯкобиановChain rule через время: каждый шаг добавляет матричный множитель. T-1 таких умножений — источник затухания или взрыва.
diag(1-hₖ²)Якобиан tanhПроизводная tanh'(z)=1−tanh²(z)∈[0,1]. Диагональная матрица. При насыщении (|hₖ|≈1) значения близки к 0 — сильное затухание.
Wₕᵀтранспонированная рекуррентная матрицаИспользуется при backprop (chain rule). Если сингулярные числа Wₕ малы, умножение сжимает градиент; если велики — взрывает.
Интуиция: каждый шаг назад умножает градиент на diag(tanh')·Wₕᵀ. При T=50 и норме произведения 0.8 — градиент уменьшается в 0.8⁴⁹≈0.00002×.
Аналогия: ксерокопирование ксерокопии 50 раз — сигнал исчезает.
Числовой пример: T=10, каждый шаг умножает норму на 0.7: 0.7⁹≈0.04 — градиент уменьшается в 25 раз.

diag(1-hₖ²) — Якобиан tanh: tanh'(z) = 1 - tanh²(z) ∈ [0,1]

Норма произведения матриц

‖∂hₜ/∂h₁‖ ≤ (λ_max · max|tanh'|)^(T-1)
Что означают символы
‖∂hₜ/∂h₁‖норма суммарного ЯкобианаНасколько hₜ чувствительно к h₁. Если близко к 0 — дальние зависимости исчезают; если >1 — взрыв.
λ_maxнаибольшее сингулярное число WₕCharacterizes norm-amplification per step from Wₕ. λ_max > 1 → potential explosion; λ_max < 1 → potential vanishing.
max|tanh'|максимум производной tanhtanh'(z)=1−tanh²(z)∈[0,1]; максимум = 1 достигается при z=0. В насыщении (|z|≫0) производная → 0 и усиливает затухание.
(T-1)степень = число шагов назадПроизведение умножается T-1 раз. Экспоненциальный рост или затухание в зависимости от базы λ_max·max|tanh'|.
Интуиция: если произведение λ_max·max|tanh'| меньше 1, норма градиента убывает экспоненциально с глубиной T — модель «не видит» дальние шаги.
Аналогия: мяч, теряющий 20% кинетической энергии при каждом отскоке: после 50 отскоков останется 0.8⁵⁰≈0.00014 от начальной энергии.
Числовой пример: λ_max=0.9, max|tanh'|=0.8 → база=0.72; T=20: 0.72¹⁹≈0.00045 — практически ноль.

где λ_max — наибольшее сингулярное число Wₕ.

Затухание

λ_max · max|tanh'| < 1
→ градиент → 0 экспоненциально
→ RNN «забывает» далёкое прошлое
Взрыв

λ_max · max|tanh'| > 1
→ градиент → ∞ экспоненциально
→ обучение взрывается (NaN)
Как бороться со взрывом: Gradient Clipping — обрезаем норму: if ‖g‖ > threshold: g ← g · threshold/‖g‖
Как бороться с затуханием: LSTM/GRU, ResNet-like skip connections, инициализация Wₕ = I

Теория глубже

Cell state работает как почти линейная магистраль памяти

Главная идея LSTM - отделить долгосрочную память c_t от краткосрочного hidden state h_t. Cell state обновляется через forget и input gates, поэтому информация может пройти через много шагов с меньшим разрушением.

Gates не просто добавляют параметры. Они дают модели право решать, что забыть, что записать и что показать наружу. Это делает память управляемой, а не полностью перезаписываемой на каждом шаге.

  • Forget gate контролирует сохранение старой памяти.
  • Input gate контролирует запись новой информации.
  • Output gate контролирует, какая часть памяти влияет на hidden state.

3. LSTM — Long Short-Term Memory (Hochreiter & Schmidhuber, 1997)

LSTM добавляет cell state cₜ — «конвейерная лента» с минимальными трансформациями, управляемая тремя воротами.

Forget Gate fₜ
fₜ = σ(Wf·[hₜ₋₁, xₜ] + bf)
Что означают символы
fₜforget gateВектор значений ∈(0,1) размером H. fₜ[i]≈0 → «забыть» i-й элемент cell state; fₜ[i]≈1 → «сохранить».
σsigmoid (не std!)σ(z)=1/(1+e⁻ᶻ) ∈(0,1). Используется для всех LSTM-gates — поэлементно задаёт «степень открытия» ворот.
Wfматрица весов forget gateРазмер H×(H+n). Обучается определять, когда старая память больше не нужна.
[hₜ₋₁, xₜ]конкатенация состояния и входаВектор размера (H+n): сначала прошлый hidden state, затем текущий вход. Все 4 ворота LSTM используют ту же конкатенацию.
Интуиция: forget gate решает, что из накопленной памяти cₜ₋₁ стоит сохранить, глядя на текущий вход и прошлое состояние.
Аналогия: редактор, просматривающий черновик: «это предложение оставим, это удалим».
Числовой пример: fₜ=[0.1, 0.9, 0.05, 1.0] → 1-й и 3-й элементы cell state почти стёрты, 2-й и 4-й — сохранены.
«Что забыть из cell state?» fₜ∈(0,1) поэлементно
Input Gate iₜ
iₜ = σ(Wi·[hₜ₋₁, xₜ] + bi)
c̃ₜ = tanh(Wc·[hₜ₋₁, xₜ] + bc)
Что означают символы
iₜinput gateВектор ∈(0,1) — «насколько много» записать. iₜ[i]≈0 → не обновляем i-й элемент; iₜ[i]≈1 → полностью записываем кандидата.
c̃ₜкандидат cell state«Что хотим записать» — вектор ∈(−1,1) из tanh. Предлагает новое содержимое; iₜ контролирует, сколько действительно запишется.
σ / tanhразные нелинейностиσ для gate (открыть/закрыть ∈(0,1)); tanh для кандидата (содержимое ∈(−1,1)). Разделение «сколько» и «что».
Wi, Wcвеса gate и кандидатаДва разных набора весов H×(H+n). Wi обучается решать «когда писать»; Wc — «что писать».
Интуиция: input gate разделяет «что предлагается» (c̃ₜ от tanh) и «сколько записываем» (iₜ от σ). Вместе они добавляют новую информацию в cell state.
Аналогия: c̃ₜ — черновик записи в ежедневнике; iₜ — решение «записать полностью, частично или вовсе не записывать».
Числовой пример: iₜ=[0,1,0.5], c̃ₜ=[0.8,−0.3,0.6] → вклад в cₜ: [0, −0.3, 0.3]. Первый элемент не обновляется вообще.
«Что записать?» iₜ — сколько, c̃ₜ — что
Output Gate oₜ
oₜ = σ(Wo·[hₜ₋₁, xₜ] + bo)
Что означают символы
oₜoutput gateВектор ∈(0,1) — «насколько много» из cₜ переходит в hₜ. oₜ≈0 → скрываем cell state от внешнего мира; oₜ≈1 → полностью экспонируем.
σsigmoidσ(z)∈(0,1) — как у других gates. Определяет степень «прозрачности» между cₜ и hₜ.
Woвеса output gateH×(H+n). Обучается решать, какая часть долгосрочной памяти cₜ нужна прямо сейчас.
[hₜ₋₁, xₜ]конкатенацияТа же конкатенация (H+n), что и у fₜ и iₜ. Output gate учится «открываться» именно когда релевантная память нужна.
Интуиция: oₜ фильтрует cₜ перед формированием hₜ: hₜ = oₜ ⊙ tanh(cₜ). Cell state — «внутренняя память»; hₜ — «то, что показывается наружу».
Аналогия: ноутбук с личными записями (cₜ): output gate решает, что сегодня открыть коллеге, а что оставить для себя.
Числовой пример: cₜ=[2,−1,3], oₜ=[1,0,0.5] → tanh(cₜ)≈[0.96,−0.76,0.99] → hₜ≈[0.96, 0, 0.50].
«Что вывести в hₜ?»

Обновление cell state и hidden state

cₜ = fₜ ⊙ cₜ₋₁ + iₜ ⊙ c̃ₜ hₜ = oₜ ⊙ tanh(cₜ)
Что означают символы
cₜновый cell stateДолгосрочная память LSTM — «конвейерная лента». Обновляется аддитивно: старая память частично сохраняется, новая добавляется.
fₜ ⊙ cₜ₋₁взвешенная старая памятьForget gate fₜ поэлементно умножает старый cell state: fₜ≈0 → стираем, fₜ≈1 → сохраняем. Нет матричного умножения → градиент течёт напрямую.
iₜ ⊙ c̃ₜвзвешенная новая информацияInput gate iₜ контролирует, сколько кандидата c̃ₜ записывается. Аддитивное обновление — ключ к gradient highway.
⊙произведение АдамараПоэлементное умножение (не матричное). Каждый из H элементов умножается независимо — нет смешения между элементами.
hₜ = oₜ ⊙ tanh(cₜ)hidden state из cell stateOutput gate oₜ фильтрует cₜ; tanh сжимает cell state в (−1,1). hₜ — это то, что видно «снаружи»; cₜ — внутренняя память.
Интуиция: cₜ обновляется аддитивно (не мультипликативно через Wₕ) — поэтому ∂cₜ/∂cₜ₋₁ = fₜ, а не произведение матриц с tanh'. Это «highway» для градиента.
Аналогия: записная книжка: на каждом шаге часть старых записей стираем (fₜ), часть новых добавляем (iₜ·c̃ₜ), и показываем коллеге только нужное (oₜ·tanh).
Числовой пример: cₜ₋₁=2, fₜ=0.9, iₜ=0.8, c̃ₜ=0.5: cₜ=0.9·2+0.8·0.5=1.8+0.4=2.2. При oₜ=0.7: hₜ=0.7·tanh(2.2)≈0.7·0.976=0.68.

⊙ — поэлементное произведение (Hadamard product)

Почему LSTM решает vanishing gradient?
Градиент по cₜ₋₁: ∂cₜ/∂cₜ₋₁ = fₜ — это просто умножение на fₜ, без матричного произведения и tanh'!
При fₜ ≈ 1 градиент течёт практически без затухания через T шагов.

Параметры LSTM

4 · (H·H + H·n + H) = 4H(H + n + 1) параметров
Что означают символы
4число gates × матриц весовЧетыре независимых набора весов: для f (forget), i (input), c̃ (candidate), o (output). Каждый обучается своей задаче.
H·Hрекуррентные веса одного gateЧасть Wf, Wi и т.д., отвечающая за hₜ₋₁. Размер H×H для каждого из 4 gates.
H·nвходные веса одного gateЧасть матриц весов, отвечающая за xₜ. Размер H×n для каждого gate.
Hbias одного gateВектор bias размера H на каждый из 4 gates. Итого: 4·(H²+Hn+H) = 4H(H+n+1).
Интуиция: LSTM в 4 раза тяжелее RNN с тем же H из-за 4 независимых gate-матриц — это цена управляемой памяти.
Аналогия: вместо одного рычага управления (RNN) — четыре отдельных тумблера (f, i, c̃, o), каждый со своим контроллером.
Числовой пример: H=128, n=32: 4·128·(128+32+1) = 512·161 = 82 432 параметра. Базовый RNN: 128·(128+32+1) = 20 608.

В 4 раза больше, чем у RNN — за каждый из 4 вентилей (f, i, c̃, o)

4. GRU — Gated Recurrent Unit (Cho et al., 2014)

GRU — упрощённая версия LSTM: два ворота вместо трёх, нет отдельного cell state.

Reset Gate rₜ
rₜ = σ(Wr·[hₜ₋₁, xₜ] + br)
Что означают символы
rₜreset gateВектор ∈(0,1). Контролирует, сколько прошлого hₜ₋₁ используется при вычислении кандидата h̃ₜ. rₜ≈0 → игнорировать прошлое.
σsigmoidСжимает линейное преобразование в (0,1) — «степень открытия» ворот. Не std dev!
Wrвеса reset gateМатрица H×(H+n). Обучается определять, когда старый контекст мешает формированию нового кандидата.
[hₜ₋₁, xₜ]конкатенацияВектор (H+n). Reset gate видит оба сигнала и решает: «этот вход требует «обнуления» истории?»
Интуиция: reset gate определяет, насколько прошлый контекст hₜ₋₁ участвует в создании нового «предложения» h̃ₜ. rₜ≈0 позволяет начать «с чистого листа».
Аналогия: переключение темы в разговоре: «всё, что было сказано раньше, не важно для следующего ответа».
Числовой пример: rₜ=[0, 1] → в кандидате h̃ₜ первый элемент hₜ₋₁ обнулён: rₜ⊙hₜ₋₁=[0·h₁,1·h₂]=[0,h₂].
«Насколько забыть прошлое при вычислении кандидата?»
Update Gate zₜ
zₜ = σ(Wz·[hₜ₋₁, xₜ] + bz)
Что означают символы
zₜupdate gateВектор ∈(0,1). Контролирует интерполяцию: zₜ→0 → hₜ≈hₜ₋₁ (держим прошлое); zₜ→1 → hₜ≈h̃ₜ (полностью обновляемся).
σsigmoidВыход ∈(0,1) — весовой коэффициент для линейной интерполяции в следующей строке формулы.
Wzвеса update gateH×(H+n). Обучается определять, когда нужно «отпустить» прошлое и принять новое состояние.
[hₜ₋₁, xₜ]конкатенацияВектор (H+n). Update gate смотрит на оба сигнала и решает: «на сколько процентов менять состояние?»
Интуиция: update gate — это «частота обновления» для каждого элемента hₜ. Он заменяет и forget gate, и input gate из LSTM одним параметром.
Аналогия: термостат: «на 70% держим старую температуру, на 30% принимаем новое значение».
Числовой пример: zₜ=0.3, hₜ₋₁=2.0, h̃ₜ=−1.0 → hₜ=(1−0.3)·2.0+0.3·(−1.0)=1.4+(−0.3)=1.1.
«Сколько старого vs нового состояния взять?»
h̃ₜ = tanh(W·[rₜ ⊙ hₜ₋₁, xₜ] + b) ← кандидат hₜ = (1 - zₜ) ⊙ hₜ₋₁ + zₜ ⊙ h̃ₜ ← линейная интерполяция
Что означают символы
h̃ₜкандидат hidden state«Что хочет записать» GRU, если бы обновлялся полностью. tanh сжимает в (−1,1). Использует только rₜ-гейтированное прошлое.
rₜ ⊙ hₜ₋₁reset-гейтированное прошлоеReset gate rₜ фильтрует hₜ₋₁ перед кандидатом: rₜ≈0 → кандидат игнорирует прошлое; rₜ≈1 → использует полностью.
(1−zₜ) ⊙ hₜ₋₁вес старого состоянияДоля прошлого hₜ₋₁ в итоговом hₜ. zₜ→0 → hₜ≈hₜ₋₁ (почти не обновляемся — как большой forget gate в LSTM).
zₜ ⊙ h̃ₜвес нового кандидатаДоля кандидата h̃ₜ в итоговом hₜ. Конвекс-комбинация с (1−zₜ): сумма весов = 1, поэтому норма hₜ контролирована.
⊙произведение АдамараПоэлементное умножение. Каждый из H элементов hₜ интерполируется независимо — GRU может «помнить» одни элементы и обновлять другие.
Интуиция: GRU объединяет forget+input gates LSTM в один update gate zₜ через конвекс-комбинацию. Нет отдельного cell state — hₜ несёт и долго-, и краткосрочную память.
Аналогия: диджей с фейдером: zₜ=0.3 → 70% старой дорожки + 30% новой.
Числовой пример: hₜ₋₁=1.0, h̃ₜ=−0.5, zₜ=0.8 → hₜ=(1−0.8)·1.0+0.8·(−0.5)=0.2+(−0.4)=−0.2.
Интуиция zₜ: при zₜ→0 hₜ≈hₜ₋₁ (игнорируем вход, помним прошлое). При zₜ→1 hₜ≈h̃ₜ (полностью обновляемся).

Сравнение RNN / GRU / LSTM

МодельВоротаПараметрыCell StateСкорость
RNN0H(n+H+1)+O(H+1)нетбыстрее
GRU2 (r, z)3H(n+H+1)+O(H+1)нетсредняя

На практике GRU ≈ LSTM по качеству на большинстве задач, но в 1.3× быстрее.

Параметры GRU

3 · (H·(H+n) + H) = 3H(H + n + 1) параметров
Что означают символы
3число матриц весовТри набора: Wr (reset gate), Wz (update gate), W (candidate h̃ₜ). Меньше, чем 4 у LSTM — нет отдельного output gate и cell state.
H·(H+n)размер каждой матрицыH выходов × (H+n) входов = H·H (рекуррентная часть) + H·n (входная часть). В LSTM они записаны раздельно, здесь объединены.
Hbias каждого gateВектор размера H на каждый из 3 наборов. Итого: 3·(H(H+n)+H) = 3H(H+n+1).
Интуиция: GRU дешевле LSTM: 3H(H+n+1) против 4H(H+n+1) — экономия 25% параметров за счёт объединения forget+input gates в один update gate.
Аналогия: GRU — «облегчённый LSTM»: тот же функционал памяти, но в более компактном корпусе.
Числовой пример: H=128, n=32: GRU=3·128·161=61 824; LSTM=4·128·161=82 432. GRU на ~25% легче.

5. Backpropagation Through Time (BPTT)

Развёртываем RNN в глубокую сеть по оси времени и применяем обычный backprop:

L = Σₜ Lₜ(yₜ, ŷₜ) ← суммарный loss по всем шагам
Что означают символы
Lсуммарный lossСкалярная потеря по всей последовательности. Градиент ∂L/∂Wₕ вычисляется по всем T шагам суммарно.
Lₜ(yₜ, ŷₜ)loss на шаге tПотеря между целевым yₜ и предсказанием ŷₜ = Wᵧ·hₜ. Например, cross-entropy для языковой модели (следующий токен).
yₜцелевое значение на шаге tGround truth: следующее слово, метка, значение временного ряда. Зависит от задачи (many-to-many, many-to-one).
ŷₜпредсказание на шаге tВыход модели: ŷₜ = Wᵧ·hₜ (или после softmax). Зависит от hₜ, который в свою очередь зависит от Wₕ через все предыдущие шаги.
Σₜсумма по времениВклад каждого шага суммируется. Из-за этого ∂L/∂Wₕ содержит сумму по t — и для каждого t нужен backprop через k≤t шагов назад.
Интуиция: каждый шаг получает свою ошибку, а backprop «разносит» её назад через всю историю к параметрам Wₕ, Wₓ. Поэтому полный BPTT = T · T/2 операций.
Аналогия: проверить всё эссе целиком и выставить общую оценку — но для обратной связи нужно объяснить каждую ошибку, начиная с самого раннего предложения.
Числовой пример: T=5, cross-entropy на каждом шаге: L=L₁+L₂+L₃+L₄+L₅. ∂L/∂Wₕ = сумма 5 вкладов, каждый из которых проходит назад на k шагов.

Градиент по Wₕ

∂L/∂Wₕ = Σₜ ∂Lₜ/∂Wₕ ∂Lₜ/∂Wₕ = Σ_{k=1}^{t} ( ∂Lₜ/∂hₜ · Π_{j=k+1}^{t} ∂hⱼ/∂hⱼ₋₁ ) · ∂hₖ/∂Wₕ
Что означают символы
∂L/∂Wₕсуммарный градиент по рекуррентным весамСкладывается из вкладов всех T шагов. Wₕ используется одинаково на каждом шаге (shared), поэтому все вклады суммируются.
∂Lₜ/∂hₜградиент loss t по hₜ«С чем пришёл backprop от потери шага t?» Это начальный сигнал, который нужно распространить назад на k шагов.
Π_{j=k+1}^{t} ∂hⱼ/∂hⱼ₋₁цепочка ЯкобиановПроизведение (t−k) матриц diag(tanh')·Wₕᵀ. Именно здесь происходит затухание или взрыв при большом (t−k).
∂hₖ/∂Wₕ = hₖ₋₁ᵀвклад шага kНа каждом шаге k, где Wₕ применялся, его вклад в потерю = hₖ₋₁ᵀ (внешнее произведение с incoming gradient).
Σ_{k=1}^{t}сумма по всем прошлым шагамWₕ влияет на hₜ через все шаги k≤t. Для T=100 и каждого t это до 100 слагаемых — O(T²) операций суммарно.
Интуиция: полный BPTT должен пройти от каждого шага t назад ко всем k≤t. Это дорого и страдает от затухающего градиента при больших (t−k).
Аналогия: отследить ошибку в финальном отчёте обратно ко всем решениям, принятым с самого начала проекта.
Числовой пример: T=5, t=4: ∂L₄/∂Wₕ = сумма 4 слагаемых, k=1: произведение 3 Якобианов; k=4: 0 Якобианов (просто hₖ₋₁ᵀ).
Шаг 1.∂hₜ/∂hₜ₋₁ = diag(1-hₜ²) · Wₕᵀ ← Якобиан одного шага
Шаг 2.∂hₜ/∂h₁ = Π_{k=2}^{t} diag(1-hₖ²) · Wₕᵀ ← произведение Якобианов
Шаг 3.∂hₖ/∂Wₕ = hₖ₋₁ᵀ ← вклад на шаге k

Truncated BPTT

Полный BPTT при T=1000 — 1000 шагов назад. На практике: Truncated BPTT — обрезаем backprop до k₂=35-200 шагов, forward pass можно не обрезать.
∂L/∂Wₕ ≈ Σ_{t} Σ_{k=t-k₂}^{t} ( ∂Lₜ/∂hₜ · Π ∂hⱼ/∂hⱼ₋₁ ) · ∂hₖ/∂Wₕ
Что означают символы
≈приближениеTruncated BPTT — приближение полного градиента. Зависимости длиннее k₂ шагов игнорируются в backprop (но не в forward pass).
k₂длина окна усеченияМаксимальное число шагов, на которое разворачивается backprop. Обычно k₂ = 35–200. Компромисс: точность vs скорость и память.
Σ_{k=t-k₂}^{t}ограниченный backpropВместо суммы от k=1 до t — только от t−k₂ до t. Gradient не течёт дальше k₂ шагов назад.
Π ∂hⱼ/∂hⱼ₋₁произведение Якобианов в окнеМаксимум k₂ матриц в произведении (вместо t). Уменьшает и вычислительные затраты, и степень затухания.
Интуиция: обрезаем backprop до k₂ шагов назад — hidden state при этом всё равно несёт историю дальше в forward pass, но gradient туда не доходит.
Аналогия: исправляешь ошибку в главе 50 книги, но изучаешь только главы 45–50 для анализа причин — более ранние не рассматриваешь.
Числовой пример: T=1000, k₂=50: backprop идёт только 50 шагов. Память forward-прохода несёт всю историю, но gradient — только последние 50 позиций.

Gradients для LSTM через время

∂cₜ/∂cₜ₋₁ = fₜ ← только поэлементное умножение, нет Wₕ, нет tanh'!
Что означают символы
∂cₜ/∂cₜ₋₁градиент cell stateНасколько изменение cₜ₋₁ влияет на cₜ. По формуле cₜ = fₜ⊙cₜ₋₁ + iₜ⊙c̃ₜ: производная по cₜ₋₁ = fₜ (поэлементно).
fₜforget gate как «шоссе»Просто поэлементное умножение на fₜ ∈(0,1). Нет матричного произведения Wₕᵀ. Нет Якобиана tanh. Gradient течёт почти без затухания при fₜ≈1.
нет Wₕнет матричного умноженияВ RNN: ∂hₜ/∂hₜ₋₁ = diag(tanh')·Wₕᵀ — нормы могут затухать. В LSTM через c: только fₜ — нет опасного умножения матриц.
нет tanh'нет Якобиана нелинейностиОбновление cₜ аддитивно (не через tanh напрямую). tanh применяется только при переходе c→h, но gradient по c−c пути его не проходит.
Интуиция: вместо произведения T матриц с tanh' (RNN) — произведение T скаляров fₜ∈(0,1). При fₜ≈1 gradient по cₜ пути практически не затухает даже через 100+ шагов.
Аналогия: RNN передаёт послание по испорченному телефону через 50 человек; LSTM — по прямому телефонному кабелю с регулируемым объёмом.
Числовой пример: T=50, fₜ=0.99 (постоянно): 0.99⁵⁰≈0.605 — gradient сохраняет 60%. RNN с нормой 0.8: 0.8⁵⁰≈0.000143 — практически ноль.

Это «шоссе градиента» (gradient highway) — ключевое преимущество LSTM над RNN.

Числовой пример: BPTT для RNN (T=3)

Дано:T=3, H=1, Wₕ=0.5, hₜ=tanh(Wₕ·hₜ₋₁+Wₓ·xₜ), h₀=0, x=[1,1,1], L=h₃²

∂L/∂h₃= 2·h₃ = 1.90
∂L/∂Wₕ= ∂L/∂h₃ · [(1-h₃²)·h₂ + (1-h₃²)·Wₕ·(1-h₂²)·h₁ + …]
Теория глубже

Truncated BPTT ограничивает длину кредитного назначения

Полный BPTT через длинную sequence дорог по памяти и вычислениям. Truncated BPTT режет историю на окна и не передаёт gradient бесконечно далеко назад.

Это ускоряет обучение и стабилизирует память, но вводит bias: модель может не получить прямой gradient для зависимостей длиннее окна. Поэтому длина окна - это архитектурно-оптимизационный компромисс.

  • Короткое окно дешевле, но хуже для long-range dependencies.
  • Длинное окно точнее, но дороже и менее стабильно.
  • Hidden state можно переносить вперёд, даже если gradient назад обрезан.

import numpy as np def sigmoid(x): return 1 / (1 + np.exp(-x)) def lstm_step(x, h_prev, c_prev, W, b): """ x: (n,) h_prev,c_prev: (H,) W: (4H, n+H) b: (4H,) """ combined = np.concatenate([h_prev, x]) # (H+n,) gates = W @ combined + b # (4H,) H = len(h_prev) f = sigmoid(gates[0*H : 1*H]) # forget i = sigmoid(gates[1*H : 2*H]) # input o = sigmoid(gates[3*H : 4*H]) # output c_tilde = np.tanh(gates[2*H : 3*H]) # candidate c = f * c_prev + i * c_tilde # new cell h = o * np.tanh(c) # new hidden return h, c

Как закрепить тему: RNN / LSTM

Этот блок связывает теорию с практическим решением: где применять идею, какие ошибки проверять и какие ручки менять в эксперименте.

Когда применять

  • Используй тему «RNN / LSTM», когда выбираешь inductive bias: локальность, последовательность, attention, residual flow или нормализацию.
  • Архитектура должна соответствовать структуре данных: изображение, текст, последовательность и табличные признаки требуют разных предположений.
  • Перед усложнением архитектуры проверь, что bottleneck действительно в модели, а не в данных, loss или обучении.

Типичные ошибки

  • Копировать архитектуру без учёта размера данных, бюджета памяти и целевой метрики.
  • Сравнивать модели с разным compute budget и делать вывод, что одна архитектура всегда лучше.
  • Игнорировать shape flow: stride, padding, heads, channels и sequence length напрямую определяют память и скорость.

Что тюнить

  • Настраивай depth, width, normalization, dropout, kernel/head sizes и residual structure.
  • Следи за memory footprint: иногда ограничение sequence length или feature map важнее числа параметров.
  • Для Transformer-подобных моделей отдельно проверяй attention mask, positional encoding и scaling.

Упражнения: RNN / LSTM

Проверка темы - понимать память, gates и gradient flow во времени.

[Foundational] Hidden state

Что хранит \(h_t\) в RNN? Решение: сжатое представление прошлого контекста после обработки шага t.

[Foundational] LSTM gates

Назови роли forget, input и output gates. Решение: забыть старую память, записать новую, выбрать что вывести.

[Intermediate] Vanishing gradient

Почему обычная RNN плохо хранит дальний контекст? Решение: градиент многократно умножается на Jacobian и может исчезать или взрываться.

[Intermediate] GRU vs LSTM

Когда GRU может быть удобнее LSTM? Решение: меньше параметров, проще, иногда достаточно для умеренно длинных зависимостей.

[Hard] Bidirectional RNN

Почему bidirectional RNN нельзя напрямую использовать для causal generation? Решение: она смотрит в будущее, а causal inference должен использовать только прошлый контекст.

Проверь себя

Вопросы для самопроверки

  1. Почему простые RNN страдают от vanishing gradient?

    При backpropagation через время (BPTT) градиент умножается на матрицу весов Wₕ на каждом шаге. Если σ_max(Wₕ) < 1 — градиент экспоненциально убывает с длиной последовательности. Это означает, что RNN не может «помнить» зависимости через 10+ шагов.

  2. Как LSTM решает проблему долгосрочных зависимостей?

    LSTM вводит cell state c — прямой «конвейер» памяти с аддитивными, а не мультипликативными обновлениями. Гейты (forget, input, output) обучаются контролировать что сохранять, добавлять и читать. Аддитивное обновление c ← f⊙c + i⊙g предотвращает vanishing gradient.

  3. Зачем в LSTM нужны три гейта (forget, input, output)?

    Forget gate решает, что удалить из памяти (f = σ(...)). Input gate решает, что добавить в память (i⊙g). Output gate фильтрует, что передать как hidden state (o⊙tanh(c)). Разделение позволяет независимо управлять чтением и записью в память.

  4. В чём преимущества трансформеров над LSTM для длинных последовательностей?

    Трансформер обращается к любому токену напрямую через attention за O(1) «шагов», тогда как LSTM должен «протащить» информацию через все промежуточные шаги за O(n). Трансформеры также лучше параллелизируются при обучении. Недостаток: attention O(n²) по памяти и вычислениям.

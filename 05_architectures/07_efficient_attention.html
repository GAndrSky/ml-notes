5.7 Efficient Attention

Когда последовательность становится длинной, классический attention начинает проигрывать по памяти и пропускной способности. Здесь разбираем, как именно FlashAttention, sparse attention и Longformer-подходи меняют экономику вычислений.

Почему важно

O(L²) memory — узкое место стандартного attention при длинных контекстах. FlashAttention, Multi-Query Attention, GQA — способы масштабировать внимание до 1M токенов.

Что узнаешь
  • Почему наивный attention требует O(L²) памяти и где это узкое место
  • FlashAttention — IO-aware алгоритм: tiling, recomputation, без materialization
  • MQA и GQA — как сократить KV-heads без потери качества
  • Linear Attention — как аппроксимировать softmax для O(L) сложности
Нужно знать
  • Multi-Head Attention (5.3)
  • GPU/HBM memory hierarchy
  • Transformer Architecture (5.4)

Где узкое место обычного attention

\text{Attention}(Q,K,V)=softmax(QK^T / sqrt(d))V
Что означают символы
Qматрица запросовQuery: каждый токен спрашивает «что мне нужно?». Размер L×d, где L — длина последовательности.
Kматрица ключейKey: каждый токен объявляет «что я предлагаю?». Скалярное произведение Qᵢ·Kⱼ = сходство i-го и j-го токенов.
Vматрица значенийValue: то, что реально передаётся от токена j к токену i, если attention score высок.
QK^Tматрица scoresL×L матрица попарных сходств — именно она создаёт O(L²) стоимость по памяти и вычислениям.
√dмасштабирующий делительбез деления dot products растут пропорционально √d → softmax насыщается → градиенты исчезают. Деление удерживает дисперсию ~1.
softmaxнормализация весовпреобразует scores в вероятности (0,1), сумма по строке = 1. Каждый токен получает распределение внимания по всем остальным.
Интуиция: QK^T — дорогой шаг: L=4096 даёт матрицу 4096×4096=16.7M чисел на каждую голову. FlashAttention вычисляет тот же результат, не материализуя эту матрицу целиком в памяти.
Аналогия: вместо того чтобы выписать всю таблицу умножения на листе, считаешь нужные ячейки на ходу и сразу суммируешь — ответ тот же, бумаги нужно в разы меньше.
Числовой пример: L=4096, d=64 (размер одной головы). QK^T: 4096²×4 байта=64 МБ только на scores одной головы. При 32 головах — 2 ГБ. Именно поэтому FlashAttention экономит память через tiling, а не хранит всё сразу.
Время: O(n² d) Память: O(n²) Матрица attention weights размера n × n становится главным bottleneck при длинном контексте.
Проблема часто не только во FLOPs, а в том, что веса attention приходится materialize в памяти, и GPU начинает упираться в bandwidth и SRAM/HBM traffic.
Теория глубже

O(n^2) attention ограничивает не только FLOPs, но и память

Обычный attention строит матрицу scores размера sequence length на sequence length для каждой head. При росте контекста в 2 раза эта матрица растёт примерно в 4 раза.

На практике bottleneck часто находится в памяти и bandwidth, а не только в количестве умножений. Поэтому efficient attention методы пытаются уменьшить materialization attention matrix, IO между HBM/SRAM или само число разрешённых связей.

  • Длинный контекст резко увеличивает activation memory.
  • Attention matrix часто дороже, чем кажется по формуле FLOPs.
  • KV-cache переносит часть стоимости в autoregressive inference.

Три стратегии ускорения

FlashAttention
Не меняет саму математику, а меняет порядок вычислений и memory layout.
Sparse / local
Уменьшает число реально считаемых пар токенов.
Longformer / block patterns
Делает контекст структурированным: локальное окно + отдельные глобальные токены.

Почему FlashAttention важен

FlashAttention не “аппроксимирует” softmax-attention. Он вычисляет тот же результат, но по блокам, не сохраняя огромную матрицу attention целиком.

tile Q, K, V по блокам поддерживать running max и running sum для stable softmax сразу накапливать output не materialize полную attention matrix
Это один из лучших примеров, где ускорение пришло не из новой модели, а из лучшей численной и memory-aware реализации.
Теория глубже

FlashAttention ускоряет exact attention через IO-aware вычисление

FlashAttention не меняет математический результат attention. Он меняет способ вычисления: обрабатывает блоки Q/K/V так, чтобы не материализовать огромную attention matrix в медленной памяти.

Ключевая идея - online softmax и tiling. Это уменьшает чтение/запись в HBM и делает attention быстрее и экономнее по памяти, особенно на длинных последовательностях.

  • Результат остаётся exact attention, а не approximation.
  • Экономия идёт через memory IO.
  • Польза растёт с sequence length и размером batch/heads.

Теория глубже

Sparse и approximate attention меняют inductive bias модели

Sparse attention запрещает часть связей между токенами. Это экономит compute, но задаёт гипотезу: не все токены должны напрямую смотреть друг на друга. Approximate attention пытается заменить полную матрицу более дешёвой оценкой.

Такие методы требуют проверки на задаче. Если важная зависимость попала в запрещённую или плохо аппроксимированную область, модель может терять качество даже при хорошей скорости.

  • Local attention хорошо подходит для локального контекста.
  • Global tokens помогают переносить summary по всей sequence.
  • Approximation нужно сравнивать не только по speed, но и по downstream quality.

Как закрепить тему: Efficient Attention

Этот блок связывает теорию с практическим решением: где применять идею, какие ошибки проверять и какие ручки менять в эксперименте.

Когда применять

  • Используй тему «Efficient Attention», когда выбираешь inductive bias: локальность, последовательность, attention, residual flow или нормализацию.
  • Архитектура должна соответствовать структуре данных: изображение, текст, последовательность и табличные признаки требуют разных предположений.
  • Перед усложнением архитектуры проверь, что bottleneck действительно в модели, а не в данных, loss или обучении.

Типичные ошибки

  • Копировать архитектуру без учёта размера данных, бюджета памяти и целевой метрики.
  • Сравнивать модели с разным compute budget и делать вывод, что одна архитектура всегда лучше.
  • Игнорировать shape flow: stride, padding, heads, channels и sequence length напрямую определяют память и скорость.

Что тюнить

  • Настраивай depth, width, normalization, dropout, kernel/head sizes и residual structure.
  • Следи за memory footprint: иногда ограничение sequence length или feature map важнее числа параметров.
  • Для Transformer-подобных моделей отдельно проверяй attention mask, positional encoding и scaling.

Упражнения: Efficient Attention

Проверка темы - понимать, что bottleneck attention часто в памяти/IO, а не только во FLOPs.

[Foundational] Complexity

Сколько attention scores у sequence length 2048? Решение: \(2048^2 \approx 4.19\) млн scores на head.

[Foundational] Почему O(L²)?

Решение: каждый query сравнивается с каждым key.

[Intermediate] FlashAttention

Что оптимизирует FlashAttention? Решение: memory IO: считает attention блоками и не материализует огромную матрицу целиком.

[Intermediate] Sparse attention

Что теряется при sparse pattern? Решение: не все токены видят друг друга напрямую, поэтому inductive bias становится важным.

[Hard] Longformer tradeoff

Почему local+global attention может быть хорошим компромиссом? Решение: локальные окна ловят близкий контекст, global tokens переносят информацию через документ.

Проверь себя

Вопросы для самопроверки

  1. Почему обычный attention O(n²) становится проблемой при длинном контексте?

    Матрица attention n×n при n=16k токенов занимает 16k×16k×2 байт ≈ 512 MB только для одной головы. Для 32 голов × 32 слоя это терабайты. Вычисление тоже O(n²), что делает длинные контексты (100k+ токенов) недоступными без специальных методов.

  2. Что такое FlashAttention и почему он быстрее стандартного?

    FlashAttention реструктурирует вычисление attention для IO-efficiency: вместо материализации всей матрицы n×n в HBM (медленная память GPU), он обрабатывает блоки в SRAM (быстрая). Результат: 2-4× ускорение при той же точности. Это алгоритмическая, а не математическая оптимизация.

  3. Как работает Multi-Query Attention (MQA) и зачем он нужен при инференсе?

    MQA использует одну пару K,V для всех голов (только Q множественный). Это в h раз снижает размер KV-кеша (h — число голов), что критично при инференсе: KV-кеш — основной потребитель памяти. Ускоряет autoregressive decoding при небольшой потере качества.

  4. Что такое Grouped Query Attention (GQA) и чем он отличается от MQA?

    GQA — компромисс: G групп ключей/значений на h голов (1 < G < h). Головы внутри группы делят одну пару KV. При G=1 — MQA, при G=h — стандартный MHA. GQA даёт хороший баланс между скоростью и качеством; используется в LLaMA 2/3, Gemma.

5.7 Efficient Attention

Когда последовательность становится длинной, классический attention начинает проигрывать по памяти и пропускной способности. Здесь разбираем, как именно FlashAttention, sparse attention и Longformer-подходи меняют экономику вычислений.

Где узкое место обычного attention

\text{Attention}(Q,K,V)=softmax(QK^T / sqrt(d))V
Время: O(n² d) Память: O(n²) Матрица attention weights размера n × n становится главным bottleneck при длинном контексте.
Проблема часто не только во FLOPs, а в том, что веса attention приходится materialize в памяти, и GPU начинает упираться в bandwidth и SRAM/HBM traffic.

Три стратегии ускорения

FlashAttention
Не меняет саму математику, а меняет порядок вычислений и memory layout.
Sparse / local
Уменьшает число реально считаемых пар токенов.
Longformer / block patterns
Делает контекст структурированным: локальное окно + отдельные глобальные токены.

Почему FlashAttention важен

FlashAttention не “аппроксимирует” softmax-attention. Он вычисляет тот же результат, но по блокам, не сохраняя огромную матрицу attention целиком.

tile Q, K, V по блокам поддерживать running max и running sum для stable softmax сразу накапливать output не materialize полную attention matrix
Это один из лучших примеров, где ускорение пришло не из новой модели, а из лучшей численной и memory-aware реализации.

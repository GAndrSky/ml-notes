5.10 Segmentation

Segmentation predicts a label for each pixel, not just one label for the whole image or one box for an object.

TL;DR

Segmentation is dense prediction. Semantic segmentation assigns a class to every pixel. Instance segmentation separates object identities. U-Net remains the core mental model: downsample for context, upsample for resolution, use skip connections to preserve detail.

Why this matters

Segmentation is used when exact shape matters: tumors, roads, lanes, product defects, organs, satellite parcels, document layout, clothing, and image editing. The output is not "there is a dog"; it is "these exact pixels are dog".

U-Net architecture

U-Net has an encoder-decoder shape. The encoder compresses spatial resolution and extracts context. The decoder restores resolution. Skip connections copy high-resolution features from encoder to decoder.

decoder_l = upsample(decoder_{l+1}) concat encoder_l
What the symbols mean
encoder_llocal detailHigh-resolution features from encoder level l.
decoder_lreconstructed featureDecoder representation at the same spatial scale.
concatchannel mergeStacks features along channel dimension, preserving detail and context.
upsampleresolution increaseExpands lower-resolution features back toward mask size.
Intuition: deep layers know what is present; skip connections recover exactly where it is.
Analogy: combine a city map for global layout with a street-level photo for precise boundaries.

Dice coefficient and Dice loss

Dice is common for imbalanced masks because background pixels often dominate cross-entropy.

Dice(p,y) = (2 sum_i p_i y_i + eps) / (sum_i p_i + sum_i y_i + eps)
What the symbols mean
p_ipredicted mask valueProbability that pixel i belongs to the foreground/class.
y_itrue mask valueGround truth label for pixel i, usually 0 or 1.
sum p_i y_isoft overlapLarge when predictions align with the true mask.
epsilonstability termPrevents division by zero when masks are empty.
Intuition: Dice rewards overlap relative to total predicted and true mask size.
Analogy: two transparent cutouts are good if they cover the same shape.
Numeric example: if overlap sum is 30, predicted area is 50, true area is 40, Dice = 60/90 = 0.667.

Runnable code: Dice score

# What it does: compute binary Dice score for segmentation masks
# Input shapes: pred=(H,W), target=(H,W)
# Output shape: scalar
import numpy as np

def dice_score(pred, target, eps=1e-7):
    pred = pred.astype(float)
    target = target.astype(float)
    intersection = np.sum(pred * target)
    return (2 * intersection + eps) / (np.sum(pred) + np.sum(target) + eps)

pred = np.array([[1,1,0], [0,1,0], [0,0,0]])
target = np.array([[1,0,0], [0,1,1], [0,0,0]])
print(dice_score(pred, target))
# Gotchas: threshold probabilities consistently; handle empty masks explicitly.

Common mistakes

  • Using image-level labels when pixel-level masks are required.
  • Resizing masks with bilinear interpolation, which creates invalid class IDs.
  • Reporting only pixel accuracy on imbalanced masks.
  • Forgetting that data augmentation must transform image and mask identically.

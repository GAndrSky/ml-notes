2.15a Kernel PCA, ICA и Autoencoders

После обычного PCA логично задать вопрос: что делать, если структура данных нелинейна, если источники смешаны или если мы хотим учить представление прямо из данных? Здесь как раз встречаются Kernel PCA, ICA и autoencoders.

Почему важно

Kernel PCA, ICA и автоэнкодеры — нелинейные расширения классической PCA. Каждый ищет скрытую структуру по-разному: через kernel trick, независимость компонент или бутылочное горло сети.

Теория глубже

Autoencoder полезен только если bottleneck заставляет учить структуру

Если autoencoder слишком мощный, он может почти скопировать вход и не выучить полезное представление. Bottleneck, noise, sparsity, weight decay и архитектурные ограничения нужны, чтобы модель сохраняла не каждую деталь, а устойчивую структуру данных.

В отличие от PCA, autoencoder оптимизируется gradient descent и может быть нелинейным. Это даёт гибкость, но добавляет риск нестабильного обучения и зависимости от архитектуры.

  • Undercomplete bottleneck ограничивает размер latent vector.
  • Denoising autoencoder учится восстанавливать чистый сигнал из зашумлённого.
  • Variational autoencoder добавляет вероятностную структуру latent space.
Теория глубже

Kernel PCA делает PCA после неявного нелинейного lift

Обычная PCA ищет линейные компоненты в исходных признаках. Kernel PCA сначала неявно переносит точки в feature space через kernel, а затем делает PCA по kernel matrix. Это позволяет распрямлять структуры, которые в исходном пространстве выглядят изогнутыми.

Цена подхода - масштабируемость и интерпретируемость. Kernel matrix растёт как n на n, а компоненты сложнее объяснять через исходные признаки.

  • RBF kernel полезен для гладкой нелинейной структуры.
  • Polynomial kernel полезен для interactions признаков.
  • gamma в kernel PCA влияет на то, насколько локальной становится геометрия.
Теория глубже

Kernel PCA, ICA и Autoencoder отвечают на разные вопросы

Kernel PCA спрашивает: можно ли найти нелинейное пространство, где вариативность описывается проще. ICA спрашивает: можно ли разложить наблюдения на статистически независимые источники. Autoencoder спрашивает: какой bottleneck позволяет восстановить вход с минимальной ошибкой.

Эти методы легко спутать, потому что все дают low-dimensional representation. Но критерий успеха у них разный, поэтому один метод не является прямой заменой другого.

  • Kernel PCA - геометрия и нелинейные компоненты.
  • ICA - независимые скрытые источники.
  • Autoencoder - обучаемое сжатие через reconstruction objective.

Kernel PCA

\tilde{K} = K - 1K - K1 + 1K1
Что означают символы
K̃центрированная kernel-матрицаРезультат центрирования: K̃ = K − 1K − K1 + 1K1. Необходима для корректного Kernel PCA — без центрирования первая компонента уйдёт в «среднее», а не в «разброс».
Kисходная kernel-матрица (n×n)K_{ij} = K(x_i, x_j) — попарные сходства всех объектов. Заменяет матрицу ковариации из обычного PCA.
1матрица из 1/n (центрирующая матрица)1 = (1/n)·𝟙𝟙^T, где 𝟙 — вектор единиц. Вычитание 1K центрирует строки kernel-матрицы, вычитание K1 — столбцы.
1K1двойное центрированиеДобавляется, чтобы не вычесть «среднее» дважды. Итог: K̃ — это kernel-матрица с вычтенными средними по строкам, столбцам и всей матрице.
Интуиция: Kernel PCA делает PCA в скрытом feature space φ. Центрирование K̃ — это обязательный аналог вычитания среднего в обычном PCA. После этого eigendecomposition K̃ даёт нелинейные главные компоненты.
Аналогия: PCA нормирует данные (вычитает среднее), тогда Kernel PCA нормирует матрицу сходств (центрирует K). Идея та же — уйти от «смещения» к «разбросу».
Числовой пример: n=3, K=[1,0.8,0.2; 0.8,1,0.3; 0.2,0.3,1]. Центрирование: вычесть среднее каждой строки, каждого столбца, добавить общее среднее — получим K̃ для eigendecomposition.
Теория по месту

Нелинейное сжатие ищет скрытую геометрию

Kernel PCA делает PCA в скрытом пространстве похожестей, ICA ищет независимые источники, autoencoder учит сжатое представление через реконструкцию.

Все три подхода отвечают на разный вопрос: где главные оси variance, где независимые факторы, где компактный код, из которого можно восстановить вход.

Обычный PCA ищет главные компоненты в исходном пространстве. Kernel PCA сначала строит similarity matrix, центрирует её и делает eigendecomposition уже в kernel space. Это позволяет развернуть нелинейные многообразия.

Если данные похожи на “кольца”, “спирали” или “дуги”, а линейный PCA теряет структуру, kernel PCA часто помогает увидеть полезную нелинейную геометрию.

ICA

x = A s наблюдение = смесь скрытых источников ICA ищет W такое, что: s_hat = W x компоненты s_hat максимально независимы
Как не ошибиться

Реконструкция не равна пониманию

Autoencoder может хорошо восстанавливать данные и при этом учить representation, который не полезен для нужной задачи.

Kernel и autoencoder методы чувствительны к гиперпараметрам. Красивый 2D plot не гарантирует надежную структуру в исходных данных.

ICA не пытается поймать дисперсию как PCA. Оно ищет представление, где компоненты статистически независимы и как можно менее гауссовы. Это важно для blind source separation.

Если источники гауссовы или плохо различаются по higher-order statistics, ICA теряет почву под ногами. Это не универсальная замена PCA.

Autoencoder

z = f_θ(x), x_hat = g_φ(z), min ||x - x_hat||²
Что означают символы
z = f_θ(x)encoder — сжатие в латентное пространствоf_θ — нейронная сеть с параметрами θ. Берёт объект x (например, изображение) и выдаёт вектор z меньшей размерности — «сжатое представление».
x̂ = g_φ(z)decoder — реконструкция из latent codeg_φ — нейронная сеть с параметрами φ. Берёт latent code z и восстанавливает x̂ — попытку воспроизвести исходный объект x.
min ||x − x̂||²reconstruction loss (MSE)Цель обучения: параметры θ и φ подбираются так, чтобы реконструкция x̂ как можно меньше отличалась от оригинала x.
zlatent code / bottleneckСжатое представление размерности d_z ≪ d_x. Bottleneck заставляет автоэнкодер учить содержательные признаки: нельзя просто скопировать x — нужно «понять» его структуру.
Интуиция: Autoencoder учится «сжать-и-восстановить». Encoder находит компактное z, decoder из z строит x̂ ≈ x. Всё, что нельзя восстановить из z, теряется — bottleneck вынуждает выучивать главное.
Аналогия: Пересказ книги в 3 предложениях (encoder) и восстановление из пересказа (decoder). Хорошие 3 предложения — «хорошее z». Качество восстановления — reconstruction loss.
Числовой пример: Вход: изображение 784px → encoder → z ∈ ℝ³² → decoder → x̂ 784px. MSE(x, x̂) обучает encoder запоминать самое важное в 32 числах.

Если encoder и decoder достаточно гибкие, autoencoder может учить более сложное нелинейное сжатие, чем PCA. Линейный autoencoder с MSE и без лишних трюков, кстати, тесно связан с PCA.

Важный мост к deep learning: autoencoder — это уже не просто метод уменьшения размерности, а полноценная обучаемая архитектура представлений.

Теоретический слой

Kernel PCA, ICA и Autoencoders как разные latent views

Когда данные лежат на нелинейном manifold-е, линейная PCA может требовать слишком много компонент. Kernel PCA, ICA и autoencoders строят разные latent representations.

Эти методы решают разные задачи: Kernel PCA сохраняет kernel-геометрию, ICA ищет независимые источники, autoencoder учит сжатие через reconstruction.

Геометрическая интуиция

  • Kernel PCA делает PCA в неявном нелинейном feature space.
  • Autoencoder сжимает вход в bottleneck и восстанавливает через decoder.
  • ICA поворачивает пространство к независимым источникам, а не просто некоррелированным осям.

Вероятностная интерпретация

  • ICA предполагает latent sources и статистическую независимость.
  • Обычный autoencoder не задаёт полноценное распределение, в отличие от VAE.
  • Reconstruction error может быть proxy для anomaly, но не calibrated probability.

Оптимизационный смысл

  • Kernel PCA ограничен размером kernel matrix.
  • Autoencoder чувствителен к архитектуре, bottleneck, noise и регуляризации.
  • Слишком мощный autoencoder может выучить почти identity mapping.

Практика и типичные ошибки

  • Выбирай метод по цели: visualization, compression, denoising, anomaly detection или feature learning.
  • Проверяй downstream metric, а не только reconstruction.
  • Для autoencoder анализируй latent space и validation reconstruction.

Мини-пример: Если точки лежат на окружности, linear PCA видит две координаты, а kernel PCA может раскрыть нелинейную структуру.

Как закрепить тему: Kernel PCA, ICA, Autoencoders

Этот блок связывает теорию с практическим решением: где применять идею, какие ошибки проверять и какие ручки менять в эксперименте.

Когда применять

  • Используй тему «Kernel PCA, ICA, Autoencoders» как практический инструмент для табличных данных, baseline-моделей и быстрых проверок гипотез.
  • Классические модели особенно полезны, когда важны скорость, интерпретируемость, небольшой датасет или строгая валидация.
  • Перед нейросетью проверь, какой результат даёт простой baseline: он задаёт нижнюю планку качества.

Типичные ошибки

  • Подбирать модель до постановки метрики: без правильной метрики можно оптимизировать не ту задачу.
  • Допускать leakage: scaling, encoding, feature selection и tuning должны обучаться только на train внутри pipeline.
  • Сравнивать модели без одинакового split и одинакового preprocessing.

Что тюнить

  • Начни с данных: пропуски, выбросы, encoding, scaling и баланс классов часто важнее выбора алгоритма.
  • Потом меняй сложность модели: регуляризацию, глубину, число соседей, число деревьев или learning rate в зависимости от темы.
  • Фиксируй validation protocol до тюнинга, иначе улучшение может быть подгонкой под конкретный split.

Упражнения: Kernel PCA, ICA, Autoencoders

Цель - различать линейное сжатие, нелинейное manifold learning и learned reconstruction.

[Foundational] Kernel PCA intuition

Когда Kernel PCA полезнее обычного PCA? Решение: когда структура данных нелинейна и линейные компоненты плохо раскрывают manifold.

[Foundational] ICA goal

Чем ICA отличается от PCA? Решение: PCA ищет некоррелированные направления с большой variance; ICA ищет статистически независимые источники.

[Intermediate] Autoencoder bottleneck

Почему bottleneck заставляет модель сжимать информацию? Решение: latent dimension меньше входа, поэтому сеть должна хранить только структуру, нужную для восстановления.

[Intermediate] Reconstruction loss

Как понять, что autoencoder просто запоминает данные? Решение: train reconstruction низкий, validation хуже; слишком большая latent dimension или модель слишком мощная.

[Hard] Nonlinear PCA

Почему autoencoder можно рассматривать как нелинейный PCA, но не как прямую замену PCA? Решение: он обучает нелинейное кодирование, но зависит от архитектуры, loss и оптимизации.

Проверь себя

Вопросы для самопроверки

  1. Чем Kernel PCA принципиально отличается от обычного PCA?

    Обычный PCA ищет линейные направления максимальной дисперсии. Kernel PCA неявно отображает данные в высокомерное пространство через ядро и применяет там PCA — получая нелинейные главные компоненты. Это позволяет «расплетать» нелинейные структуры, невидимые в исходном пространстве.

  2. Что ICA ищет в данных и чем это отличается от PCA?

    PCA ищет некоррелированные компоненты максимальной дисперсии. ICA (Independent Component Analysis) ищет статистически независимые компоненты, минимизируя взаимную информацию. ICA применяется для разделения смешанных сигналов (cocktail party problem), где PCA бессилен.

  3. Почему автоэнкодер является нелинейным аналогом PCA?

    Линейный автоэнкодер с одним скрытым слоем и MSE-потерей обучает то же подпространство, что и PCA. При добавлении нелинейных активаций он может моделировать нелинейные многообразия (manifolds). Латентное пространство — это аналог главных компонент, но нелинейный.

  4. Что такое variational autoencoder и зачем нужна регуляризация латентного пространства?

    VAE вместо точечного латентного вектора предсказывает распределение N(μ, σ²), из которого сэмплирует z. KL-дивергенция к N(0,1) регуляризует латентное пространство: делает его непрерывным и плотно заполненным, что позволяет сэмплировать z и генерировать новые данные.

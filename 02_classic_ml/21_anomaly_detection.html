2.21 Anomaly Detection

Anomaly detection is not just classification with rare positives. Often you have few labels, changing behavior, asymmetric costs, and a business threshold that matters more than the raw model score.

TL;DR

Anomaly detection learns a score for how unusual an object is, then converts that score into an action using a threshold. The hard parts are defining "normal", handling drift, choosing threshold, and evaluating with rare or delayed labels.

Why this matters

Anomaly detection appears in fraud, monitoring, cybersecurity, manufacturing defects, medical screening, data-quality alerts, and model monitoring. In all of these, false positives waste attention and false negatives can be expensive.

Use anomaly detection when
  • Positive labels are scarce or delayed.
  • You mostly know what normal looks like.
  • You need ranking or alerting rather than direct classification.
  • The anomaly type may change over time.
Be careful when
  • Normal behavior is highly multi-modal.
  • Outliers in training data are actually valid rare subgroups.
  • Threshold costs are not specified.
  • Data distribution drifts every week.

Anomaly score and threshold

is_anomaly(x) = 1 if score(x) >= tau else 0
What the symbols mean
xobjectTransaction, sensor reading, user session, image, log event, or data row.
score(x)anomaly scoreHigher means more unusual according to the model or rule.
tauthresholdBusiness decision boundary between normal and alert/action.
1/0decision1 means raise anomaly flag; 0 means treat as normal.
Intuition: the model ranks suspiciousness; the threshold turns ranking into an action.
Analogy: a smoke detector has a sensitivity knob; the same smoke score can trigger or not trigger depending on threshold.

Threshold choice should be based on cost, capacity, or target alert rate. For example, a fraud team may only review the top 0.5% riskiest transactions.

Isolation Forest

Isolation Forest isolates anomalies by randomly splitting features. Unusual points tend to be separated with fewer splits because they sit in sparse regions.

score(x) = 2 ^ (- E[h(x)] / c(n))
What the symbols mean
h(x)path lengthNumber of splits needed to isolate point x in one tree.
E[h(x)]average path lengthMean path length across many random isolation trees.
c(n)normalizerExpected path length for unsuccessful search in a sample of size n.
scoreanomaly scoreShorter paths produce larger anomaly scores.
Intuition: isolated points are easy to separate from the crowd.
Numeric example: if a point needs only 3 splits while most need 12, it is likely unusual.

Good for

  • Tabular data with mixed feature scales after preprocessing.
  • Fast unsupervised baseline.
  • High-dimensional sparse-ish anomalies.

Limitations

  • Score calibration is not a probability.
  • Can flag rare but valid subgroups.
  • Threshold still needs business tuning.

One-Class SVM

One-Class SVM tries to separate normal data from the origin in a feature space. Points outside the learned boundary are treated as anomalies.

f(x) = sign(sum_i alpha_i K(x_i, x) - rho)
What the symbols mean
KkernelSimilarity function, often RBF for nonlinear boundaries.
alpha_isupport weightsLearned coefficients for support vectors.
rhooffsetBoundary threshold in feature space.
f(x)decisionPositive for inlier, negative for outlier in common implementations.
Intuition: learn the region occupied by normal data; anything outside is suspicious.
Analogy: draw a fence around known safe behavior and flag points outside the fence.

One-Class SVM can work well on moderate data, but it is sensitive to feature scaling and kernel parameters.

Reconstruction-based anomaly detection

Autoencoders or PCA can learn to reconstruct normal data. If an object reconstructs poorly, it may not follow the normal structure.

score(x) = ||x - decoder(encoder(x))||^2
What the symbols mean
encodercompressorMaps input into a lower-dimensional latent representation.
decoderreconstructorMaps latent representation back to input space.
x - x_hatreconstruction errorDifference between original object and reconstructed object.
||.||^2squared normLarge errors get penalized more strongly.
Intuition: if the model only learned normal patterns, abnormal patterns are hard to reconstruct.
Analogy: a stencil for normal shapes leaves obvious gaps when used on an unusual shape.

Minimal sklearn baseline

# What it does: trains an Isolation Forest and selects top-risk anomalies.
# Input shapes: X_train=(n_train,d), X_test=(n_test,d)
# Output shapes: scores=(n_test,), flags=(n_test,)
import numpy as np
from sklearn.ensemble import IsolationForest
from sklearn.preprocessing import StandardScaler
from sklearn.pipeline import make_pipeline

rng = np.random.default_rng(0)
normal = rng.normal(size=(500, 2))
outliers = rng.normal(loc=5.0, scale=0.4, size=(20, 2))
X = np.vstack([normal, outliers])

model = make_pipeline(
    StandardScaler(),
    IsolationForest(n_estimators=200, contamination=0.04, random_state=0),
)
model.fit(X)

# sklearn decision_function: lower means more abnormal, so negate it.
scores = -model.decision_function(X)
tau = np.quantile(scores, 0.96)
flags = scores >= tau

print("flagged rate:", flags.mean())
print("top scores:", np.sort(scores)[-5:])

# Gotchas:
# 1. contamination is not the true anomaly rate unless you know it.
# 2. Standardize features when distance or split behavior depends on scale.

Evaluation

If labels exist, treat anomaly scores as a ranking and use PR-AUC, recall at fixed false-positive rate, precision at top K, or cost-weighted metrics. If labels do not exist, evaluate by review yield, alert volume, stability, and manual inspection.

precision@K = true_anomalies_in_top_K / K
What the symbols mean
Kreview capacityNumber of highest-score objects a team can inspect.
top_Khighest scoresObjects ranked as most suspicious by the model.
precisionreview yieldFraction of reviewed alerts that are real anomalies.
Intuition: if humans review only 100 alerts, the top 100 quality matters more than average score quality.
Numeric example: if 37 of top 100 alerts are real anomalies, precision@100 is 0.37.

Typical mistakes

Modeling mistakes
  • Interpreting anomaly score as calibrated probability.
  • Training on data contaminated by known anomalies without checking.
  • Ignoring rare but valid populations.
  • Using default threshold without business cost.
Production mistakes
  • No alert budget or review workflow.
  • No drift monitoring of score distribution.
  • No feedback loop from reviewed alerts.
  • No segment-level analysis of false positives.

Intermediate

Compare methods

When would you choose Isolation Forest over One-Class SVM?

Show solution

Use Isolation Forest as a scalable tabular baseline with fewer kernel-sensitivity issues. Use One-Class SVM for smaller, well-scaled data where a smooth boundary is useful.

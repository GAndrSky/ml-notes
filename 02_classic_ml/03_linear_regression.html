Пошагово Градиентный спуск на числах — y = wx, три точки
Шаг 1 из
Шаг 1

Начальные значения

Данные: три точки (x, y). Модель: ŷ = w·x (без смещения). Старт: w = 0.

x = [1, 2, 3], y_true = [2, 4, 6] w = 0, lr = 0.1
Шаг 2

Forward: предсказания ŷ

Подставляем текущий w в модель.

ŷ = w · x = 0 × [1,2,3] = [0, 0, 0]
Предсказания нулевые — модель ещё ничему не научилась.
Шаг 3

Loss: MSE

MSE = среднее квадратичное отклонение предсказания от правильного ответа.

L = (1/n) Σ(ŷᵢ - yᵢ)² = ((0-2)² + (0-4)² + (0-6)²) / 3 = (4 + 16 + 36) / 3 = 18.67
L = 18.67 — высокий loss, нужно обновить w.
Шаг 4

Gradient ∂L/∂w

Производная MSE по w. Правило цепочки: ∂L/∂w = (2/n) Σ (ŷᵢ - yᵢ) · xᵢ.

∂L/∂w = (2/3) × [(0-2)·1 + (0-4)·2 + (0-6)·3] = (2/3) × [-2 - 8 - 18] = (2/3) × (-28) = -18.67
Отрицательный градиент → нужно увеличить w.
Шаг 5

Обновление w

Шаг градиентного спуска: w ← w − lr · ∂L/∂w.

w_new = 0 − 0.1 × (−18.67) = 0 + 1.867 = 1.867
Шаг 6

Проверка: новые предсказания

Считаем ŷ с обновлённым w = 1.867.

ŷ = 1.867 × [1, 2, 3] = [1.87, 3.73, 5.60] L = ((1.87-2)² + (3.73-4)² + (5.60-6)²) / 3 ≈ (0.017 + 0.073 + 0.160) / 3 ≈ 0.083
Loss упал с 18.67 → 0.083 за один шаг. Правильный ответ: y = 2x, w → 2.

Что такое линейная регрессия

Линейная регрессия — это одна из самых базовых и самых важных моделей в ML. Она пытается предсказывать числовую величину как линейную комбинацию признаков.

\hat{y} = w_1 x_1 + w_2 x_2 + \dots + w_d x_d + b
Что означают символы
ŷпредсказание моделиЧисловой ответ модели: взвешенная сумма всех d признаков плюс свободный член b.
w_jвес j-го признакаКоэффициент при x_j: при росте x_j на 1 прогноз ŷ изменяется на w_j.
x_jj-й признак объектаЧисловое значение j-й характеристики объекта (j = 1, ..., d).
bсвободный член (bias)Базовый уровень предсказания при x_1=...=x_d=0.
Интуиция: Предсказание — взвешенная сумма признаков. Каждый вес w_j показывает "цену" единицы признака x_j — независимо от других признаков.
Аналогия: Как смета: итоговая цена = площадь·цена_м² + этаж·доплата_этаж + … + базовый тариф.
Числовой пример: d=2, w=[0.5,2], x=[50,3], b=10 → ŷ=25+6+10=41.
Теория по месту

Линейная регрессия как проекция

Линейная регрессия ищет плоскость, которая проходит через облако данных так, чтобы средний квадрат вертикальных промахов был минимальным. В геометрическом смысле модель проектирует вектор ответов на пространство, натянутое признаками.

Коэффициент не просто число в формуле. Он показывает изменение прогноза при изменении признака на одну единицу при прочих равных, поэтому масштаб признаков напрямую влияет на читаемость весов.

Здесь:

  • x_1, ..., x_d — признаки объекта,
  • w_1, ..., w_d — веса модели,
  • b — bias / intercept,
  • ŷ — предсказание модели.
Интуиция. Линейная регрессия — это попытка объяснить целевую переменную как взвешенную сумму факторов. Например, цена квартиры может зависеть от площади, района, этажа, состояния ремонта и так далее. Модель учится, какой вклад даёт каждый фактор.
Несмотря на простоту, линейная регрессия — это не “детская” модель. Это фундамент для понимания: оптимизации, лосса, градиентов, регуляризации, интерпретации коэффициентов и bias-variance tradeoff.
Теория глубже

Предсказание и статистический вывод — разные режимы

Для prediction линейная регрессия может быть полезной даже когда реальные связи не идеально линейны: она даёт стабильный baseline и хорошо работает при адекватных признаках. Для inference требования жёстче: важны независимость ошибок, отсутствие сильной эндогенности, разумная форма шума и корректная интерпретация коэффициентов.

Если цель — предсказывать, главный критерий validation quality. Если цель — объяснять причинный эффект признака, одной линейной регрессии недостаточно: нужны дизайн исследования, контроль confounders и проверка assumptions.

  • Linearity: средний ответ примерно выражается линейной комбинацией признаков.
  • Exogeneity: ошибка не должна систематически зависеть от признаков.
  • Homoscedasticity: масштаб шума не должен резко меняться по диапазону прогнозов.

Форма модели: от одной переменной к многомерному случаю

Одна переменная

\hat{y} = wx + b
Что означают символы
ŷпредсказание моделиЧисловой ответ на одном объекте: wx + b — прямая линия в пространстве x.
wнаклон прямойКоэффициент при x: на сколько меняется ŷ при увеличении x на 1 единицу.
xзначение признакаЕдинственный числовой признак объекта (одномерный случай).
bсвободный член (intercept)Сдвиг прямой: значение ŷ при x=0.
Интуиция: Это уравнение прямой: w — наклон, b — точка пересечения с осью y. Задача — найти прямую, ближайшую к облаку точек.
Аналогия: Цена такси: b — посадочный тариф, w — цена за километр, x — расстояние.
Числовой пример: w=2, b=1, x=3 → ŷ=2·3+1=7.

Геометрически это просто прямая на плоскости. Модель подбирает наклон w и сдвиг b, чтобы линия как можно лучше описывала зависимость между x и y.

Много признаков

\hat{y} = \mathbf{w}^T \mathbf{x} + b
Что означают символы
ŷпредсказание моделиСкаляр — число, которое модель выдаёт для данного объекта. Это и есть «ответ» линейной регрессии.
wвектор весов признаковКаждый элемент w_j показывает, насколько j-й признак влияет на ответ. Большой |w_j| — признак важен, знак задаёт направление.
xвектор признаков объектаd числовых характеристик одного наблюдения. Например, для квартиры: площадь, этаж, число комнат, …
bсвободный член (bias)Сдвиг: базовый уровень предсказания при x=0. Без него гиперплоскость вынуждена проходить через начало координат.
Интуиция: w^T x — скалярное произведение, «взвешенная сумма» признаков. Добавление b позволяет смещать гиперплоскость в любое место пространства.
Аналогия: Как смета: итоговая цена = стоимость₁·кол-во₁ + стоимость₂·кол-во₂ + … + базовый тариф. Веса w — тарифы, признаки x — количества, b — базовый тариф.
Числовой пример: w=[0.5, 2], x=[50, 3], b=10. ŷ = 0.5·50 + 2·3 + 10 = 25+6+10 = 41 (например, тысяч рублей аренды).

В векторной форме линейная регрессия описывает гиперплоскость в пространстве признаков. В 2D это линия, в 3D — плоскость, дальше — гиперплоскость.

\mathbf{x}^{(i)} \in \mathbb{R}^d,\quad \hat{y}^{(i)} = \mathbf{w}^T \mathbf{x}^{(i)} + b
Что означают символы
x^(i)i-й объект выборкиВектор в R^d — d признаков одного наблюдения. Верхний индекс (i) нумерует объект, нижний — признак.
dразмерность признакового пространстваЧисло признаков в каждом объекте. Например, d=10 означает 10 числовых колонок в таблице данных.
ŷ^(i)предсказание для i-го объектаРезультат применения модели к i-му объекту: скалярное произведение w^T x^(i) + b.
wвектор весовd коэффициентов, по одному на каждый признак. Настраиваются в процессе обучения.
bсвободный членСдвиг — позволяет предсказанию не равняться нулю при x=0.
Интуиция: Каждый объект — точка в d-мерном пространстве. Модель строит гиперплоскость; w задаёт её наклон, b — положение. Предсказание — высота этой плоскости над точкой x^(i).
Аналогия: Как оценка квартиры: x^(i)=[50м², 3 этаж, 2 комнаты], w=[500, -50, 100] — тарифы за м², этаж, комнату. Итог: линейная цена без торга.
Числовой пример: x^(i)=[2,3], w=[1,2], b=0.5. ŷ^(i) = 1·2 + 2·3 + 0.5 = 2+6+0.5 = 8.5.
Важно: “линейная” регрессия означает линейность по параметрам w и b. Даже если в признаки заранее добавить x^2, x^3, log x, модель всё ещё будет линейной регрессией, если остаётся линейной по коэффициентам.

Что именно мы минимизируем

Чтобы понять, какие веса хорошие, нужна функция ошибки. Для линейной регрессии классический выбор — MSE, mean squared error.

Ошибка на одном объекте

e^{(i)} = \hat{y}^{(i)} - y^{(i)}
Что означают символы
e^(i)ошибка на i-м объектеSigned residual: положительный — прогноз завышен, отрицательный — занижен.
ŷ^(i)прогноз моделиПредсказание линейной регрессии для i-го объекта.
y^(i)истинный ответПравильное числовое значение из обучающей выборки (target).
Интуиция: Ошибка со знаком: e^(i) > 0 если модель завысила прогноз, e^(i) < 0 если занизила.
Аналогия: Как разница между обещанным и реальным: +2 означает "сказали 8, оказалось 10".
Числовой пример: ŷ^(1)=8, y^(1)=10 → e^(1)=8−10=−2 (недооценка). ŷ^(2)=12, y^(2)=10 → e^(2)=+2 (переоценка).

Squared error

(e^{(i)})^2 = (\hat{y}^{(i)} - y^{(i)})^2
Что означают символы
(e^(i))^2квадрат ошибкиПоложительная мера отклонения: всегда ≥ 0, большие ошибки штрафуются непропорционально.
ŷ^(i)прогноз моделиПредсказание для i-го объекта: w^T x^(i) + b.
y^(i)истинный ответПравильное значение из обучающей выборки.
(·)^2возведение в квадратДелает ошибки положительными и наказывает крупные промахи сильнее: ошибка 2 → штраф 4, ошибка 4 → штраф 16.
Интуиция: Квадрат нейтрализует знак ошибки и нелинейно штрафует крупные промахи — именно это делает MSE чувствительным к выбросам.
Аналогия: Как площадь квадрата, построенного на промахе: мелкий промах — маленький квадрат, крупный — огромный.
Числовой пример: ŷ^(1)=8, y^(1)=10 → ошибка=−2, squared error=4. ŷ^(2)=4, y^(2)=10 → ошибка=−6, squared error=36.

MSE на всём train

L(\mathbf{w}, b) = \frac{1}{n} \sum_{i=1}^{n} (\hat{y}^{(i)} - y^{(i)})^2
Что означают символы
L(w,b)MSE loss — целевая функцияMean Squared Error по всей обучающей выборке — то, что минимизируется при обучении регрессии.
1/nнормировка на размер выборкиДелим на n, чтобы loss не зависел от размера датасета — сравниваем модели независимо от n.
Σ_{i=1}^nсумма по всем объектамСуммируем квадрат ошибки по каждому из n обучающих объектов.
ŷ^(i)предсказание модели для объекта iВывод модели w^T x^(i) + b — то, что модель предсказала для i-го объекта.
y^(i)истинное значение для объекта iПравильный ответ из обучающей выборки для i-го объекта.
(ŷ−y)²квадрат ошибкиШтрафует за отклонение: большие ошибки наказываются непропорционально сильно, квадрат всегда ≥ 0.
Интуиция: MSE измеряет среднюю «площадь» квадратных отклонений — чем меньше ошибается модель на каждом объекте, тем меньше loss.
Аналогия: Среднее расстояние в квадрате от прогнозов до реальных значений — хочем, чтобы точки лежали как можно ближе к линии.
Числовой пример: n=3, ошибки: 1, -2, 3 → MSE = (1²+4+9)/3 = 14/3 ≈ 4.67.

Подставляя модель:

L(\mathbf{w}, b) = \frac{1}{n} \sum_{i=1}^{n} (\mathbf{w}^T \mathbf{x}^{(i)} + b - y^{(i)})^2
Что означают символы
L(w,b)MSE loss — функция потерьСреднеквадратичная ошибка по всей выборке. Чем меньше L, тем лучше модель описывает данные.
1/nнормировка на размер выборкиУсредняет сумму квадратов ошибок по n объектам, делая loss независимым от размера выборки.
Σ_{i=1}^nсумма по всем объектамСуммируем квадраты ошибок по каждому из n объектов обучающей выборки.
w^T x^(i) + bпрогноз в развёрнутом видеЯвная подстановка модели вместо ŷ. Показывает, от каких параметров (w, b) зависит loss — именно по ним берётся градиент.
y^(i)истинный ответ i-го объектаМетка из обучающей выборки, которую модель пытается воспроизвести. Разность (прогноз − y^(i)) — это residual.
(·)²квадрат ошибкиДелает все ошибки положительными и штрафует большие промахи сильнее: ошибка 2 даёт штраф 4, ошибка 4 — штраф 16.
Интуиция: Это та же MSE, что и выше, но с явно подставленной моделью. Теперь видно: L зависит от w и b — именно по ним будем дифференцировать при gradient descent.
Аналогия: Дистанция стрелка: каждый выстрел промахивается на (прогноз − цель), промахи возводим в квадрат и усредняем. Задача — настроить прицел (w, b) так, чтобы среднее отклонение было минимальным.
Числовой пример: n=2, объекты: y^(1)=10, прогноз=8 → ошибка²=4; y^(2)=5, прогноз=6 → ошибка²=1. L = (4+1)/2 = 2.5.
Почему квадрат ошибки? Потому что:
  • большие промахи штрафуются сильнее,
  • ошибки разных знаков не взаимоуничтожаются,
  • получается гладкая, удобно оптимизируемая функция.
Линейная регрессия в классическом виде — это задача least squares: найти такую прямую / гиперплоскость, чтобы сумма квадратов отклонений была минимальна.

Геометрический смысл least squares

Если у нас одна переменная, то модель проводит линию через облако точек. Но линия почти никогда не проходит через все точки идеально. Тогда мы выбираем ту, у которой суммарные квадраты вертикальных отклонений минимальны.

\min_{w,b}\sum_{i=1}^n (y^{(i)} - (wx^{(i)}+b))^2
Что означают символы
min_{w,b}оптимизация по w и bИщем веса и bias, при которых суммарная ошибка минимальна.
Σ_{i=1}^nсумма по объектамСуммирует квадраты ошибок для каждого из n обучающих примеров.
nчисло объектовРазмер обучающей выборки — количество примеров в сумме.
y^(i)истинный ответПравильное числовое значение для i-го объекта (target).
wx^(i)+bпрогноз моделиЛинейная комбинация признаков: w·x^(i) плюс свободный член b.
Интуиция: Ищем линию, которая минимизирует суммарные квадраты вертикальных расстояний от точек до линии — квадраты residuals.
Аналогия: Как натянуть резинку через облако точек: минимизируем суммарное напряжение (квадраты отклонений).
Числовой пример: y^(1)=10, прогноз=8 → (10-8)²=4. y^(2)=6, прогноз=5 → (6-5)²=1. Итого SSE=5.

Что минимизируется

  • не просто количество ошибок,
  • не абсолютные отклонения,
  • а именно квадраты residuals.

Residual

r^{(i)} = y^{(i)} - \hat{y}^{(i)}
Что означают символы
r^(i)residual (остаток)Разность между истинным значением и прогнозом для i-го объекта.
y^(i)истинный ответПравильное числовое значение из обучающей выборки (target).
ŷ^(i)прогноз моделиПредсказание линейной регрессии для i-го объекта: w^T x^(i) + b.
Интуиция: Residual — вертикальное расстояние от точки до линии регрессии. Least squares минимизирует сумму их квадратов.
Аналогия: Как промах стрелка: r^(i) — насколько выстрел (прогноз) не попал в мишень (истинный ответ).
Числовой пример: y^(1)=10, ŷ^(1)=8 → r^(1)=10−8=2. Знак важен: +2 означает недооценку.

Это вертикальное расстояние от точки до линии предсказания.

MSE очень чувствителен к выбросам: если ошибка выросла в 10 раз, вклад в loss вырастает в 100 раз. Поэтому линейная регрессия может сильно страдать от extreme values.

Least squares как ортогональная проекция

В матричном виде модель ищет точку Xw в пространстве всех линейных комбинаций столбцов X. Вектор y обычно не лежит точно в этом пространстве, поэтому алгоритм выбирает ближайшую точку по евклидовой дистанции.

Residual y−Xw в оптимуме ортогонален каждому столбцу X. Это сильная геометрическая идея: после обучения ни один признак не должен содержать линейно извлекаемый остаточный сигнал.

  • Столбцы X задают подпространство возможных прогнозов.
  • Xw — конкретная точка в этом подпространстве.
  • Residual — часть target, которую линейная модель не смогла объяснить.

Градиентный взгляд: как модель учится

Чтобы минимизировать MSE, можно использовать gradient descent. Для этого нужно понять, как loss меняется при изменении параметров.

Для случая одной переменной

\hat{y}^{(i)} = wx^{(i)} + b
Что означают символы
ŷ^(i)предсказание для i-го объектаЧисло, которое модель выдаёт для i-го примера из обучающей выборки.
wвес (скаляр)Наклон прямой — единственный коэффициент в одномерном случае.
x^(i)признак i-го объектаЗначение единственного признака для i-го обучающего примера.
bсвободный членСдвиг прямой — базовый уровень прогноза при x=0.
Интуиция: Одномерная линейная регрессия: прямая линия y=wx+b, где w — наклон, b — сдвиг. Задача — найти w и b, минимизирующие MSE.
Аналогия: Как прямая на графике цены: w — скорость роста (рублей/м²), b — базовая ставка.
Числовой пример: w=2, b=1, x^(1)=3 → ŷ^(1)=2·3+1=7.
L(w,b)=\frac{1}{n}\sum_{i=1}^n (wx^{(i)} + b - y^{(i)})^2
Что означают символы
L(w,b)MSE — цель минимизацииСреднеквадратичная ошибка по n объектам. Это поверхность в пространстве (w,b), по которой движется gradient descent.
1/nнормировкаУсредняем ошибки по всей выборке, чтобы масштаб loss не зависел от n.
wx^(i) + bпрогноз для i-го объектаЛинейная функция для одномерного случая (w — скаляр). Здесь видно, что L квадратична по w и b — поверхность выпуклая, минимум единственный.
y^(i)истинный ответМетка i-го объекта. Разность (wx^(i)+b − y^(i)) — это signed residual (ошибка со знаком).
wвес (скаляр в 1D случае)Наклон прямой. Оптимизируется gradient descent-ом.
bсвободный членСдвиг прямой. Оптимизируется вместе с w.
Интуиция: MSE — выпуклая парабола по w и b. Gradient descent находит её дно — это и есть оптимальные параметры прямой.
Аналогия: Шарик в чаше (выпуклая поверхность MSE). Градиент — уклон в текущей точке, шаг — движение вниз. После нескольких шагов шарик достигает дна.
Числовой пример: w=1, b=0, x^(1)=2, y^(1)=5. Residual = 1·2+0−5 = −3. Вклад в loss: 9. После нормировки на n вычитается из суммы.

Градиенты

\frac{\partial L}{\partial w} = \frac{2}{n}\sum_{i=1}^n (wx^{(i)} + b - y^{(i)})x^{(i)}
Что означают символы
∂L/∂wчастная производная loss по wПоказывает, в какую сторону и насколько нужно изменить w, чтобы уменьшить L. Gradient descent двигает w в направлении −∂L/∂w.
2/nконстанта из правила дифференцированияПоявляется при дифференцировании (·)² по цепному правилу. На обучение влияет через масштаб — можно поглотить в learning rate.
wx^(i)+b−y^(i)residual (ошибка) i-го объектаSigned residual: прогноз минус истина. При переобучении можно думать так: градиент — это взвешенная сумма ошибок, взвешенная на признак x^(i).
x^(i)признак i-го объектаМножитель от дифференцирования w·x^(i) по w. Объекты с большими |x^(i)| вносят больший вклад в градиент.
Интуиция: Градиент по w — это сумма произведений (ошибка × признак). Если модель систематически завышает прогнозы (residual>0) у объектов с большим x, нужно уменьшить w.
Аналогия: Тренер корректирует ошибки: чем больше значение признака x^(i) у ошибочного объекта, тем сильнее «дёргается» вес w в нужную сторону.
Числовой пример: n=1, x^(1)=2, prогноз=8, y^(1)=5. Residual=3. ∂L/∂w = (2/1)·3·2 = 12. Значит w нужно уменьшить.
\frac{\partial L}{\partial b} = \frac{2}{n}\sum_{i=1}^n (wx^{(i)} + b - y^{(i)})
Что означают символы
∂L/∂bчастная производная loss по bПоказывает, насколько сдвиг b влияет на loss. Gradient descent корректирует b по этому градиенту.
2/nкоэффициент из дифференцированияАналогично градиенту по w — от правила дифференцирования квадрата.
wx^(i)+b−y^(i)residual i-го объектаОшибка со знаком. Производная b·const по b = const, поэтому нет множителя x^(i) (в отличие от ∂L/∂w). Градиент — просто среднее всех ошибок.
w, bтекущие параметрыПараметры модели в текущей итерации. Входят в вычисление residual, который определяет направление и величину обновления.
Интуиция: ∂L/∂b — среднее всех ошибок (без взвешивания на x). Если модель систематически завышает, ∂L/∂b>0, и b уменьшается — прямая сдвигается вниз.
Аналогия: Корректировка нулевого уровня весов: если в среднем прибор врёт на +5, надо сдвинуть нулевую отметку на −5. Это ровно то, что делает ∂L/∂b.
Числовой пример: 3 объекта с ошибками +2, −1, +3. ∂L/∂b = (2/3)·(2−1+3) = (2/3)·4 ≈ 2.67. Значит b нужно уменьшить.
Шаг 1. Считаем предсказания ŷ.
Шаг 2. Считаем ошибки ŷ - y.
Шаг 3. Через эти ошибки считаем градиенты по w и b.
Шаг 4. Делаем шаг в сторону уменьшения loss.

Обновление параметров

w \leftarrow w - \eta \frac{\partial L}{\partial w}
Что означают символы
w ←обновление весаПрисваивание нового значения: старый w заменяется результатом шага в сторону антиградиента.
ηlearning rate (шаг обучения)Контролирует размер шага. Слишком большой — алгоритм расходится, слишком маленький — сходится медленно. Подбирается как гиперпараметр.
∂L/∂wградиент loss по wНаправление наибольшего роста L. Знак минус разворачивает движение в сторону убывания — это и есть gradient descent.
wтекущий вес (правая сторона)Значение w до обновления. После итерации w = w_old − η·(∂L/∂w). Повторяется до сходимости.
Интуиция: Маленький шаг в направлении «под уклон» на выпуклой поверхности MSE. При правильном η каждая итерация уменьшает L, пока не достигнем минимума.
Аналогия: Альпинист спускается в тумане: он не видит дно, но чувствует уклон (градиент) и делает шаг в сторону склона. η — длина шага. Слишком большой шаг — перескочишь через дно.
Числовой пример: w=3.0, η=0.1, ∂L/∂w=5.0. Новое w = 3.0 − 0.1·5.0 = 2.5. Loss уменьшился — движемся к минимуму.
b \leftarrow b - \eta \frac{\partial L}{\partial b}
Что означают символы
b ←обновление biasАналогично обновлению w: bias корректируется на шаг η в направлении антиградиента.
ηlearning rateТот же гиперпараметр, что и для w. Часто для w и b используют единый η.
∂L/∂bградиент loss по bСреднее всех residuals (без весов x). Если прямая систематически выше данных, ∂L/∂b>0 и b уменьшается.
bтекущий bias (правая сторона)Значение до обновления. После итерации b = b_old − η·(∂L/∂b). W и b обновляются одновременно на каждой итерации.
Интуиция: Bias — вертикальный сдвиг прямой. Если в среднем прогнозы выше меток, нужно сдвинуть прямую вниз — это делает ∂L/∂b с правильным знаком.
Аналогия: Калибровка весов: сначала настраиваем чувствительность (w), потом сдвигаем нулевую отметку (b). GD делает оба шага одновременно.
Числовой пример: b=1.0, η=0.1, ∂L/∂b=2.67. Новое b = 1.0 − 0.1·2.67 = 0.733. Прямая немного опустилась — меньше систематически завышает.
Как не ошибиться

Где линейная модель обманывает

Если связь нелинейная, остатки будут иметь структуру: дугу, веер, разные режимы. Это сигнал, что модель недоописывает задачу, а не просто плохо обучилась.

Мультиколлинеарность делает коэффициенты нестабильными: прогноз может оставаться нормальным, но интерпретация отдельных весов становится ненадежной.

Здесь η — learning rate.

Если learning rate слишком большой — оптимизация может прыгать и расходиться. Если слишком маленький — обучение будет идти очень медленно.

Интерактив: один шаг gradient descent

Здесь используется тот же набор точек, что и в графике сверху. Можно руками смотреть, как gradient descent двигает параметры к лучшему решению.

Матричная запись

Для всего dataset удобно перейти к матричной форме.

X \in \mathbb{R}^{n \times d},\quad \mathbf{w} \in \mathbb{R}^{d},\quad \mathbf{y} \in \mathbb{R}^{n}
Что означают символы
X ∈ R^{n×d}матрица данныхn строк — объекты, d столбцов — признаки.
nчисло объектовКоличество строк в матрице X — размер обучающей выборки.
dчисло признаковКоличество столбцов в X и длина вектора весов w.
w ∈ R^dвектор весовd коэффициентов: по одному на каждый признак.
y ∈ R^nвектор целевых значенийn правильных ответов, соответствующих строкам X.
Интуиция: Матричная запись позволяет компактно выразить n уравнений: каждая строка X — один объект, каждый столбец — один признак.
Аналогия: Как таблица в Excel: n строк (объекты) × d столбцов (признаки) = матрица X.
Числовой пример: 50 объектов с 10 признаками: X ∈ R^{50×10}, w ∈ R^10, y ∈ R^50.
\hat{\mathbf{y}} = X\mathbf{w} + b\mathbf{1}
Что означают символы
ŷвектор предсказанийn-мерный вектор ŷ ∈ R^n — предсказания для всех n объектов сразу.
Xматрица признаковn×d матрица данных: каждая строка — объект, каждый столбец — признак.
wвектор весовd коэффициентов — по одному на каждый признак. Xw = матрично-векторное произведение.
b·1bias (broadcast)Скаляр b, умноженный на вектор единиц 1 ∈ R^n — добавляет b ко всем n предсказаниям.
Интуиция: Матричная форма вычисляет все n предсказаний за одно умножение: ŷ = Xw + b·1, где 1 — столбец единиц.
Аналогия: Как таблица умножения: одна операция Xw сразу даёт взвешенные суммы для всех строк, плюс константа b.
Числовой пример: X ∈ R^{3×2}, w=[1,2], b=0.5: ŷ_i = x_{i1}·1 + x_{i2}·2 + 0.5 для i=1,2,3.

Если bias включить прямо в матрицу признаков, добавив столбец единиц:

\tilde{X} = [X \ \mathbf{1}],\quad \tilde{\mathbf{w}} = \begin{bmatrix} \mathbf{w}\\ b \end{bmatrix}
Что означают символы
X̃расширенная матрица признаковX с добавленным столбцом единиц: X̃ ∈ R^{n×(d+1)}. Позволяет включить bias b в вектор параметров.
1вектор единицn-мерный столбец единиц — добавляется к X справа для учёта bias.
w̃расширенный вектор параметровВектор [w; b] ∈ R^{d+1}: веса w и bias b объединены в один вектор размера d+1.
bсвободный членПоследний элемент w̃: скаляр-сдвиг, теперь включённый в вектор параметров w̃.
Интуиция: Трюк с добавлением единичного столбца позволяет трактовать bias как ещё один вес — тогда X̃w̃ включает и все Xw-слагаемые, и b.
Аналогия: Как добавить константу в таблицу: вместо отдельного учёта фиксированного сбора просто добавляем колонку "1" в данные.
Числовой пример: X=[[2],[3]], 1=[[1],[1]] → X̃=[[2,1],[3,1]], w̃=[0.5,1]: X̃w̃=[2,2.5].
\hat{\mathbf{y}} = \tilde{X}\tilde{\mathbf{w}}
Что означают символы
ŷвектор предсказанийn-мерный вектор: ŷ = X̃w̃ вычисляет предсказания для всех n объектов сразу.
X̃расширенная матрицаX с добавленным столбцом единиц: X̃ ∈ R^{n×(d+1)}.
w̃расширенный вектор [w; b]d весов признаков плюс bias — всё в одном векторе. Упрощает запись: вместо Xw+b достаточно X̃w̃.
Интуиция: Единственное матричное умножение X̃w̃ заменяет n отдельных вычислений — компактная запись для нормального уравнения.
Аналогия: Как калькулятор цен оптом: одна операция вместо расчёта для каждого клиента.
Числовой пример: n=2, d=1: X̃=[[2,1],[3,1]], w̃=[0.5,1] → ŷ=[2,2.5].

MSE в матричной форме

L(\tilde{\mathbf{w}})=\frac{1}{n}\|\tilde{X}\tilde{\mathbf{w}}-\mathbf{y}\|_2^2
Что означают символы
L(w̃)MSE loss в матричной формеТа же среднеквадратичная ошибка, что и поэлементно, но записана через матрицы. Удобна для вывода аналитического решения (normal equation).
1/nнормировка на размер выборкиДелает loss независимым от n — усредняем квадрат нормы по числу объектов.
X̃расширенная матрица признаковМатрица X с добавленным столбцом единиц: X̃ = [X | 1]. Позволяет включить bias b в вектор параметров w̃ без отдельного учёта.
w̃расширенный вектор параметровВектор [w; b] — веса и bias объединены в один вектор. Результат X̃w̃ даёт все предсказания сразу.
yвектор истинных метокВсе n целевых значений из обучающей выборки, собранные в столбец.
||·||₂²квадрат L2-нормы вектора ошибок||X̃w̃−y||₂² = Σ(ŷ_i − y_i)². Это сумма квадратов всех residuals — тот же MSE·n.
Интуиция: Матричная запись компактно кодирует то, что поэлементно занимало несколько строк. Она открывает путь к normal equation: берём производную, приравниваем к нулю — и сразу получаем w* = (X̃^T X̃)^{-1} X̃^T y.
Аналогия: Вместо 1000 уравнений (по объекту) — одно матричное уравнение. Как тензор в физике: тот же смысл, но компактная запись позволяет оперировать всей системой разом.
Числовой пример: n=2: X̃w̃=[8,6], y=[10,5]. Разность=[−2,1]. ||·||₂²=4+1=5. L=5/2=2.5 — та же MSE, что и поэлементно.
Это уже очень важный мост к линейной алгебре: линейная регрессия — это задача минимизации квадратичной нормы ошибки.

Нормальное уравнение: closed-form решение

Для линейной регрессии есть не только gradient descent. В чистом least squares случае можно получить решение аналитически.

\mathbf{w}^* = (X^T X)^{-1} X^T \mathbf{y}
Что означают символы
w*оптимальный вектор параметровАналитическое решение задачи least squares — параметры, при которых MSE минимален. Это точный минимум, не приближение.
X^T Xматрица Грама (d×d)Произведение транспонированной матрицы признаков на саму себя. Квадратная, симметричная. Если обратима, normal equation имеет единственное решение.
(X^T X)^{-1}обратная матрица ГрамаСуществует, когда признаки не мультиколлинеарны. При мультиколлинеарности X^T X вырождена — нужна псевдообратная или регуляризация (Ridge).
X^T yвектор корреляций признаков с целевой переменнойd-мерный вектор: j-й элемент — скалярное произведение j-го признака (столбца X) с вектором меток y. Кодирует, насколько каждый признак коррелирует с y.
yвектор целевых значенийn-мерный столбец с истинными метками обучающей выборки.
Интуиция: Normal equation — это результат приравнивания градиента ∂L/∂w к нулю и решения системы. В одно действие даёт точный ответ без итераций, но требует O(d³) для обращения матрицы.
Аналогия: Как решить систему линейных уравнений через обратную матрицу: Ax=b → x=A⁻¹b. Здесь A=X^T X, b=X^T y. Метод Крамера для regression.
Числовой пример: X=[[1],[2],[3]], y=[2,4,5]. X^T X=[[14]], X^T y=[[24]]. w* = 24/14 ≈ 1.71. Прямая y=1.71x — оптимальный наклон.

Это и есть normal equation.

Плюсы

  • не нужен learning rate,
  • не нужны итерации,
  • получаем точное решение least squares.

Минусы

  • дорого для больших данных,
  • нужно обращаться с матрицей X^T X,
  • могут быть проблемы с вырожденностью / мультиколлинеарностью.
На практике часто используют не прямое обращение матрицы, а более устойчивые методы: QR, SVD, lstsq-решатели.
Смысл closed-form решения важен концептуально: линейная регрессия — одна из редких моделей, где связь между геометрией, алгеброй и оптимизацией видна особенно прозрачно.
Теория глубже

Почему closed-form решение может быть численно хрупким

Нормальное уравнение выглядит элегантно, но матрица XᵀX усиливает проблемы conditioning. Если признаки сильно коррелируют или имеют разные масштабы, обратная матрица становится нестабильной: маленький шум в данных может резко менять коэффициенты.

Поэтому на практике часто используют SVD, QR-разложение, регуляризацию Ridge или итерационные методы. Это не усложнение ради усложнения, а защита от плохой геометрии признакового пространства.

  • Multicollinearity делает коэффициенты неустойчивыми.
  • Scaling улучшает conditioning и ускоряет gradient descent.
  • Ridge добавляет λI и делает задачу более устойчивой.

Статистический смысл линейной регрессии

Часто линейную регрессию пишут так:

y = \mathbf{w}^T\mathbf{x} + b + \varepsilon
Что означают символы
yнаблюдаемый ответТо, что мы реально измеряем в данных. В отличие от ŷ, это «правда» — но она содержит шум ε.
w^T x + bдетерминированный сигналЛинейный предиктор — «идеальный» прогноз без шума. Это μ в нормальном распределении ответа y|x.
εслучайный шумВсё, что модель не может объяснить: ошибки измерения, пропущенные признаки, случайность. Предполагается ε~N(0,σ²).
wвектор весов признаковПараметры линейного предиктора. В статистической трактовке — истинные коэффициенты генерирующей модели.
bсвободный членБазовый уровень y при x=0. В статистике соответствует intercept.
Интуиция: Модель говорит: «данные = сигнал + шум». Задача регрессии — оценить сигнал (w, b), не переобучившись на шум ε. Предположение ε~N(0,σ²) делает MSE правильным criterion-ом по MLE.
Аналогия: Измерение температуры: истинная t° = линейный тренд + погрешность прибора. Модель ищет тренд, не подгоняя кривую под каждый шум термометра.
Числовой пример: w=2, b=1, x=3, ε=0.3. Наблюдаем y = 2·3+1+0.3 = 7.3. Модель предсказывает ŷ=7 (без шума). Residual=0.3=ε.

где ε — шум, который модель не может объяснить.

Если предположить, что:

  • ошибки независимы,
  • ошибки имеют нулевое среднее,
  • ошибки нормально распределены: ε ~ N(0, σ²),

то минимизация MSE эквивалентна максимизации правдоподобия.

p(y \mid x) = \mathcal{N}(\mathbf{w}^T \mathbf{x} + b,\sigma^2)
Что означают символы
p(y|x)условная плотность ответаВероятностная модель: при данных признаках x ответ y имеет нормальное распределение. Это и есть вероятностная интерпретация линейной регрессии.
N(·, ·)нормальное распределениеПервый аргумент — среднее (предсказание), второй — дисперсия шума σ². Предположение гауссового шума делает MSE оптимальным критерием MLE.
w^T x + bлинейный предиктор — среднее предсказанияСкалярное произведение весов w и признаков x плюс смещение b. Это μ нормального распределения — «центральное предсказание» для данного x.
σ²дисперсия шума — неопределённость предсказанияПараметр разброса вокруг предсказания. Малый σ² — данные плотно около прямой. Большой σ² — данные сильно шумят.
Интуиция: p(y|x)=N(w^T x+b, σ²) говорит: наблюдаемый y — это линейный сигнал плюс гауссовский шум. Это предположение и делает MSE оптимальным критерием по MLE.
Аналогия: Как измерение с погрешностью: истинное значение w^T x+b, но прибор «шумит» с дисперсией σ². Каждое наблюдение немного отличается от идеала.
Числовой пример: w=[1,2], x=[3,4], b=0.5. Среднее: 1·3+2·4+0.5=12.5. При σ²=1: y~N(12.5,1), большинство значений в [11.5, 13.5].
Это очень важная связь: MSE — не просто удобная функция ошибки, а логичный выбор при гауссовском шуме.

Когда линейная регрессия работает хорошо

Ситуация Что происходит Комментарий
Связь почти линейная модель хорошо аппроксимирует зависимость идеальный сценарий
Немного шума least squares даёт устойчивую оценку если нет тяжёлых выбросов
Много признаков, но мало данных может переобучаться понадобится регуляризация
Сильная нелинейность модель систематически недообучается нужны новые признаки или другая модель
Сильные выбросы MSE начинает тянуть решение к outliers нужны robust-подходы
Линейная регрессия не “плохая”, если зависимость сложная. Она просто имеет ограниченный класс функций. Это вопрос bias модели, а не “устарелости”.

Коэффициенты и интерпретация

Один из главных плюсов линейной регрессии — интерпретируемость.

\hat{y} = w_1x_1 + w_2x_2 + \dots + w_dx_d + b
Что означают символы
ŷпредсказание моделиСкаляр — числовой ответ модели для одного объекта: взвешенная сумма признаков плюс bias.
w_jвес j-го признакаКоэффициент: при росте x_j на 1 прогноз меняется на w_j (при прочих равных).
x_jj-й признак объектаЧисловое значение j-й характеристики объекта (j = 1, ..., d).
bсвободный член (bias)Базовый уровень предсказания при x_1=...=x_d=0.
Интуиция: Предсказание — это "смета": каждый признак вносит вклад w_j·x_j, и всё складывается плюс базовая ставка b.
Аналогия: Оценка квартиры: ŷ = 500·м² + 100·этаж + (−200)·шум + 50000. Каждый фактор имеет свой "тариф" w_j.
Числовой пример: w=[0.5, 2], x=[50, 3], b=10: ŷ=0.5·50+2·3+10=25+6+10=41.

При прочих равных коэффициент w_j показывает, как меняется предсказание при увеличении признака x_j на 1.

\Delta x_j = 1 \Rightarrow \Delta \hat{y} = w_j
Что означают символы
Δx_j = 1увеличение признака j на 1Единичное изменение j-го признака при прочих признаках фиксированных.
Δŷизменение прогнозаНасколько меняется предсказание при Δx_j = 1 — ровно на w_j.
w_jкоэффициент j-го признакаНаклон: при росте x_j на 1 единицу прогноз меняется ровно на w_j (при прочих равных).
Интуиция: Это главное свойство линейной интерпретации: w_j — это "сколько даёт один дополнительный день/метр/рубль" признака j.
Аналогия: Как цена за единицу товара: если w_j=500, каждый дополнительный м² квартиры добавляет 500 к цене.
Числовой пример: w_j=2.5 для признака "площадь". Площадь+1 м² → ŷ+2.5 тыс. руб. — при прочих признаках неизменных.
Но эта интерпретация корректна только если:
  • масштабы признаков понятны,
  • нет сильной мультиколлинеарности,
  • мы говорим “при прочих равных”,
  • модель вообще адекватно описывает зависимость.
Если признаки сильно коррелируют, коэффициенты могут стать нестабильными: знак и величина весов начинают “делить ответственность” друг с другом. Это одна из причин, почему дальше появится регуляризация.

Интерактив: эффект масштаба признака на коэффициент

Если заменить признак x на x' = 10x, то “правильный” коэффициент должен уменьшиться в 10 раз, чтобы предсказания остались теми же. Поэтому сравнивать веса признаков без учёта scale опасно.

Bias, variance и роль линейной регрессии

Линейная регрессия — модель с довольно высоким inductive bias: она предполагает, что зависимость близка к линейной.

Плюсы такого bias

  • простая и понятная модель,
  • часто мало variance,
  • устойчива при умеренном количестве признаков,
  • легко интерпретируется.

Минусы

  • может сильно недообучаться на сложных данных,
  • не умеет сама ловить нелинейности,
  • чувствительна к выбросам и коллинеарности.
Именно поэтому линейная регрессия — отличный baseline: она быстро показывает, есть ли в данных хотя бы простая линейная структура.

Частые ошибки

  • использовать линейную регрессию для classification,
  • не проверять выбросы,
  • интерпретировать коэффициенты без scaling и без понимания единиц измерения,
  • забывать про leakage в preprocessing,
  • считать высокий train R² доказательством хорошей модели,
  • игнорировать residual analysis,
  • не замечать мультиколлинеарность.
Хороший workflow с линейной регрессией почти всегда включает: preprocessing, baseline, анализ residuals, сравнение train/val ошибки и проверку, не нужна ли регуляризация.

Takeaway

Линейная регрессия — это базовая модель для предсказания числовой цели через линейную комбинацию признаков.

  • Модель имеет вид ŷ = w^T x + b.
  • Классический loss — MSE, сумма / среднее квадратов ошибок.

Вероятностный взгляд: линейная регрессия как MLE

Очень полезно думать о линейной регрессии не просто как о прямой или гиперплоскости, а как о вероятностной модели. Мы предполагаем, что правильный ответ состоит из линейного сигнала и гауссова шума. Тогда оптимизация MSE появляется не случайно, а как прямое следствие maximum likelihood.

y_i = w^T x_i + b + ε_i, ε_i ~ N(0, σ²)
Что означают символы
y_iнаблюдаемый ответ i-го объектаТо, что мы измерили или наблюдали. Это y_i = детерминированная часть + случайный шум.
w^T x_i + bлинейный сигнал — детерминированная частьПредсказание модели для i-го объекта. Скалярное произведение весов и признаков плюс смещение. Это «идеальный ответ» без шума.
ε_iслучайный шум — ошибка наблюденияГауссовский шум для i-го объекта. Независимый от всего остального. Моделирует неточности измерений и пропущенные факторы.
N(0, σ²)нормальное распределение шумаШум несмещён (среднее 0) и имеет дисперсию σ². Это предположение и приводит к MSE как оптимальному критерию обучения.
Интуиция: Данные = сигнал + шум. w^T x_i+b — это то, что модель предсказывает. ε_i — то, что она не может объяснить. Предположение ε~N(0,σ²) → MSE — оптимальный loss.
Аналогия: Как температура в комнате: есть закономерность (зависит от сезона, времени суток), но каждое измерение содержит небольшую случайную погрешность ε.
Числовой пример: w^T x_i+b=5.2, σ=0.3. Наблюдаем y_i=5.4. ε_i=5.4−5.2=0.2. Вероятность такого ε при N(0,0.09): f(0.2)≈1.07. Не нулевая, вполне ожидаемо.
-log p(y|X,w,b) ∝ Σ (y_i - ŷ_i)²
Что означают символы
-log p(y|X,w,b)отрицательный log-likelihood — NLLЭто то, что минимизируем при обучении. -log превращает максимизацию likelihood в минимизацию loss. При гауссовском шуме NLL пропорционален MSE.
∝пропорционально — знак скрытой константыРавенство с точностью до константы (1/2σ²). σ² не зависит от w и b, поэтому аргмин NLL совпадает с аргмин суммы квадратов.
Σ_iсумма по всем объектам датасетаСуммируем квадрат ошибки по всем n объектам. Это и есть MSE (без деления на n), который оптимизирует линейная регрессия.
(y_i - ŷ_i)²квадрат остатка (residual)Насколько предсказание ŷ_i=w^T x_i+b отличается от истинного ответа y_i. Квадрат штрафует большие ошибки сильнее малых.
Интуиция: Минимизация NLL при гауссовском шуме = минимизация MSE. Именно поэтому MSE — не произвольный выбор, а следствие вероятностного предположения о природе шума.
Аналогия: Как регулировка антенны: NLL говорит «насколько плохо параметры объясняют данные». Минимизировать NLL = найти параметры, максимально объясняющие данные.
Числовой пример: y_i=10, ŷ_i=8: остаток 2, квадрат 4. y_i=10, ŷ_i=12: остаток -2, квадрат 4. Сумма квадратов нечувствительна к знаку ошибки.
Это одна из самых важных связей во всём classic ML: выбор loss-функции почти всегда implicitly говорит, какой шум и какую статистическую природу ошибок мы считаем правдоподобными.

Геометрическая интуиция least squares

В терминах линейной алгебры линейная регрессия — это задача проекции. Мы ищем такую линейную комбинацию признаков, чтобы вектор предсказаний лежал как можно ближе к вектору истинных ответов. Остаток — это та часть сигнала, которую выбранное линейное подпространство уже не умеет объяснить.

ŷ = Xw, r = y - ŷ
Что означают символы
ŷвектор предсказаний (n×1)Все предсказания модели сразу в одном векторе. ŷ = Xw означает: умножаем матрицу признаков на вектор весов, получаем n предсказаний.
Xматрица объектов-признаков (n×d)Design matrix: n строк (объекты), d столбцов (признаки). Каждая строка — один объект x_i^T. Умножение Xw даёт скалярные произведения w^T x_i для всех i.
wвектор весов (d×1)Коэффициенты линейной модели. Произведение Xw эквивалентно вычислению w^T x_i для каждого объекта одновременно.
rвектор остатков (residuals)r = y − ŷ: насколько предсказания отличаются от истинных ответов. Каждый компонент r_i = y_i − ŷ_i. MSE = ||r||²/n.
Интуиция: ŷ = Xw — это матричная запись n вычислений w^T x_i сразу. r = y − ŷ — это вектор ошибок. В оптимуме r перпендикулярен пространству признаков.
Аналогия: Как проекция: ŷ — это проекция вектора y на линейное подпространство, натянутое на столбцы X. r — вектор «того, что осталось» после проекции.
Числовой пример: X=[[1,2],[3,4]], w=[1,0.5], y=[3,6]. ŷ=[1·1+0.5·2, 1·3+0.5·4]=[2,5]. r=[3−2, 6−5]=[1,1]. MSE=(1+1)/2=1.
X^T r = 0
Что означают символы
X^Tтранспонированная матрица признаков (d×n)Транспозиция design matrix. Строки X^T — это признаки (d штук), столбцы — объекты. Умножение X^T r даёт вектор d скалярных произведений каждого признака с остатками.
rвектор остатков r = y − ŷРазность между истинными ответами и предсказаниями. В оптимуме least squares X^T r = 0 — необходимое условие минимума MSE.
0нулевой вектор — ортогональность остатковX^T r = 0 означает: вектор остатков r ортогонален каждому признаку (каждому столбцу X). Всё, что линейная модель могла объяснить — объяснила.
Интуиция: X^T r = 0 — геометрическое условие оптимума: остатки ортогональны пространству признаков. Это то же самое, что градиент MSE равен нулю.
Аналогия: Как тень на экране: оптимальная проекция ŷ даёт остаток r, перпендикулярный плоскости экрана (пространству X). Дальше «передвинуть проекцию» в пространстве X невозможно.
Числовой пример: Из предыдущего: X=[[1,2],[3,4]], r=[1,1]. X^T r=[1·1+3·1, 2·1+4·1]=[4,6] ≠ 0 → не оптимум. При ŵ=(X^T X)^{-1}X^T y: r ⊥ X. ✓

Последняя формула говорит очень красивую вещь: в оптимуме residual ортогонален пространству признаков. То есть всё, что линейная модель могла объяснить в доступной ей геометрии, она уже объяснила; всё оставшееся лежит вне этого пространства.

Когда линейная регрессия действительно уместна

Хороший режим
Связь примерно аддитивна, шум умеренный, признаки интерпретируемы, а тебе важны baseline и понятные коэффициенты.
Пограничный режим
Есть умеренная нелинейность, но её можно частично исправить feature engineering, логарифмами, взаимодействиями и регуляризацией.
Плохой режим
Сильная нелинейность, тяжёлые выбросы, сложные взаимодействия и зависимость, которую линейное подпространство просто не умеет выразить.

Главная ошибка новичка здесь — путать “модель плохо обучилась” с “линейный bias не соответствует структуре задачи”. Иногда проблема не в optimizer-е, а в том, что сам мир не линейный.

Теоретический слой

Линейная регрессия как проекция на признаки

Линейная регрессия ищет такую линейную комбинацию признаков, которая минимизирует средний квадрат ошибки. Это не только baseline, но и база для bias-variance, residual analysis, regularization и probabilistic interpretation.

Смысл модели хорошо виден через остатки. Если residuals систематичны, меняются по масштабу или зависят от признака, линейная форма недоописала структуру данных.

Геометрическая интуиция

  • Предсказания лежат в подпространстве, натянутом на признаки; обучение проецирует target на это подпространство.
  • Коэффициент задаёт наклон гиперплоскости по своей оси при прочих равных.
  • Коррелированные признаки делают коэффициенты нестабильными: форма предсказания может быть похожей, а отдельные веса сильно разными.

Вероятностная интерпретация

  • При Gaussian noise и постоянной дисперсии MSE соответствует maximum likelihood.
  • Residuals можно читать как реализацию шума, который модель не объяснила.
  • Доверительные интервалы для коэффициентов имеют смысл только при выполнении статистических предположений.

Оптимизационный смысл

  • OLS имеет closed-form решение, но на больших данных часто используют gradient descent.
  • MSE даёт выпуклую квадратичную поверхность, где локальный минимум является глобальным.
  • Scaling улучшает conditioning и ускоряет численную оптимизацию.

Практика и типичные ошибки

  • Проверь residual plots, выбросы, multicollinearity и масштаб признаков.
  • Не интерпретируй коэффициенты без учёта scaling-а и encoding-а.
  • Если веса нестабильны, сравни с Ridge/Lasso.

Мини-пример: Если коэффициент площади равен 1200, то в рамках линейной модели каждый дополнительный метр добавляет около 1200 к цене при фиксированных остальных признаках.

  • Классические модели особенно полезны, когда важны скорость, интерпретируемость, небольшой датасет или строгая валидация.
  • Перед нейросетью проверь, какой результат даёт простой baseline: он задаёт нижнюю планку качества.

Типичные ошибки

  • Подбирать модель до постановки метрики: без правильной метрики можно оптимизировать не ту задачу.
  • Допускать leakage: scaling, encoding, feature selection и tuning должны обучаться только на train внутри pipeline.
  • Сравнивать модели без одинакового split и одинакового preprocessing.

Что тюнить

  • Начни с данных: пропуски, выбросы, encoding, scaling и баланс классов часто важнее выбора алгоритма.
  • Потом меняй сложность модели: регуляризацию, глубину, число соседей, число деревьев или learning rate в зависимости от темы.
  • Фиксируй validation protocol до тюнинга, иначе улучшение может быть подгонкой под конкретный split.

Упражнения: линейная регрессия

Проверка темы - уметь вывести loss, реализовать обучение и объяснить, когда линейность ломается.

[Foundational] Выведи градиент MSE

Для \(L=\frac{1}{n}\|Xw-y\|^2\) выведи \(\nabla_w L\). Решение: \(\nabla_w L=\frac{2}{n}X^T(Xw-y)\).

[Foundational] Интерпретация веса

Что означает \(w_j=3\), если остальные признаки фиксированы? Решение: при росте \(x_j\) на 1 prediction растёт примерно на 3 единицы target, если масштаб признака не менялся.

[Intermediate] NumPy OLS

Реализуй closed-form \(w=(X^TX)^{-1}X^Ty\) и сравни с sklearn. Решение: на практике лучше использовать solve/pinv, а не явную inverse.

[Intermediate] Диагностика residuals

Какие проблемы видны на графике residuals vs prediction? Решение: нелинейность, heteroscedasticity, outliers, missing interactions.

[Hard] Multicollinearity

Объясни, почему коррелирующие признаки делают коэффициенты нестабильными. Решение: \(X^TX\) становится плохо обусловленной, малые изменения данных сильно меняют \(w\).

Проверь себя

Вопросы для самопроверки

  1. Что минимизирует линейная регрессия при обучении методом наименьших квадратов?

    MSE — среднеквадратичную ошибку: L = (1/n)·Σ(yᵢ - wᵀxᵢ)². Минимизация MSE эквивалентна максимизации правдоподобия при предположении о нормальном распределении ошибок (Gaussian noise).

  2. Что такое нормальное уравнение и когда его использовать?

    Нормальное уравнение даёт аналитическое решение: w = (XᵀX)⁻¹Xᵀy. Оно точно, но требует инвертирования матрицы — O(p³) по времени. При малом числе признаков (p < несколько тысяч) это быстро; при большом p лучше использовать итеративный градиентный спуск.

  3. Почему мультиколлинеарность делает коэффициенты нестабильными?

    Если признаки сильно коррелируют, матрица XᵀX становится почти вырожденной (плохо обусловленной). Малые изменения в данных приводят к большим изменениям в коэффициентах w. Решения: Ridge-регуляризация, удаление/агрегация коррелированных признаков.

  4. Как R² интерпретируется и каков его диапазон?

    R² показывает долю дисперсии целевой переменной, объяснённую моделью: R² = 1 - SS_res/SS_tot. Значение 1 — идеальное предсказание; 0 — модель не лучше просто предсказывать среднее; отрицательные значения возможны, если модель хуже базовой.

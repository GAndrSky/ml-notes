2.13 SVM

Maximum margin, hard/soft margin, kernel trick, RBF / polynomial / linear kernels, параметры C и gamma.

Почему важно

SVM ищет максимально широкий «коридор» между классами. Kernel trick позволяет работать в миллионномерных пространствах признаков без явного вычисления координат.

Что пытается сделать SVM

Support Vector Machine ищет не просто границу между классами, а такую разделяющую гиперплоскость, которая оставляет максимальный зазор между классами.

Идея SVM: из всех разделяющих гиперплоскостей выбрать ту, у которой margin максимален
Теория по месту

SVM максимизирует зазор

SVM ищет границу, у которой максимальный margin до ближайших объектов. Эти ближайшие объекты и есть support vectors: они фактически держат границу.

Параметр C задает цену нарушения margin. Большой C стремится классифицировать train жестче, маленький C допускает больше ошибок ради широкой и устойчивой границы.

Интуиция. Если между классами есть несколько возможных прямых, SVM предпочитает не ту, что “едва-едва разделяет”, а ту, которая проходит максимально далеко от ближайших точек обоих классов.
Это делает решение более устойчивым: небольшие шумы в данных с меньшей вероятностью сдвинут границу.

2.13.1 Maximum margin intuition

Пусть есть разделяющая гиперплоскость:

wᵀx + b = 0
Что означают символы
w, bгиперплоскостьПараметры разделяющей границы.
Cцена нарушенияНасколько сильно штрафуются ошибки margin.

Интуиция: Формула ищет границу с большим margin и штрафует объекты, которые нарушают зазор.
Аналогия: Как провести коридор между двумя группами точек: граница должна быть не вплотную к объектам.
Числовой пример: Если margin violation равен 0, объект не добавляет hinge-loss; если violation 0.7, он штрафуется.

Тогда две margin-линии:

wᵀx + b = +1 wᵀx + b = -1

А ширина margin равна:

margin width = 2 / ||w||
Что означают символы
margin widthширина полосы разделенияРасстояние между двумя параллельными гиперплоскостями w^T x + b = ±1, ограничивающими margin. SVM максимизирует это расстояние.
2два полу-marginКаждая из двух опорных гиперплоскостей находится на расстоянии 1/||w|| от центральной границы. Суммарный margin = 1/||w|| + 1/||w|| = 2/||w||.
||w||норма вектора весов (L2)Длина вектора нормали к разделяющей гиперплоскости. Чем больше ||w||, тем уже margin. Максимизировать margin = минимизировать ||w||.
Интуиция: Margin — это «коридор безопасности» между классами. SVM ищет гиперплоскость с наибольшим коридором. Чем меньше ||w||, тем шире margin и тем устойчивее решение к новым данным.
Аналогия: Как провести двойную разделительную линию на дороге: хочется, чтобы коридор между потоками был как можно шире для безопасности.
Числовой пример: ||w||=1 → margin=2 (широкий). ||w||=4 → margin=0.5 (узкий). Максимизация margin = минимизация ||w||² в objective SVM.
Значит, максимизировать margin — это то же самое, что минимизировать ||w||.
Важно: решение определяют не все точки выборки, а только ближайшие к границе точки — support vectors.
Теория глубже

Margin - это геометрический запас прочности

SVM выбирает не просто разделяющую линию, а линию с максимальным зазором до ближайших объектов. Этот зазор можно читать как устойчивость к небольшим сдвигам данных: если точка немного шумит, она всё ещё остаётся на правильной стороне boundary.

Именно support vectors определяют boundary. Далёкие точки почти не влияют на решение, потому что они уже имеют достаточный margin и не несут информации о границе.

  • Большой margin обычно лучше переносится на новые данные.
  • Support vectors - точки, которые ограничивают зазор.
  • Если классы сильно пересекаются, maximum margin без ошибок становится невозможен.

2.13.2 Hard margin SVM

В идеальном случае данные линейно разделимы и можно потребовать, чтобы все точки лежали по правильную сторону margin.

Оптимизация hard margin SVM: minimize (1/2) ||w||² subject to y_i (wᵀx_i + b) ≥ 1 для всех i
Эти ограничения говорят: каждая точка не просто должна быть по правильную сторону, а должна лежать не ближе margin-линии.
Hard margin почти бесполезен на реальных noisy-данных: достаточно одного выброса или одной плохо размеченной точки, и строгая линейная разделимость ломается.

2.13.3 Soft margin SVM

В реальных задачах данные часто неидеальны. Поэтому вводят slack variables ξ_i, которые позволяют некоторым точкам нарушать margin или даже оказываться не на той стороне.

Soft margin SVM: minimize (1/2) ||w||² + C Σ_i ξ_i subject to y_i (wᵀx_i + b) ≥ 1 - ξ_i ξ_i ≥ 0
Здесь есть компромисс: большой margin против малого числа нарушений.
Параметр C управляет тем, насколько сильно мы штрафуем нарушения margin.
Если C большое: модель очень не любит ошибки → старается классифицировать почти всё правильно → более жёсткая граница → выше риск overfitting Если C маленькое: модель терпимее к нарушениям → margin шире → выше bias, но больше устойчивость
Теория глубже

C задаёт цену нарушения margin

Soft margin разрешает ошибаться или заходить внутрь margin, но штрафует такие нарушения. Параметр C управляет тем, насколько дорого модели игнорировать проблемные точки.

Большой C заставляет SVM сильнее подгоняться под train и делать меньше нарушений. Маленький C допускает больше нарушений ради более широкой и спокойной границы.

  • C вверх: меньше train-ошибок, выше риск переобучения.
  • C вниз: шире margin, больше bias, устойчивее к шуму.
  • C нельзя выбирать без scaling, потому что расстояния и margin зависят от масштаба признаков.

Support vectors — почему они называются именно так

В оптимуме не все точки одинаково важны. Большинство точек можно слегка подвигать, и решение почти не изменится. Но есть несколько ключевых точек — ближайших к границе. Именно они “подпирают” optimal hyperplane.

Решение SVM в dual form зависит только от support vectors w = Σ_i α_i y_i x_i где α_i > 0 только у support vectors
Как не ошибиться

Когда margin перестает помогать

Если признаки не масштабированы, SVM может строить margin вокруг численно крупного признака и игнорировать остальные.

Kernel SVM может быть очень сильным на средних данных, но плохо масштабируется по числу объектов. Для больших датасетов часто используют линейные приближения или другие модели.

То есть support vectors — это точки, которые реально задают границу. Остальные часто вообще не входят в итоговую формулу с ненулевым весом.
Это одна из причин, почему SVM концептуально красив: финальное решение строится на критических граничных точках.

2.13.4 Kernel trick

Линейная граница хороша, но что делать, если классы не разделяются прямой? Можно перейти в пространство более высокой размерности, где разделение станет линейным.

Явное отображение: x → φ(x) Линейная модель в новом пространстве: wᵀ φ(x) + b Но вместо явного φ(x) можно считать только скалярные произведения: K(x_i, x_j) = φ(x_i)ᵀ φ(x_j)
Kernel trick позволяет не строить огромное новое пространство явно, а работать так, будто мы уже туда перешли.
Это похоже на магию: мы не выписываем все новые признаки, но всё равно можем строить нелинейную границу.
Цена этой магии: kernel SVM может быть дорогим по памяти и времени на больших датасетах, потому что работает через pairwise similarities.

2.13.5 Популярные kernels

Kernel Формула Интуиция Когда полезен

Polynomial K(x,z)=(γxᵀz + r)^d Ловит полиномиальные взаимодействия Когда нужны более сложные, но ещё структурные границы

2.13.6 RBF kernel и параметр gamma

В RBF kernel параметр gamma управляет радиусом влияния отдельной точки.

K(x,z) = exp(-γ ||x-z||²) Если γ маленькое: влияние каждой точки широкое → граница более гладкая Если γ большое: влияние точки очень локальное → граница может стать очень извилистой
Маленькое gamma → более гладкая, глобальная форма границы.
Слишком маленькое gamma может дать underfitting.
Большое gamma → модель реагирует на локальные детали.
Слишком большое gamma может привести к overfitting: каждая точка начинает “тянуть” границу под себя.

Hinge loss — связь с оптимизацией SVM

В soft-margin SVM часто удобно думать через hinge loss.

Для y ∈ {-1, +1}: L_hinge = max(0, 1 - y · f(x)) Полная цель: (1/2)||w||² + C Σ_i max(0, 1 - y_i f(x_i))
Если точка классифицирована уверенно и лежит за margin: y f(x) ≥ 1, то loss = 0.
Если точка внутри margin или ошибочно классифицирована, hinge loss становится положительным и штрафует модель.

Когда SVM особенно хорош

  • на задачах с не слишком большим числом объектов, но большим числом признаков;
  • на текстовых данных с sparse high-dimensional representation;
  • когда линейная или умеренно нелинейная граница реально подходит;
  • когда нужен сильный классификационный baseline.
  • Linear SVM особенно хорош в high-dimensional spaces;
  • RBF-SVM часто даёт сильное качество на небольших/средних датасетах;
  • kernel SVM красив и мощен, но плохо масштабируется на очень большие наборы данных.

Когда SVM неудобен

1. Очень большие датасеты: kernel matrix и pairwise similarities становятся дорогими.
2. Нужны хорошо калиброванные вероятности: у SVM это не базовый output.
3. Плохой preprocessing: SVM чувствителен к масштабу признаков.
4. Слишком грубый подбор C и gamma: модель легко можно сделать либо слишком жёсткой, либо слишком извилистой.
Для SVM масштабирование признаков почти всегда обязательно: standardization — это не опция, а почти стандартная часть pipeline.

Практический pipeline для SVM

1. Сделать train / validation / test split.
2. Масштабировать признаки через StandardScaler.
3. Начать с linear kernel как базового варианта.
4. Если линейной границы мало — попробовать RBF.
5. Подбирать C и gamma через cross-validation.
6. Оценивать не только accuracy, но нужные метрики задачи.

Код: sklearn

from sklearn.pipeline import Pipeline from sklearn.preprocessing import StandardScaler from sklearn.svm import LinearSVC from sklearn.metrics import classification_report model = Pipeline([ ("scaler", StandardScaler()), ("clf", LinearSVC(C=1.0, random_state=42)) ]) model.fit(X_train, y_train) pred = model.predict(X_valid) print(classification_report(y_valid, pred))
from sklearn.pipeline import Pipeline from sklearn.preprocessing import StandardScaler from sklearn.svm import SVC from sklearn.metrics import classification_report model = Pipeline([ ("scaler", StandardScaler()), ("clf", SVC( kernel="rbf", C=1.0, gamma="scale" )) ]) model.fit(X_train, y_train) pred = model.predict(X_valid) print(classification_report(y_valid, pred))
from sklearn.pipeline import Pipeline from sklearn.preprocessing import StandardScaler from sklearn.svm import SVC from sklearn.model_selection import GridSearchCV pipe = Pipeline([ ("scaler", StandardScaler()), ("svc", SVC(kernel="rbf")) ]) param_grid = { "svc__C": [0.1, 1, 10, 100], "svc__gamma": ["scale", 0.01, 0.1, 1.0] } grid = GridSearchCV( pipe, param_grid=param_grid, cv=5, scoring="f1_macro", n_jobs=-1 ) grid.fit(X_train, y_train) print("Best params:", grid.best_params_) print("Best CV score:", grid.best_score_)

Частые ошибки

Ошибка 1. Запускать SVM без масштабирования признаков.
Ошибка 2. Ставить большой C и большой gamma одновременно, получая слишком “ломаную” границу.
Ошибка 3. Использовать kernel SVM на очень большом датасете и удивляться скорости.
Ошибка 4. Путать score SVM с вероятностью.
Ошибка 5. Не различать linear SVM и logistic regression по идее и loss.
Ошибка 6. Подбирать C и gamma “на глаз” без CV.

Как объяснить SVM на собеседовании

1. SVM ищет гиперплоскость с максимальным margin между классами.
2. Hard margin требует идеальной разделимости, soft margin допускает нарушения через slack variables.
3. Параметр C задаёт компромисс между шириной margin и штрафом за ошибки.
4. Kernel trick позволяет строить нелинейные границы через работу со скалярными произведениями в feature space.
5. В RBF-SVM параметр gamma контролирует радиус влияния точек и гладкость границы.

Итоговый takeaway

SVM — это модель про геометрию и margin, а не в первую очередь про вероятности.
Support vectors — ключевые точки, которые реально определяют границу.
Soft margin делает SVM применимым к noisy real-world данным.
Kernel trick позволяет делать нелинейную классификацию без явного перехода в огромное пространство признаков.
C управляет жёсткостью штрафа за ошибки, gamma — локальностью RBF-границы.
На практике почти всегда: scaling + CV + осторожный подбор C/gamma.
Ментальная формула: Linear SVM = максимальный margin, RBF-SVM = максимальный margin в скрытом нелинейном пространстве.

Почему maximum margin связан с generalization

Главная интуиция SVM не в том, что он просто проводит разделяющую линию, а в том, что он выбирает среди всех таких линий самую “осторожную”. Большой margin означает, что небольшие колебания данных с меньшей вероятностью переведут объект через границу. Это делает решение более устойчивым к шуму и переобучению.

marginширина разделяющего коридораГеометрическое расстояние между двумя гиперплоскостями w^T x + b = ±1. SVM максимизирует этот зазор — он связан с обобщающей способностью.
2два полу-marginКаждая опорная гиперплоскость (w^T x + b = +1 и = −1) отстоит от центральной на 1/||w||. Сумма: 2/||w||.
||w||L2-норма вектора весовДлина нормали к разделяющей гиперплоскости. Большой ||w|| → узкий margin → хуже обобщение. Минимизация ||w||² при SVM = максимизация margin.
Интуиция: Большой margin означает: граница решения далека от обоих классов. Это делает SVM устойчивым к шуму — небольшие отклонения данных не пересекут разделительную полосу. Теория VC-размерности гарантирует: большой margin → хорошая generalization.
Аналогия: Дорога с широкой разделительной полосой безопаснее, чем со сплошной линией. Чем шире полоса, тем меньше риска при небольшом отклонении от курса.
Числовой пример: w=[3,4] → ||w||=5, margin=2/5=0.4. w=[1,0] → ||w||=1, margin=2 — этот вариант SVM предпочтёт как более устойчивый.
Именно поэтому в жёстком линейном случае SVM можно читать как задачу: разделить данные и одновременно минимизировать норму весов. Геометрическая устойчивость и регуляризация здесь оказываются одной и той же идеей, только записанной разными языками.

Kernel trick математически

Самая сильная идея kernel methods в том, что нам не нужно явно строить огромное нелинейное feature space. Достаточно уметь быстро считать скалярное произведение в этом скрытом пространстве. Тогда алгоритм ведёт себя так, будто он работает в гораздо более богатом представлении, хотя в коде мы видим только kernel-функцию между исходными объектами.

K(x_i, x_j) = ⟨φ(x_i), φ(x_j)⟩
Что означают символы
K(x_i, x_j)kernel-функция — «сходство» двух объектовСкаляр: насколько два объекта похожи в скрытом feature space φ. Вычисляется быстро напрямую, без явного построения φ.
φ(x_i), φ(x_j)нелинейное отображение в feature spaceФункция, переводящая объект x из исходного пространства в (потенциально бесконечномерное) признаковое пространство. Kernel trick позволяет не вычислять φ явно.
⟨·, ·⟩скалярное произведение в feature spaceМера угла и длины в feature space. K(x_i,x_j) = ⟨φ(x_i),φ(x_j)⟩ — kernel = скалярное произведение образов объектов.
x_i, x_jдва объекта в исходном пространствеПара объектов, для которых вычисляем сходство. Kernel позволяет SVM работать с любыми объектами, для которых определена K (тексты, графы, изображения).
Интуиция: Kernel trick: «считаем скалярное произведение в огромном пространстве φ, не строя его». Линейный SVM в feature space φ ≡ нелинейный SVM в исходном пространстве x.
Аналогия: Как проверить сходство текстов через tf-idf без явного вычисления вектора: достаточно знать скалярное произведение (K), а не сам вектор (φ).
Числовой пример: Линейное ядро: K(x_i,x_j)=x_i^T x_j. x_i=[1,2], x_j=[3,1]. K=1·3+2·1=5. Это ровно то, что дало бы скалярное произведение в исходном пространстве.
K_RBF(x_i, x_j) = exp(-γ ||x_i - x_j||²)
Что означают символы
K_RBF(x_i, x_j)RBF kernel (сходство)Гауссова мера схожести двух объектов SVM: от 0 (далеко) до 1 (совпадение).
γширина RBFПараметр затухания сходства: большой γ → только ближайшие объекты влияют на решение SVM.
||x_i − x_j||²квадрат евклидова расстоянияМера удалённости двух объектов x_i и x_j в исходном пространстве признаков.
x_i, x_jдва объектаПара обучающих примеров, для которых вычисляется kernel-сходство.
Интуиция: RBF превращает "физическое расстояние" в "сходство". Большой γ → SVM опирается только на ближайших соседей, малый → глобально гладкая граница.
Аналогия: Как влияние голоса на встрече: γ регулирует, насколько далеко "слышны" support vectors при принятии решения.
Числовой пример: x_i=(0,0), x_j=(1,1): ||x_i−x_j||²=2. γ=0.5 → K=e^{−1}≈0.37. γ=0.1 → K=e^{−0.2}≈0.82.

Это и есть kernel trick: вместо явного φ(x) мы напрямую считаем сходство, которое соответствует скалярному произведению в скрытом пространстве. RBF особенно силён, потому что задаёт гладкую локальную “область влияния” вокруг каждого обучающего примера.

Как читать C и gamma вместе

Большой C
Модель меньше прощает ошибки на train и сильнее пытается подогнаться под каждый проблемный объект.
Большой gamma
RBF-ядро делает influence radius очень маленьким, и граница становится более локальной и изогнутой.
Опасная пара
Большой C и большой gamma часто дают очень гибкую, но переобученную границу.

Полезная инженерная интуиция такая: C контролирует, насколько сильно мы наказываем ошибки, а gamma — насколько локальной будет сама геометрия решения. Поэтому их почти всегда надо читать вместе, а не как независимые ручки.

Теория глубже

C и gamma управляют разными сторонами сложности RBF-SVM

gamma задаёт радиус влияния точки в RBF kernel. Большой gamma делает влияние локальным: boundary может изгибаться вокруг отдельных объектов. Маленький gamma делает похожесть глобальной: boundary становится гладкой.

C решает, насколько сильно модель обязана исправлять нарушения этой boundary. Поэтому опасная комбинация - большой gamma и большой C: локальная гибкость плюс высокий штраф ошибок часто превращают SVM в memorization.

  • Большой gamma + большой C: максимальная гибкость и риск overfit.
  • Маленький gamma + маленький C: гладко, но может underfit.
  • Grid search обычно смотрит C и gamma вместе в логарифмической шкале.
Теоретический слой

SVM как максимизация зазора

SVM ищет разделяющую границу с максимальным margin между классами. Модель не просто классифицирует train points, а выбирает границу с максимальным запасом до критичных объектов.

В soft-margin SVM разрешены нарушения margin. Параметр C задаёт компромисс между шириной margin и ценой ошибок на train.

Геометрическая интуиция

  • Support vectors лежат на margin или нарушают его; именно они определяют границу.
  • Большой margin обычно даёт более устойчивое решение.
  • Kernel SVM строит линейную границу в скрытом feature space, которая выглядит нелинейной в исходном пространстве.

Вероятностная интерпретация

  • SVM напрямую не оценивает вероятность, а оптимизирует margin-based score.
  • Для probabilities нужна calibration: Platt scaling или isotonic regression.
  • Hinge loss заботится о стороне margin, а не о калиброванной уверенности.

Оптимизационный смысл

  • Hinge loss выпуклый, но недифференцируемый в точке margin.
  • Большой C меньше терпит нарушения, маленький C сильнее регуляризует.
  • Kernel matrix дорогая по памяти и времени на больших данных.

Практика и типичные ошибки

  • Всегда масштабируй признаки перед SVM.
  • Для RBF тюнь C и gamma вместе.
  • На больших датасетах проверь linear SVM или approximate kernels.

Мини-пример: Если C слишком велик, SVM может согнуть границу вокруг noisy points. При меньшем C модель допустит нарушения, но сохранит широкий margin.

Как закрепить тему: SVM

Этот блок связывает теорию с практическим решением: где применять идею, какие ошибки проверять и какие ручки менять в эксперименте.

Когда применять

  • Используй тему «SVM» как практический инструмент для табличных данных, baseline-моделей и быстрых проверок гипотез.
  • Классические модели особенно полезны, когда важны скорость, интерпретируемость, небольшой датасет или строгая валидация.
  • Перед нейросетью проверь, какой результат даёт простой baseline: он задаёт нижнюю планку качества.

Типичные ошибки

  • Подбирать модель до постановки метрики: без правильной метрики можно оптимизировать не ту задачу.
  • Допускать leakage: scaling, encoding, feature selection и tuning должны обучаться только на train внутри pipeline.
  • Сравнивать модели без одинакового split и одинакового preprocessing.

Что тюнить

  • Начни с данных: пропуски, выбросы, encoding, scaling и баланс классов часто важнее выбора алгоритма.
  • Потом меняй сложность модели: регуляризацию, глубину, число соседей, число деревьев или learning rate в зависимости от темы.
  • Фиксируй validation protocol до тюнинга, иначе улучшение может быть подгонкой под конкретный split.

Упражнения: Support Vector Machines

Проверка темы - понимать margin, support vectors и цену ошибок через \(C\).

[Foundational] Margin руками

Объясни, почему SVM выбирает не любую разделяющую прямую, а прямую с максимальным margin. Решение: большой margin делает решение устойчивее к малым изменениям данных.

[Foundational] Роль support vectors

Почему не все train-объекты влияют на итоговую границу? Решение: границу задают точки около margin или нарушающие его; далёкие уверенные точки не меняют optimum.

[Intermediate] Параметр C

Что меняется при очень большом и очень маленьком \(C\)? Решение: большой \(C\) сильнее штрафует ошибки и может overfit; маленький \(C\) допускает ошибки ради широкого margin.

[Intermediate] Код SVM pipeline

Собери Pipeline(StandardScaler, SVC) и GridSearchCV по \(C\), gamma, kernel. Решение: scaler внутри pipeline обязателен для корректного CV.

[Hard] Hinge loss vs log loss

Почему SVM не стремится давать вероятности напрямую? Решение: hinge loss оптимизирует margin и ошибки классификации, а не likelihood вероятностной модели.

Проверь себя

Вопросы для самопроверки

  1. Что такое «максимизация зазора» (margin maximization) в SVM?

    SVM ищет гиперплоскость, которая не просто разделяет классы, но делает это с максимальным расстоянием (margin) до ближайших примеров каждого класса — support vectors. Большой margin означает лучшее обобщение: модель не «прижимается» к данным.

  2. Что такое kernel trick и зачем он нужен?

    Kernel trick позволяет неявно работать в пространстве высокой размерности без явного вычисления координат. Функция ядра K(x,z) = φ(x)·φ(z) вычисляет скалярное произведение в исходном пространстве, заменяя дорогое преобразование φ. Это делает SVM практичным для нелинейных задач.

  3. Как параметр C влияет на жёсткость SVM?

    Маленький C — «мягкий» margin: допускаются нарушения (ошибки классификации), граница глаже, модель генерализует лучше, но может недообучиться. Большой C — жёсткий margin: нарушения сильно штрафуются, граница сложнее, возможно переобучение.

  4. Почему SVM плохо масштабируется на большие датасеты?

    SVM решает задачу квадратичного программирования, которая в наивной реализации требует O(n²)–O(n³) вычислений и памяти O(n²) для матрицы ядра. На n>100k примеров это становится непрактичным. Используют SGD-аппроксимации (LinearSVC) или нейросетевые подходы.

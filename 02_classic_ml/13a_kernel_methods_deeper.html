2.13a Kernel Methods глубже

Kernel trick — это не магия, а способ работать с линейной моделью в очень богатом пространстве признаков, не выписывая это пространство явно. Здесь связываем SVM, Mercer, RBF и геометрию похожести.

Почему важно

Kernel — это функция схожести, которая неявно отображает данные в бесконечномерное пространство. Reproducing Kernel Hilbert Space — теоретический фундамент SVM и Gaussian Processes.

Главная формула

K(x, x') = <φ(x), φ(x')>
Что означают символы
K(x, x')kernel-функция — сходство двух объектовСкаляр: «насколько x и x' похожи» в скрытом feature space φ. Вычисляется напрямую, без явного построения φ.
⟨·, ·⟩скалярное произведение в пространстве φВнутренне K определяется через скалярное произведение образов объектов. Это гарантирует корректность (теорема Mercer: K должна быть положительно полуопределённой).
φ(x)нелинейное отображение в feature spaceФункция φ: X → H, переводящая x в (возможно бесконечномерное) пространство Гильберта H. Kernel trick: K(x,x') = ⟨φ(x),φ(x')⟩ — считаем без явного φ.
x, x'два объекта в исходном пространствеПроизвольная пара объектов. Kernel позволяет сравнивать объекты любой природы (векторы, строки, графы), для которых определена функция K.
Интуиция: Kernel = «умный способ сравнивать объекты». Линейная модель в feature space φ даёт нелинейную модель в исходном пространстве. Kernel trick позволяет использовать этот эффект без вычисления φ явно.
Аналогия: Сравниваем тексты по числу общих слов (bag-of-words). Вместо хранения длинных векторов считаем только их dot product — это и есть kernel. Сходство без явного признакового пространства.
Числовой пример: Полиномиальное ядро K(x,x')=(x^T x'+c)^d. x=[1,2], x'=[3,1], c=0, d=2. K=(1·3+2·1)²=5²=25. Неявно работает в 6-мерном пространстве квадратичных признаков.
Теория по месту

Kernel trick считает сходство без явных признаков

Kernel можно читать как скалярное произведение в скрытом пространстве признаков. Мы не строим это пространство явно, но получаем эффект нелинейной модели через матрицу похожестей.

RBF kernel создает локальную похожесть: точки сильно влияют друг на друга только рядом. Gamma управляет радиусом этой локальности.

Ядро говорит не “какие у точки признаки”, а насколько две точки похожи в некотором скрытом feature space. Линейная модель в этом пространстве может соответствовать очень нелинейной границе в исходных координатах.

Kernel trick работает, потому что во многих алгоритмах данные входят только через скалярные произведения. Если заменить их на K(x, x'), алгоритм остаётся линейным “внутри”, но ведёт себя нелинейно “снаружи”.
Теория глубже

Kernel trick скрывает явное пространство признаков

Kernel можно читать как быстрый способ посчитать inner product после некоторого преобразования признаков. Мы не строим огромный вектор phi(x) явно, но получаем значение похожести так, будто работаем в этом пространстве.

Это полезно, когда разделение в исходных признаках сложное, но становится линейным после нелинейного отображения. SVM затем строит линейную границу уже в скрытом feature space.

  • Polynomial kernel добавляет взаимодействия признаков.
  • RBF kernel создаёт очень богатое локальное пространство похожести.
  • Kernel matrix хранит попарные сходства объектов и быстро растёт как O(n²).

Mercer коротко и по делу

Хорошее ядро должно порождать симметричную положительно полуопределённую Gram matrix: K_ij = K(x_i, x_j), K = K^T, z^T K z ≥ 0
Как не ошибиться

Kernel должен быть корректным и уместным

Не всякая функция похожести является хорошим kernel. Условие положительной полуопределенности нужно, чтобы оптимизация оставалась корректной.

Слишком большой gamma делает каждый объект почти отдельным островом, слишком маленький сглаживает все в почти линейную картину.

Это и есть практический смысл теоремы Mercer: если матрица похожести ведёт себя как валидное скалярное произведение, то существует пространство признаков, в котором это ядро действительно равно внутреннему произведению.

Не каждая “похожесть” годится как kernel. Если Gram matrix не PSD, алгоритм может терять выпуклость и понятную геометрию.
Теория глубже

Корректный kernel должен вести себя как скалярное произведение

Mercer condition в практическом смысле говорит: kernel matrix должна быть positive semidefinite. Тогда её можно интерпретировать как матрицу скалярных произведений в некотором feature space.

Если функция похожести не PSD, оптимизационная задача SVM может потерять выпуклость и стабильность. Поэтому не любая красивая мера похожести автоматически является хорошим kernel.

  • PSD matrix не создаёт отрицательных квадратов длины.
  • Симметрия важна: похожесть x с y равна похожести y с x.
  • Domain-specific similarity нужно проверять, прежде чем использовать как kernel.

RBF и почему gamma так важен

K_RBF(x, x') = exp(-γ ||x - x'||²)
Что означают символы
K_RBF(x, x')RBF kernel (сходство)Гауссова мера схожести двух объектов: от 0 (далеко) до 1 (x=x').
γширина RBFУправляет скоростью затухания сходства: большой γ → только ближайшие соседи влияют на модель.
||x − x'||²квадрат евклидова расстоянияМера удалённости объектов x и x' в исходном пространстве признаков.
x, x'два объектаПара точек, для которых вычисляется kernel-сходство.
Интуиция: RBF kernel задаёт "гауссово влияние": объекты близко друг к другу очень похожи (K≈1), далеко — почти не влияют (K≈0). γ задаёт радиус этой "зоны влияния".
Аналогия: Как лампочка: ярко в точке, быстро гаснет с расстоянием. γ регулирует мощность — насколько далеко распространяется свет.
Числовой пример: ||x−x'||²=1, γ=1 → K=e^{−1}≈0.37; γ=0.1 → K=e^{−0.1}≈0.90.

Маленький gamma

Похожесть затухает медленно. Модель гладкая, решение более глобальное, граница менее извилистая.

Большой gamma

Похожесть быстро падает. Каждая точка влияет локально, boundary становится гибкой, но риск переобучения растёт.

Теория глубже

gamma в RBF - это обратная ширина локального внимания

RBF kernel быстро уменьшает похожесть с расстоянием. gamma управляет скоростью этого затухания: при большом gamma только почти идентичные точки сильно похожи, при маленьком gamma похожесть размазана шире.

Геометрически gamma задаёт размер локального района, в котором обучающий объект может влиять на prediction. Поэтому он тесно связан с density данных и масштабом признаков.

  • Перед RBF почти всегда нужен scaling.
  • Большой gamma делает kernel matrix ближе к единичной матрице.
  • Маленький gamma делает многие точки похожими и сглаживает boundary.

Интерактив: kernel matrix

Смотри, как меняется матрица похожести при изменении gamma. Чем ярче диагональ и темнее всё вокруг, тем локальнее становится модель.

Что менять и как интерпретировать

Как читать визуализацию

Как читать kernel-карту

  • Яркие области похожести показывают, какие точки модель считает соседними в скрытом пространстве.
  • У RBF ширина пятен зависит от gamma.
  • Если похожесть слишком локальна, модель легко запоминает train.

Теоретический слой

Kernel methods как неявная геометрия похожести

Kernel trick позволяет вычислять dot product в feature space, не строя сами признаки явно. Это полезно, когда явное пространство огромное или бесконечномерное.

Kernel задаёт не просто функцию, а смысл похожести. От него зависит, какие объекты считаются близкими и какие границы решения считаются простыми.

Геометрическая интуиция

  • Polynomial kernel добавляет взаимодействия признаков.
  • RBF kernel делает влияние точки локальным.
  • Gamma управляет радиусом влияния: большой gamma даёт очень локальные границы.

Вероятностная интерпретация

  • Positive semidefinite kernel можно читать как корректную матрицу похожести.
  • Gaussian Process использует kernel как covariance function.
  • RBF-интуиция связана с локальным влиянием наблюдений.

Оптимизационный смысл

  • Dual formulation выражает решение через train points и kernel matrix.
  • Стоимость растёт с числом объектов.
  • Regularization нужна, чтобы kernel model не подгоняла локальный шум.

Практика и типичные ошибки

  • Подбирай kernel под домен и тип признаков.
  • Проверяй C/gamma на логарифмической сетке.
  • Если matrix слишком большая, смотри Nyström или random Fourier features.

Мини-пример: RBF с gamma=100 может считать похожими только почти идентичные объекты, и граница станет пятнистой.

Когда применять

  • Используй тему «Kernel Methods глубже» как практический инструмент для табличных данных, baseline-моделей и быстрых проверок гипотез.
  • Классические модели особенно полезны, когда важны скорость, интерпретируемость, небольшой датасет или строгая валидация.
  • Перед нейросетью проверь, какой результат даёт простой baseline: он задаёт нижнюю планку качества.

Типичные ошибки

  • Подбирать модель до постановки метрики: без правильной метрики можно оптимизировать не ту задачу.
  • Допускать leakage: scaling, encoding, feature selection и tuning должны обучаться только на train внутри pipeline.
  • Сравнивать модели без одинакового split и одинакового preprocessing.

Что тюнить

  • Начни с данных: пропуски, выбросы, encoding, scaling и баланс классов часто важнее выбора алгоритма.
  • Потом меняй сложность модели: регуляризацию, глубину, число соседей, число деревьев или learning rate в зависимости от темы.
  • Фиксируй validation protocol до тюнинга, иначе улучшение может быть подгонкой под конкретный split.

Упражнения: Kernel methods deeper

Главная проверка - понимать kernel trick как скалярное произведение в скрытом feature space.

[Foundational] Kernel matrix

Для трёх объектов опиши форму kernel matrix. Решение: это \(3\times3\) матрица, где элемент \(K_{ij}=k(x_i,x_j)\).

[Foundational] RBF intuition

Что означает большое и маленькое gamma в RBF kernel? Решение: большое gamma делает влияние точек локальным и повышает риск overfit; маленькое gamma сглаживает границу.

[Intermediate] Polynomial kernel

Почему polynomial kernel добавляет interactions? Решение: скалярное произведение в расширенном пространстве содержит произведения признаков.

[Intermediate] Реализуй RBF matrix

Напиши NumPy-функцию \(K_{ij}=\exp(-\gamma\|x_i-x_j\|^2)\). Решение: используй broadcasting или pairwise squared distances.

[Hard] Mercer's condition

Почему kernel должен быть positive semidefinite? Решение: тогда он соответствует корректному скалярному произведению в некотором feature space.

Проверь себя

Вопросы для самопроверки

  1. Какие условия должна выполнять функция, чтобы быть корректным ядром?

    По теореме Мерсера, функция K(x,z) является корректным ядром, если она симметрична и матрица Грама K_ij = K(xᵢ,xⱼ) является положительно полуопределённой для любого набора точек. Это гарантирует, что K соответствует скалярному произведению в некотором пространстве Гильберта.

  2. В чём разница между RBF-ядром и полиномиальным ядром?

    RBF K(x,z) = exp(-γ‖x-z‖²) — локальное ядро, зависящее только от расстояния; γ контролирует «ширину» влияния. Полиномиальное K(x,z) = (xᵀz+c)ᵈ — глобальное, улавливает взаимодействия признаков степени d. RBF универсальнее для большинства практических задач.

  3. Что такое kernel PCA и как он отличается от обычного PCA?

    Kernel PCA применяет обычный PCA в пространстве признаков, заданном ядром: вычисляет собственные векторы kernel matrix K_ij=K(xᵢ,xⱼ). Это позволяет найти нелинейные главные компоненты — например, развернуть спираль или разделить концентрические окружности.

  4. Почему kernel methods работают в «бесконечномерном» пространстве признаков?

    RBF-ядро соответствует скалярному произведению в пространстве бесконечного числа Гауссовых базисных функций. При этом вычислительная стоимость не зависит от размерности этого пространства — нужно лишь вычислить K(xᵢ,xⱼ) = exp(-γ‖xᵢ-xⱼ‖²), что быстро.
